# AWS Web Server Environment Deployment Plan

## Key AWS Services Required

1. **VPC** - Virtual Private Cloud for network isolation
2. **InternetGateway** - Provides internet connectivity
3. **VPCGatewayAttachment** - Attaches Internet Gateway to VPC
4. **Subnet** - Network segment within the VPC
5. **RouteTable** - Defines routing rules for traffic
6. **Route** - Routes internet-bound traffic through the Internet Gateway
7. **SubnetRouteTableAssociation** - Associates subnet with route table
8. **SecurityGroup** - Controls inbound/outbound traffic to EC2 instance
9. **EC2 Instance** - Web server running Apache and PHP
10. **IAM Role** - Grants EC2 instance permission to access S3
11. **IAM InstanceProfile** - Attaches IAM role to EC2 instance
12. **S3 Bucket** - Stores the sample web content ZIP file
13. **S3 Bucket Policy** - Controls access to the S3 bucket

## Properties and Dependencies

### Critical Dependencies:
1. **VPC must be created before InternetGateway attachment**
2. **InternetGateway must be attached to VPC before Route creation**
3. **RouteTable must be created after VPC exists**
4. **Route must reference existing InternetGateway**
5. **SecurityGroup must reference existing VPC**
6. **EC2 Instance must reference existing Subnet, SecurityGroup, and IAM InstanceProfile**
7. **S3 Bucket Policy must reference existing S3 Bucket**

### Property Dependencies:
- `VPCGatewayAttachment` depends on both VPC and InternetGateway
- `SubnetRouteTableAssociation` depends on Subnet and RouteTable
- `EC2 Instance` depends on Subnet, SecurityGroup, and IAM InstanceProfile
- `S3 Bucket Policy` depends on S3 Bucket

## Safe AWS Defaults (for missing specifications)

### Network Configuration:
- **VPC CIDR**: `10.0.0.0/16` (standard private network)
- **Subnet CIDR**: `10.0.1.0/24` (within VPC range)
- **Availability Zone**: `Fn::Select [0, Fn::GetAZs]` (first AZ in region)

### EC2 Configuration:
- **Instance Type**: `t2.micro` (as specified)
- **KeyName**: `default-key` (safe default, user can override)
- **AMI**: `resolve:ssm:/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2` (latest Amazon Linux 2)
- **UserData**: Bash script for Apache/PHP installation and S3 content retrieval

### Security Configuration:
- **SSH Port**: `22` (standard SSH)
- **HTTP Port**: `80` (standard HTTP)
- **CIDR for SSH**: `0.0.0.0/0` (world access - note: this should be restricted in production)

### S3 Configuration:
- **Bucket Name**: Auto-generated unique name using `!Ref AWS::StackName`
- **Object Key**: `sample-web-content.zip` (fixed name for simplicity)

## Dynamic Resource Fetching Strategy

### AMI Retrieval:
Use SSM Parameter Store to dynamically fetch the latest Amazon Linux 2 AMI:
```yaml
!Sub 'resolve:ssm:/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2'
```

### Availability Zone Selection:
Use `Fn::GetAZs` and `Fn::Select` to dynamically select the first available AZ:
```yaml
Fn::Select:
  - 0
  - Fn::GetAZs: !Ref AWS::Region
```

### S3 Object URL Construction:
Use `Fn::Sub` to construct the S3 URL for the user data script:
```yaml
Fn::Sub: 'https://s3.${AWS::Region}.amazonaws.com/${S3Bucket}/${S3Key}'
```

## Security Group Configuration

### Inbound Rules:
1. **SSH (Port 22)**: From `0.0.0.0/0` (world access - should be restricted in production)
2. **HTTP (Port 80)**: From `0.0.0.0/0` (world access for web content)

### Outbound Rules:
- **All traffic (Port 0-65535)**: To `0.0.0.0/0` (full internet access for package installation)

## EC2 User Data Script

The user data script will:
1. Update system packages
2. Install Apache and PHP
3. Start Apache service and enable on boot
4. Install AWS CLI
5. Download ZIP file from S3
6. Extract content to `/var/www/html/`
7. Set appropriate permissions

## IAM Role Configuration

The EC2 instance will need an IAM role with policy allowing:
- `s3:GetObject` on the specific bucket and object
- `s3:ListBucket` on the specific bucket

This ensures the instance can only access the specific S3 resources it needs.

## S3 Bucket Policy

The bucket policy will grant the EC2 instance's IAM role permission to:
- Get objects from the bucket
- List the bucket contents

This follows the principle of least privilege.

## Template Structure Plan

The CloudFormation template will follow this logical order:
1. Parameters (with safe defaults)
2. Mappings (if needed)
3. Resources in dependency order:
   - VPC and networking components
   - Security Group
   - IAM Role and InstanceProfile
   - S3 Bucket and Policy
   - EC2 Instance
4. Outputs (public IP, DNS name, etc.)

This architecture ensures all dependencies are satisfied and the environment is self-contained, with the EC2 instance automatically configured and populated with content from S3 upon launch.
