1. **EC2** - For the virtual server instance

6. **IAM Instance Profile** - To grant SSM permissions to the EC2 instance

## Properties and Dependencies Analysis

### VPC Dependencies
- VPC must exist before creating the EC2 instance
- Internet Gateway must be attached to VPC before creating route tables
- Route tables must be created before associating subnets
- Subnets must exist before creating the EC2 instance

### Security Group Dependencies
- Security Group must be created before associating with EC2 instance
- VPC must exist before creating Security Group

### IAM Dependencies
- IAM Role must be created before creating Instance Profile
- Instance Profile must be created before launching EC2 instance

### VPC Configuration
- **CIDR Block**: `10.0.0.0/16` (standard private network)
- **Subnet CIDR**: `10.0.1.0/24` (single public subnet)
- **Availability Zone**: Dynamically selected using `Fn::GetAZs` and `Fn::Select`

### EC2 Configuration
- **Instance Type**: `t3.micro` (as specified)
- **KeyName**: `default-key` (safe default parameter)
- **AMI**: Dynamically fetched using SSM Parameter Store
- **Volume Size**: `8` GB (standard minimum)

### Security Group Configuration
- **SSH Port**: `22` (standard SSH port)
- **CIDR Range**: `0.0.0.0/0` (allow all for demonstration, should be restricted in production)

### Availability Zone Selection
I'll use:
```
Fn::Select: [0, Fn::GetAZs: !Ref 'AWS::Region']
```

This dynamically selects the first available AZ in the region.

## Security Considerations

### IAM Permissions
The EC2 instance will need the `AmazonSSMManagedInstanceCore` policy to enable SSM access. This will be attached via an IAM role and instance profile.

### Security Group Design
The security group will:
- Allow SSH access (port 22) from anywhere (0.0.0.0/0)
- Allow all outbound traffic (0.0.0.0/0 on all ports)
- Be minimal and specific to this instance

## Architecture Flow

1. Create VPC with public subnet
2. Create Internet Gateway and attach to VPC
3. Create route table with default route to Internet Gateway
4. Associate route table with public subnet
5. Create IAM role with SSM permissions
6. Create instance profile from IAM role
7. Create security group allowing SSH and outbound traffic
8. Fetch latest Amazon Linux 2 AMI using SSM Parameter Store
9. Launch EC2 instance with all dependencies attached

## Parameter Design

I'll create a single parameter for the KeyName with a safe default:
- **KeyName**: Type=String, Default='default-key'

This ensures the template can deploy without manual input while allowing customization.

## Missing Information Handling

Since specific CIDR blocks weren't provided, I'll use standard defaults (10.0.0.0/16 for VPC, 10.0.1.0/24 for subnet).

Since no specific security requirements were mentioned, I'll implement a basic security group allowing SSH access and all outbound traffic.

The architecture is designed to be minimal yet functional, with all dependencies properly ordered and safe defaults provided for all parameters.
