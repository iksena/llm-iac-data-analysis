# AWS Glue CloudTrail Crawler Deployment Plan

## Key AWS Services Required
1. **AWS Glue Database** - To store metadata tables for CloudTrail logs
2. **AWS Glue Crawler** - To automatically infer schemas and create tables
3. **AWS CloudFormation** - To orchestrate the deployment
4. **AWS S3** - To store CloudTrail logs (referenced but not created)

## Architecture Analysis

### Resource Dependencies
1. **Glue Database** must be created before the **Glue Crawler** can reference it
2. **S3 Bucket** (`s3://kk-admin/cloud-trail/AWSLogs/`) must exist before the crawler can be configured
3. No VPC/Internet Gateway dependencies since Glue is serverless

### Missing Properties & Safe Defaults

**Missing Properties:**
- No specific Glue Database name provided
- No specific Crawler name provided
- No specific IAM role for Glue operations provided
- No specific schedule for crawler execution provided

**Safe Defaults I'll Use:**
- Glue Database: `cloudtrail_logs_db` (descriptive and follows naming convention)
- Glue Crawler: `cloudtrail_crawler` (clear purpose)
- IAM Role: Generated dynamically with appropriate permissions
- Crawler Schedule: `ON_DEMAND` (manual execution for control)
- Table Prefix: `cloudtrail_` (as specified in requirements)

### IAM Role Design

The Glue Crawler needs permissions to:
- Read CloudTrail logs from S3
- Create/modify Glue metadata in the Data Catalog
- Write logs to CloudWatch

I'll create an IAM role with:
- `AWSGlueServiceRole` managed policy
- Custom S3 read permissions for the CloudTrail bucket
- CloudWatch Logs permissions

### Crawler Configuration

Based on the requirements:
- **Database**: `cloudtrail_logs_db`
- **S3 Target**: `s3://kk-admin/cloud-trail/AWSLogs/`
- **Table Prefix**: `cloudtrail_` (as specified)
- **Schema Updates**: `UPDATE_IN_DATABASE` (handle schema updates)
- **Classification**: `json` (CloudTrail logs are JSON format)
- **Recrawl Policy**: `CRAWL_NEW_FOLDERS_ONLY` (efficient for append-only logs)

### Athena Integration

The crawler will automatically create tables that Athena can query. The JSON classification will ensure proper schema inference for CloudTrail's nested JSON structure.

## Detailed Implementation Plan

### 1. Parameters Section
- No parameters needed since all values have safe defaults
- This ensures the template deploys without manual input

### 2. Resources to Create

**A. IAM Role for Glue**
- Type: `AWS::IAM::Role`
- Policies: `AWSGlueServiceRole`, custom S3 policy for CloudTrail bucket
- Trust relationship: Allows Glue service to assume role

**B. Glue Database**
- Type: `AWS::Glue::Database`
- Name: `cloudtrail_logs_db`
- Catalog ID: Use `AWS::AccountId` pseudo-parameter

**C. Glue Crawler**
- Type: `AWS::Glue::Crawler`
- Database name: Reference to Glue Database
- Role: Reference to IAM Role
- Targets: S3 target with CloudTrail path
- Schema change policy: `UPDATE_IN_DATABASE`
- Recrawl policy: `CRAWL_NEW_FOLDERS_ONLY`
- Table prefix: `cloudtrail_`
- Classification: `json`

### 3. Security Considerations
- Least privilege IAM permissions (only S3 read access to CloudTrail bucket)
- No public access configurations
- All resources scoped to the account

### 4. Error Handling & Validation
- The template will validate that the S3 path exists (though it won't create it)
- Crawler will handle schema evolution automatically
- All resource names are deterministic for consistent deployments

### 5. Deployment Sequence
1. IAM Role creation
2. Glue Database creation  
3. Glue Crawler creation (references previous resources)

This architecture provides a complete, automated solution for cataloging CloudTrail logs in Glue, enabling structured querying via Athena without requiring manual configuration or parameter input during deployment.

      AssumeRolePolicyDocument:
        Version: '2012-10-17'
        Statement:
          - Effect: Allow
            Principal:
              Service: [glue.amazonaws.com]
            Action: sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/service-role/AWSGlueServiceRole
      Policies:
        - PolicyName: S3CloudTrailAccess
          PolicyDocument:
            Version: '2012-10-17'
            Statement:
              - Effect: Allow
                Action: ['s3:Get*','s3:List*']
                Resource: 'arn:aws:s3:::kk-admin/cloud-trail/AWSLogs/*'
