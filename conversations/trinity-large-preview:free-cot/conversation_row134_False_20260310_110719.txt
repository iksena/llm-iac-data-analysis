# AWS Lambda Function Deployment Architecture Plan

## Key AWS Services Required

1. **AWS Lambda** - The core compute service for running the Node.js function
2. **IAM (Identity and Access Management)** - For creating the execution role and policy
3. **AWS Systems Manager (SSM) Parameter Store** - For dynamic resource fetching (though not needed for Lambda)
4. **CloudWatch Logs** - For log storage and monitoring (implicitly required by Lambda)

## Properties and Dependencies

The Lambda function requires:
- Function name and description
- Runtime environment (Node.js)
- IAM execution role with proper permissions
- Inline code implementation
- Handler specification
- Memory allocation and timeout settings

Dependencies:
- IAM role must be created before Lambda function
- IAM policy must be attached to role before Lambda creation
- CloudWatch Logs implicitly created by Lambda service

## Safe AWS Defaults

Since specific properties weren't provided, I'll use these safe defaults:

- **Runtime**: `nodejs20.x` (latest LTS Node.js runtime)
- **Memory**: `128` MB (minimum allowed, sufficient for basic logging)
- **Timeout**: `3` seconds (default, adequate for simple operations)
- **Handler**: `index.handler` (standard Lambda convention)
- **IAM Policy**: `arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole` (standard logging permissions)

## Architecture Plan

### 1. IAM Role Creation
- Create an IAM role named `LambdaExecutionRole`
- Use the trusted entity `lambda.amazonaws.com`
- Attach the AWS managed policy `AWSLambdaBasicExecutionRole` for CloudWatch Logs access

### 2. Lambda Function Configuration
- Function name: `BasicLoggingFunction`
- Runtime: `nodejs20.x`
- Handler: `index.handler`
- Memory: `128` MB
- Timeout: `3` seconds
- Inline code: Node.js function that logs event details and returns input value

### 3. Code Implementation Strategy
The Lambda function will:
- Accept an event object as input
- Log the entire event object using `console.log()`
- Return the received event object
- This provides visibility into what triggers the function while maintaining idempotency

### 4. Security Considerations
- Use least-privilege principle through managed IAM policy
- No hardcoded secrets or sensitive data
- Function has only necessary permissions for logging

### 5. Deployment Sequence
1. Create IAM role with proper trust relationship
2. Attach managed policy to IAM role
3. Create Lambda function with inline code
4. Function automatically creates CloudWatch Log Group

This architecture provides a minimal, secure, and functional Lambda deployment that meets the business requirements while following AWS best practices.
