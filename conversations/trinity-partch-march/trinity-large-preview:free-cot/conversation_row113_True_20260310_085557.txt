No complex dependencies like VPCs, Internet Gateways, or Elastic IPs are involved in this template.

## 5. Architecture Plan

The template will follow this structure:

### Template Parameters
- **TopicName**: Parameter for SNS topic name with default value `EventBridgeDemoTopic`
- **EventSourceName**: Parameter for event source name with default value `demo.cli`

### Resources to Create

1. **SNS Topic Resource**
   - Creates the SNS topic with the specified name
   - Uses the default display name

2. **SNS Topic Policy Resource**
   - Creates a topic policy that grants EventBridge permissions to publish to the SNS topic
   - References the SNS topic ARN

3. **EventBridge Rule Resource**
   - Creates an EventBridge rule with an event pattern matching the specified source
   - Targets the SNS topic
   - References both the SNS topic ARN and the SNS topic policy ARN

### Event Pattern Design

The event pattern will match events from the specified source (`demo.cli`) using the following structure:

```yaml
EventPattern:
  source:
    - "demo.cli"
```

This pattern ensures that only events from the `demo.cli` source are matched and forwarded to the SNS topic.

### Security Considerations

- The SNS topic policy explicitly grants EventBridge permissions to publish to the topic
- No IAM roles or policies are needed beyond the SNS topic policy
- The template follows the principle of least privilege by only granting necessary permissions

1. **AWSTemplateFormatVersion**: Latest version (2010-09-09)
2. **Description**: Clear description of the template's purpose
3. **Parameters**: For topic name and event source name with safe defaults

5. **Outputs**: SNS topic ARN for reference

### Compliance with Best Practices

The template will adhere to all the critical AWS best practices and linting rules:

- All parameters will have safe default values
- No hardcoded values will be used
- Proper resource dependencies will be defined
- Security best practices will be followed
- The template will be cfn-lint compliant

This architecture provides a simple, secure, and maintainable solution for forwarding EventBridge events from the `demo.cli` source to an SNS topic.
