4. **Logging Pipeline**: CloudWatch logs → S3 bucket
5. **Monitoring**: CloudWatch metrics and alarms

## 6. Security Considerations

- **IAM Role**: Least privilege principle - only SSM and S3 permissions needed
- **Security Group**: Restrict HTTP/HTTPS to specific CIDR blocks
- **S3 Bucket Policy**: Bucket policy with proper resource-based permissions
- **SSM Document**: Use `DocumentFormat: YAML` with secure execution

3. Create VPC and subnet references (using defaults)

### Phase 2: Instance and Automation

2. Create SSM document with Nginx installation script
3. Create SSM association targeting instances with specific tag

### Phase 3: Monitoring and Logging
1. Configure CloudWatch metrics collection
2. Set up log forwarding to S3
3. Create basic alarms for instance health

The SSM association will use a filter to target instances:

This ensures only instances with the specific tag trigger the Nginx installation.
