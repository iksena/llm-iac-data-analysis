# AWS Batch Environment CloudFormation Architecture Plan

## Key AWS Services Required

1. **VPC** - Virtual Private Cloud for network isolation
2. **Subnet** - EC2 instance placement within the VPC
3. **Internet Gateway** - Internet connectivity for EC2 instances
4. **Route Table** - Network routing configuration
5. **S3 Bucket** - Storage for input/output data
6. **IAM Roles & Policies** - Permissions for Batch service and EC2 instances
7. **AWS Batch Components**:
   - **Compute Environment** - EC2-based compute resources
   - **Job Queue** - Job scheduling and prioritization
   - **Job Definition** - Container configuration and execution parameters
8. **Security Groups** - Network access control for EC2 instances

## Properties and Dependencies

### VPC Dependencies
- Internet Gateway must be attached to VPC before creating Route Table entries
- Route Table must be associated with Subnet after creation
- Subnet requires VPC ID as dependency

### AWS Batch Dependencies
- IAM Roles must exist before creating Compute Environment
- Compute Environment must exist before creating Job Queue
- Job Definition can be created independently but references Compute Environment

### S3 Dependencies
- Bucket must be created before applying Bucket Policy
- IAM Roles need S3 permissions granted via Policy

## Safe AWS Defaults (Explicitly Stated)

### Network Configuration
- **VPC CIDR**: `10.0.0.0/16` (standard private range)
- **Subnet CIDR**: `10.0.1.0/24` (single subnet for simplicity)
- **Availability Zone**: Dynamically selected using `Fn::GetAZs` and `Fn::Select`

### Compute Configuration
- **Instance Type**: `m5.large` (as specified)
- **Min vCPUs**: `0` (scale to zero when idle)
- **Max vCPUs**: `16` (moderate scale)
- **Desired vCPUs**: `0` (no pre-provisioned capacity)

### Docker Image
- **Default Image**: `amazonlinux` (as specified)
- **Container Command**: Sample script for AWS CLI operations

## Dynamic Resource Fetching Strategy

### AMI Retrieval
- Use SSM Parameter Store to fetch latest Amazon Linux 2 AMI:
  - Parameter: `/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2`
  - Method: `Fn::ImportValue` or direct SSM reference

### Availability Zones
- Use `Fn::GetAZs` to retrieve region-specific AZs
- Use `Fn::Select` to pick first available AZ:
  - `Fn::Select: [0, Fn::GetAZs: !Ref 'AWS::Region']`

## Security Architecture

### IAM Roles
1. **BatchServiceRole** - For AWS Batch service permissions
2. **EC2SpotFleetRole** - For Spot instance management (if using Spot)
3. **BatchInstanceRole** - For EC2 instances to access S3 and other services

### Security Groups
- **BatchSecurityGroup**: Allows outbound internet access for package installation
- No inbound rules needed (instances only run containers)

## S3 Bucket Configuration

### Modern Approach
- Use `AWS::S3::BucketPolicy` instead of deprecated `AccessControl`
- Policy grants:
  - BatchInstanceRole: Full S3 access for data operations
  - BatchServiceRole: Limited S3 access for job management

### Bucket Naming
- Use `Fn::Sub` to create unique bucket name: `batch-${AWS::AccountId}-${AWS::Region}`

## AWS Batch Configuration Details

### Compute Environment
- Type: `MANAGED`
- Service Role: `AWSBatchServiceRole`
- Instance Role: `EC2SpotFleetRole` or `EC2InstanceProfile`
- Instance Types: `[m5.large]`
- Allocation Strategy: `BEST_FIT` (or `SPOT_CAPACITY_OPTIMIZED` for cost savings)

### Job Queue
- Priority: `1` (default)
- Compute Environment Order: Reference to created Compute Environment

### Job Definition
- Type: `container`
- Image: Parameter with default `amazonlinux`
- Command: Sample script for AWS CLI operations
- Execution Role: `BatchInstanceRole`

## Template Structure Plan

1. **Parameters** (with safe defaults)
2. **Resources** (in dependency order)
3. **Outputs** (for easy reference)

## Critical Implementation Notes

### AMI Handling
- Use `AWS::SSM::Parameter` type or direct SSM reference
- Avoid hardcoded AMI IDs for region compatibility

### Security Best Practices
- Use least privilege IAM policies
- Enable VPC endpoints for S3 if needed
- Consider encryption at rest for S3 bucket

### Cost Optimization
- Consider Spot instances for cost savings
- Enable scale-to-zero for compute environment
- Set appropriate max vCPU limits

This architecture provides a production-ready AWS Batch environment with proper networking, security, and automation while adhering to AWS best practices and CloudFormation linting rules.
