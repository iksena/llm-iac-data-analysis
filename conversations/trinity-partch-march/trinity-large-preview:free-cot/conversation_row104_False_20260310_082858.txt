# CloudFormation Template Architecture Plan for EC2 Instance Scheduling Lab Environment

## 1. Key AWS Services Required

Based on the business requirements, the following AWS services are essential:

- **VPC** - To create the isolated network environment
- **EC2** - For deploying the five t3.nano instances and the admin instance

- **SSM Parameter Store** - To dynamically fetch the latest Amazon Linux 2 AMI
- **Security Groups** - To control inbound/outbound traffic for EC2 instances
- **Internet Gateway** - To provide internet connectivity for the public subnet
- **Route Tables** - To route traffic between the VPC and internet
- **Subnets** - To host the EC2 instances in the VPC

## 2. Properties and Dependencies

### Critical Dependencies:
1. **VPC must be created before** the Internet Gateway, Subnets, and Route Tables
2. **Internet Gateway must be attached to VPC** before creating Route Table entries
3. **Route Table must be associated with Subnet** after both are created
4. **IAM Role must be created before** the admin EC2 instance that uses it
5. **Security Group must be created before** EC2 instances that reference it

### Property Dependencies:
- Subnet requires VPC ID
- Internet Gateway requires VPC ID for attachment
- Route Table requires VPC ID and Internet Gateway ID for default route
- EC2 instances require Subnet ID, Security Group ID, and IAM Role ARN (for admin)

### CIDR Blocks:
- **VPC CIDR**: `10.0.0.0/16` (standard private range)

### EC2 Instance Properties:
- **Instance Type**: `t3.nano` (as specified)
- **KeyName**: `default-key` (safe default, user can override)
- **Volume Size**: `8` GB (standard minimum for t3.nano)
- **Volume Type**: `gp2` (standard general purpose SSD)

### Admin Instance User Data:
- Install AWS Instance Scheduler CLI via `pip install aws-instance-scheduler`
- Configure with dummy placeholder values that users must update

### Security Group:

- **Outbound**: Allow all traffic (`0.0.0.0/0`) to all ports (as specified)

```yaml
!Select [0, !GetAZs '']
```

## 5. IAM Role Configuration

- **Permissions**: `lambda:InvokeFunction`, `cloudformation:*`
- **Managed Policies**: `AWS managed policy for Lambda invocation and CloudFormation`
- **Trust Relationship**: EC2 service can assume this role

## 6. Security Group Design

Create a single security group that:
- Allows SSH inbound from anywhere (for admin access)
- Allows all outbound traffic (as specified)
- Is applied to all EC2 instances

## 7. EC2 Instance Tagging Strategy

All instances must be tagged with:
- `environment: dev` (as specified)
- Additional tags for cost allocation and identification

## 8. Admin Instance User Data Script

The user data script will:
1. Install AWS CLI v2
2. Install AWS Instance Scheduler CLI via pip
3. Configure with placeholder values
4. Output configuration status to console

## 9. Template Structure Plan

The CloudFormation template will follow this logical order:
1. Parameters (with safe defaults)
2. Mappings (if needed)
3. Resources in dependency order:
   - VPC
   - Internet Gateway + Attachment
   - Subnet
   - Route Table + Routes + Association
   - Security Group
   - IAM Role + Instance Profile
   - EC2 Instances (5 dev + 1 admin)

## 10. Compliance with AWS Best Practices

The template will:
- Use dynamic AMI fetching via SSM
- Use dynamic AZ selection
- Include safe parameter defaults
- Follow proper resource dependency ordering
- Use latest AWS best practices for IAM and security
- Include appropriate tags for cost allocation
- Use SSM for instance management
