# AWS Client VPN Monitoring Architecture Plan

## Key AWS Services Required

1. **AWS CloudWatch Dashboard** - Central visualization of VPN metrics
2. **AWS CloudWatch Logs Insights** - Query engine for analyzing VPN logs
3. **AWS Client VPN Endpoints** - Source of VPN connection logs
4. **AWS CloudWatch Logs** - Storage for VPN connection logs
5. **AWS IAM** - Permissions for dashboard and query access

## Architecture Overview

The solution will create a CloudWatch Dashboard with pre-configured Logs Insights queries organized by authentication type and usage pattern. The dashboard will display real-time VPN usage statistics including connection durations, data transfer volumes, and user activity.

## Query Organization Strategy

Queries will be organized into logical folders:
- **Connection Analytics** - Duration, frequency, and patterns
- **Data Transfer** - Ingress/egress volumes by endpoint and user
- **User Activity** - Active users, distinct user counts, authentication methods
- **Endpoint Performance** - Per-endpoint usage statistics

## Properties and Dependencies

### CloudWatch Dashboard
- **DashboardName**: `ClientVPNAnalytics-{AWS::Region}-{AWS::AccountId}`
- **DashboardBody**: JSON structure containing widgets for each query result
- **Properties**: Widgets will include tables, bar charts, and time series

### CloudWatch Logs Insights Queries
- **LogGroupName**: Dynamic reference to VPN log group (typically `/aws/vpn-endpoint`)
- **QueryDefinitionName**: Descriptive names for each query
- **QueryString**: Pre-defined SQL-like queries for VPN log analysis

- **Default TimeRange**: Last 24 hours (configurable via Parameter)

### CloudWatch Logs Insights
- **Default LogGroupName**: `/aws/vpn-endpoint` (common default for Client VPN)
- **Default TimeRange**: 24 hours (configurable via Parameter)

## Dynamic Resource Fetching

### VPN Log Group
- Use `Fn::ImportValue` to reference existing VPN log group if available
- Fallback to common default `/aws/vpn-endpoint`

### Region-Specific Handling
- Use `AWS::Region` pseudo-parameter for region-specific configurations
- Use `AWS::AccountId` for resource naming uniqueness

## Query Definitions

### Connection Duration Analysis
```sql
fields @timestamp, @message
| filter @message like /connection established/
| stats avg(duration) as avg_duration, max(duration) as max_duration, min(duration) as min_duration
```

### Data Transfer Analysis
```sql
fields @timestamp, @message, bytes_in, bytes_out
| stats sum(bytes_in) as total_ingress, sum(bytes_out) as total_egress
```

### User Activity Analysis
```sql
fields @timestamp, @message, user_name, authentication_type
| stats count(*) as connection_count by user_name, authentication_type
```

### Endpoint Usage Analysis
```sql
fields @timestamp, @message, endpoint_id
| stats count(*) as connection_count by endpoint_id
```

## Security Considerations

- **IAM Permissions**: Queries will use least-privilege IAM roles
- **Data Protection**: No sensitive data will be exposed in dashboard
- **Access Control**: Dashboard will be accessible only to authorized users
