# Architectural Deployment Plan for S3 Bucket with SQS Event Notifications

## Key AWS Services Required

1. **AWS::S3::Bucket** - To create the S3 bucket for object storage
2. **AWS::SQS::Queue** - To create the SQS queue for receiving notifications
3. **AWS::S3::BucketNotification** - To configure S3 event notifications to the SQS queue
4. **AWS::SQS::QueuePolicy** - To grant S3 service permission to send messages to the queue

## Properties and Dependencies

The deployment follows a straightforward dependency chain:
1. SQS Queue must be created first (it's a dependency for the notification configuration)
2. S3 Bucket is created next
3. Bucket Notification is configured last, referencing both the bucket and queue

No complex dependencies like VPCs, Internet Gateways, or Elastic IPs are required for this simple architecture.

## Safe AWS Defaults

Since no specific properties were provided, I'll use these safe defaults:

- **S3 Bucket Name**: A unique name generated using `!Sub` with the stack name and a timestamp to ensure uniqueness across AWS accounts
- **SQS Queue**: Standard queue with default configuration (noDLQ, no message retention changes)
- **Event Type**: `s3:ObjectCreated:*` to capture all object creation events (PUT, POST, COPY, multipart uploads)
- **IAM Permissions**: Minimal required permissions for S3 to send notifications to SQS

## Dynamic Resource Fetching

No dynamic resource fetching is needed for this template since:
- No AMIs are required (no EC2 instances)
- No SSM parameters needed for other resources
- Standard AWS services with no versioning concerns

## Security Considerations

- The SQS queue will have a policy allowing only the S3 service to send messages
- No public access will be granted to the S3 bucket
- No bucket policies will be created, keeping the bucket private by default
- The notification configuration will be scoped to the specific bucket and queue

## Architecture Plan

The template will be structured as follows:

1. **Parameters Section**: No parameters needed since all values can be safely defaulted
2. **Resources Section**:
   - Create SQS Queue first
   - Create S3 Bucket second
   - Create SQS Queue Policy third (granting S3 service permission)
   - Create S3 Bucket Notification fourth (configuring event-driven notifications)

3. **Intrinsic Functions Used**:
   - `!Sub` for generating unique bucket names
   - `!GetAtt` for referencing queue ARN in policies and notifications
   - `!Ref` for referencing resource names

4. **Linting Compliance**:
   - No hardcoded values that could cause failures
   - All resource references properly scoped
   - No deprecated properties or runtimes
   - Proper dependency ordering to prevent race conditions

## Key Implementation Details

The S3 Bucket Notification will use the `AWS::S3::BucketNotification` resource type with:
- `TopicConfigurations` array containing one configuration
- `Event` set to `s3:ObjectCreated:*`
- `Queue` referencing the SQS queue ARN
- No filters (capturing all object creations)

The SQS Queue Policy will use `AWS::SQS::QueuePolicy` with:
- `Queues` array containing the queue name
- `PolicyDocument` allowing `sqs:SendMessage` from the S3 service principal
- `Condition` ensuring the message comes from the specific bucket ARN

This architecture ensures a secure, event-driven pipeline where any object uploaded to the S3 bucket automatically triggers a message to the SQS queue, enabling downstream processing without polling or manual intervention.

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  S3EventQueue:
    Type: AWS::SQS::Queue
    Properties:
      QueueName: !Sub '${AWS::StackName}-s3-events-queue'

  S3EventQueuePolicy:
    Type: AWS::SQS::QueuePolicy
    Properties:
      Queues:
        - !Ref S3EventQueue
      PolicyDocument:
        Version: '2012-10-17'
        Id: S3EventQueuePolicy
        Statement:
          - Sid: AllowS3ToPublish
            Effect: Allow
            Principal:
              Service: s3.amazonaws.com
            Action: sqs:SendMessage
            Resource: !GetAtt S3EventQueue.Arn
            Condition:
              ArnLike:
                aws:SourceArn: !Sub 'arn:${AWS::Partition}:s3:::${S3Bucket}'

  S3Bucket:
    Type: AWS::S3::Bucket
    Properties:
      BucketName: !Sub '${AWS::StackName}-${AWS::Region}-${AWS::AccountId}-${Timestamp}'
      BucketEncryption:
        ServerSideEncryptionConfiguration:
          - ServerSideEncryptionByDefault:
              SSEAlgorithm: AES256
