             as a JSON object that captures resources, their dependencies, and
             their observable effects BEFORE writing any YAML.
  Phase 2 - Compile that I-IR into a valid AWS CloudFormation YAML template.

The I-IR schema:
{
  "resources": [
    {
      "id":      "<logical name, e.g. MyBucket>",
      "kind":    "<CloudFormation resource type, e.g. AWS::S3::Bucket>",
      "fields":  { "<prop>": "<value or ref>" },
      "effects": ["<tag>, e.g. public-read | encrypted | versioned"]
    }
  ],
  "edges": [
    { "type": "depends",  "from": "<id>", "to": "<id>" },
    { "type": "connects", "from": "<id>", "to": "<id>",
      "proto": "<tcp|udp|https>", "port": <number> }
  ],
  "invariants": ["<free-text constraint, e.g. encryption-at-rest required>"]
}

Wrap the I-IR in <i_ir>...</i_ir> tags.
Wrap the final CloudFormation template in <iac_template>...</iac_template> tags.
Do NOT include raw markdown code fences inside those tags.
