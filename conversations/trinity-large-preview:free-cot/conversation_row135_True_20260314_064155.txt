# Detailed Architectural Deployment Plan

## Key AWS Services Required

1. **AWS::ApiGateway::RestApi** - To create the regional REST API with a mock backend
2. **AWS::ApiGateway::Resource** - To define the `/hello` path resource
3. **AWS::ApiGateway::Method** - To create the GET method for the `/hello` endpoint
4. **AWS::ApiGateway::Stage** - To create the production stage for the API
5. **AWS::ApiGateway::Deployment** - To deploy the API configuration
6. **AWS::CloudFront::Distribution** - To create the CloudFront distribution
7. **AWS::IAM::Role** - For CloudFront Origin Access Identity (OAI) if needed for API Gateway access
8. **AWS::ApiGateway::Account** - To configure CloudWatch logging for API Gateway

## Properties and Dependencies

1. **API Gateway Dependencies:**
   - The `AWS::ApiGateway::Resource` depends on the `AWS::ApiGateway::RestApi`
   - The `AWS::ApiGateway::Method` depends on the `AWS::ApiGateway::Resource`
   - The `AWS::ApiGateway::Deployment` depends on the `AWS::ApiGateway::Method`
   - The `AWS::ApiGateway::Stage` depends on the `AWS::ApiGateway::Deployment`

2. **CloudFront Dependencies:**
   - The `AWS::CloudFront::Distribution` depends on the `AWS::ApiGateway::Stage` being created
   - The distribution needs the API Gateway's invoke URL as the origin

## Safe AWS Defaults

1. **API Gateway:**
   - Region: Use `AWS::Region` pseudo-parameter to ensure regional deployment
   - API Name: Default to "MockAPI" if not specified
   - Resource Path: Default to "/hello" if not specified
   - Method: Default to "GET" if not specified

2. **CloudFront:**
   - PriceClass: Default to "PriceClass_100" (US, Canada, Europe)
   - ViewerProtocolPolicy: Default to "redirect-to-https"
   - MinimumProtocolVersion: Default to "TLSv1.2_2021"
   - DefaultRootObject: Default to empty string (not needed for API)

## Dynamic Resource Fetching

1. **AMIs:** Not required for this template as we're using API Gateway and CloudFront only
2. **AZs:** Not required as we're using regional API Gateway

## Security Considerations

1. **HTTPS Enforcement:** CloudFront will enforce HTTPS for end users
2. **Origin Communication:** CloudFront will use HTTPS to communicate with API Gateway
3. **Caching:** Disabled to ensure real-time responses
4. **IAM Permissions:** Minimal permissions required for API Gateway and CloudFront

## Architecture Plan

### Step 1: API Gateway Creation
- Create a regional REST API with a mock backend
- Define the `/hello` resource
- Create a GET method with a mock integration
- Configure the method to return a static JSON response: `{"message": "hello world"}`
- Deploy the API to a production stage

### Step 2: CloudFront Distribution Configuration
- Create a CloudFront distribution with the API Gateway as the origin
- Configure the origin to use the API Gateway's invoke URL
- Set up the default cache behavior to forward all requests to the origin
- Disable caching by setting `DefaultTTL`, `MinTTL`, and `MaxTTL` to 0
- Configure HTTPS redirection for viewers
- Use the latest TLS protocol version

### Step 3: Integration and Testing
- Ensure the CloudFront distribution can reach the API Gateway
- Test the `/hello` endpoint through the CloudFront distribution
- Verify HTTPS enforcement and real-time response behavior

## Key Configuration Details

1. **API Gateway Mock Integration:**
   - Use `AWS::ApiGateway::Integration` with `Type: MOCK`
   - Configure the integration response to return the static JSON

2. **CloudFront Origin:**
   - Use the API Gateway's invoke URL as the origin domain name
   - Configure the origin protocol policy to "https-only"

3. **Cache Behavior:**
   - Set `ForwardedValues` to forward all headers, query strings, and cookies
   - Set all TTL values to 0 to disable caching

4. **Viewer Protocol Policy:**
   - Set to "redirect-to-https" to enforce HTTPS for end users

This architecture ensures a secure, real-time API endpoint accessible through CloudFront with proper HTTPS enforcement and no caching.
