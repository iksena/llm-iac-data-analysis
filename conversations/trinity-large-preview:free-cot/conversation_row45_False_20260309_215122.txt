# Detailed Architectural Deployment Plan for Amazon Data Firehose with Encryption and Logging

## Key AWS Services Required

1. **AWS::KinesisFirehose::DeliveryStream** - The primary service for creating the Data Firehose stream
2. **AWS::Logs::LogGroup** - CloudWatch Log Group for destination error logging
3. **AWS::Logs::LogStream** - CloudWatch Log Stream within the created Log Group
4. **AWS::IAM::Role** - IAM Role for Firehose to access CloudWatch Logs and KMS
5. **AWS::IAM::Policy** - IAM Policy attached to the Firehose role

## Properties and Dependencies

1. **IAM Role Creation** - Must be created before the Firehose delivery stream
2. **Log Group Creation** - Must be created before the Log Stream
3. **Log Stream Creation** - Must be created after the Log Group
4. **IAM Policy Attachment** - Must be attached to the IAM Role before Firehose creation

## Safe AWS Defaults (for missing properties)

- **Region**: `AWS::Region` (inferred from stack deployment)
- **Log Group Name**: `firehose-destination-errors` (default name)
- **Log Stream Name**: `firehose-errors` (default name)
- **KMS Key**: AWS Managed Key for Kinesis (`alias/aws/kinesis`)
- **Delivery Stream Name**: `firehose-delivery-stream` (default name)
- **IAM Role Name**: `FirehoseKinesisRole` (default name)

## Architecture Plan

### 1. IAM Role and Policy Setup
- Create an IAM Role named `FirehoseKinesisRole` with a trust policy allowing Firehose service to assume the role
- Attach an IAM Policy with the following permissions:
  - `firehose:PutRecord`, `firehose:PutRecordBatch` (for Firehose operations)
  - `logs:CreateLogGroup`, `logs:CreateLogStream`, `logs:PutLogEvents` (for CloudWatch Logs access)
  - `kms:Decrypt`, `kms:Encrypt` (for KMS key access)

### 2. CloudWatch Logs Setup
- Create a Log Group named `firehose-destination-errors` in the current region
- Create a Log Stream named `firehose-errors` within the Log Group

### 3. Data Firehose Delivery Stream Configuration
- **Delivery Stream Name**: `firehose-delivery-stream`
- **Delivery Stream Type**: `DirectPut` (default for custom destinations)
- **Encryption Configuration**:
  - **KeyType**: `AWS_OWNED_CMK` (AWS Managed Key)
  - **KeyId**: `alias/aws/kinesis` (default AWS Managed KMS key for Kinesis)
- **Destination Configuration**:
  - **CloudWatch Logs Destination**:
    - **LogGroupARN**: ARN of the created Log Group
    - **LogStreamARN**: ARN of the created Log Stream
    - **ProcessingConfiguration**: Enabled with Lambda processor (if needed)
    - **S3BackupMode**: `Disabled` (since we're using CloudWatch Logs)

### 4. Dynamic Resource Fetching
- Use `AWS::Region` pseudo-parameter for region-specific resources
- Use `AWS::AccountId` pseudo-parameter for ARN construction
- Use `AWS::StackName` pseudo-parameter for resource naming consistency

### 5. Security Considerations
- Use AWS Managed KMS keys to avoid key management overhead
- Implement least privilege IAM policies
- Enable server-side encryption by default
- Use secure ARNs for all resource references

### 6. Error Handling and Monitoring
- Enable CloudWatch Logs for destination errors
- Configure appropriate log retention policies
- Set up metrics and alarms for delivery stream monitoring

### 7. Template Structure
1. Parameters (with safe defaults)
2. Resources (in dependency order):
   - IAM Role
   - IAM Policy
   - Log Group
   - Log Stream
   - Firehose Delivery Stream
3. Outputs (optional: delivery stream ARN, log group ARN)

This architecture ensures a secure, production-ready Data Firehose deployment with proper error logging and encryption, following AWS best practices and CloudFormation linting rules.
