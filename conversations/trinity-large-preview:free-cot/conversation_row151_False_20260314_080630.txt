## Architecture Overview

The solution will create a serverless architecture where:
- A Lambda function provides stock price data
- A Bedrock Agent uses Claude 3 Sonnet to handle natural language queries
- IAM roles ensure secure communication between services
- The agent invokes the Lambda function through a custom action

### 1. Lambda Function
- **Runtime**: Python 3.12 (latest supported)
- **Handler**: `lambda_function.lambda_handler`
- **Memory**: 128 MB (default)
- **Timeout**: 10 seconds (default)
- **Permissions**: Basic Lambda execution role with CloudWatch Logs access

### 2. Bedrock Agent Configuration
- **Model**: Claude 3 Sonnet (or latest available via SSM)
- **Instructions**: "You are a stock market assistant. When users ask about stock prices, use the getStockPrice action with the stock symbol to retrieve current prices."
- **Action Group**: Custom action for stock price lookup

### 3. IAM Roles and Policies
- **Lambda Execution Role**: AWSLambdaBasicExecutionRole
- **Bedrock Agent Role**: Custom role with bedrock:InvokeModel permission
- **Service-to-Service Trust**: Lambda role trusted by Bedrock Agent

- **FunctionName**: `StockPriceLookupFunction`
- **Handler**: `lambda_function.lambda_handler`
- **Runtime**: `python3.12`

- **Code**: Inline code returning hard-coded price of 100

### Bedrock Agent Properties
- **AgentName**: `StockMarketAssistant`

### Action Group Configuration

- Returns JSON: `{"price": 100, "symbol": event["stockSymbol"]}`

## Security and Permissions Plan

### Lambda Execution Role
```yaml
Policies:
  - PolicyName: LambdaLogging
    PolicyDocument:
      Version: "2012-10-17"
      Statement:
        - Effect: Allow
          Action:
            - logs:CreateLogGroup
            - logs:CreateLogStream
            - logs:PutLogEvents
          Resource: "arn:aws:logs:*:*:*"
```

### Bedrock Agent Role
```yaml
Policies:
  - PolicyName: BedrockInvokeModel
    PolicyDocument:
      Version: "2012-10-17"
      Statement:
        - Effect: Allow
          Action: bedrock:InvokeModel
          Resource: "*"
```

## Service Integration Flow

1. User queries Bedrock Agent about stock price
2. Agent recognizes stock-related query
3. Agent invokes `getStockPrice` action with stock symbol
4. Action triggers Lambda function
5. Lambda returns hard-coded price (100)
6. Agent formats response for user

## Error Handling and Validation

- **Lambda Errors**: Return structured error JSON
- **Bedrock Validation**: Ensure stock symbol parameter is provided
- **IAM Trust**: Configure proper service-to-service trust relationships
- **Model Availability**: Check if Claude 3 Sonnet is available in region

## Deployment Sequence

1. Create IAM roles
2. Deploy Lambda function
3. Create Bedrock Agent
4. Configure action group
5. Link agent to Lambda function
6. Test integration

This architecture ensures secure, scalable integration between Bedrock and Lambda while following AWS best practices and CloudFormation linting rules.
