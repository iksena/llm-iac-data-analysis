# Detailed Architectural Deployment Plan for Amazon ECR Repository with Repository Policy

## 1. Key AWS Services Required

Based on the business need, the following AWS services are required:

- **AWS::ECR::Repository** - To create the Amazon ECR repository for storing Kubernetes container images
- **AWS::ECR::RepositoryPolicy** - To define the repository policy granting permissions to the AWS account root user
- **AWS::IAM::ManagedPolicy** (optional) - If we need to create a reusable policy for other services to reference

## 2. Properties and Dependencies

For this ECR repository deployment, the key properties and dependencies are:

- **Repository Name**: Required property for the ECR repository
- **Repository Policy**: Required to grant permissions to the AWS account root user
- **AWS Account ID**: Needed to reference the root user in the policy
- **AWS Region**: Required for ARN construction in the policy

Dependencies:
- No external dependencies like VPCs or Internet Gateways are required for ECR
- The template can be deployed independently

## 3. Safe AWS Defaults

Since specific properties are missing from the business requirement, I will use the following safe defaults:

- **Repository Name**: Default to "kubernetes-repo" (can be parameterized)
- **AWS Account ID**: Will use `AWS::AccountId` pseudo parameter to dynamically reference the current account
- **AWS Region**: Will use `AWS::Region` pseudo parameter for ARN construction
- **Repository Policy Actions**: Use standard ECR actions (ecr:GetDownloadUrlForLayer, ecr:BatchGetImage, ecr:BatchCheckLayerAvailability, ecr:PutImage, ecr:InitiateLayerUpload, ecr:UploadLayerPart, ecr:CompleteLayerUpload)

## 4. Dynamic Resource Fetching

For this specific use case, dynamic resource fetching is not required because:

- ECR repositories don't require AMIs
- No other dynamic resources are needed
- The AWS Account ID and Region can be obtained using pseudo parameters

## Architecture Plan

The CloudFormation template will consist of:

1. **ECR Repository Resource**: Creates the repository with a specified name
2. **Repository Policy Resource**: Attaches a policy granting the AWS account root user permissions to perform critical ECR actions

The repository policy will grant the following permissions to the root user:
- Read operations: GetDownloadUrlForLayer, BatchGetImage, BatchCheckLayerAvailability
- Write operations: PutImage, InitiateLayerUpload, UploadLayerPart, CompleteLayerUpload

This allows the root user to fully manage container images in the repository, which is essential for Kubernetes container image management.

## Security Considerations

- The policy grants permissions to the account root user, which has full administrative access
- This is appropriate for Kubernetes container image management where the root user needs to push and pull images
- No additional IAM roles or users are granted access, keeping the permissions minimal

## Template Structure

The CloudFormation template will follow this structure:

1. Parameters section (with safe defaults)
2. Resources section with:
   - ECR Repository resource
   - ECR Repository Policy resource
3. Outputs section (optional, to return repository details)

This architecture provides a minimal, secure, and production-ready ECR repository with appropriate permissions for Kubernetes container image management.

Description: CloudFormation template to create an Amazon ECR repository with repository policy for Kubernetes container image management

Parameters:
  RepositoryName:
    Type: String
    Default: kubernetes-repo
    Description: Name of the ECR repository to create

Resources:
  ECRRepository:
    Type: AWS::ECR::Repository
    Properties:
      RepositoryName: !Ref RepositoryName

  ECRRepositoryPolicy:
    Type: AWS::ECR::RepositoryPolicy
    Properties:
      RepositoryName: !Ref ECRRepository
      PolicyText:
        Version: '2012-10-17'
        Statement:
          - Sid: RootUserFullAccess
            Effect: Allow
            Principal:
              AWS: !Sub 'arn:aws:iam::${AWS::AccountId}:root'

              - ecr:GetDownloadUrlForLayer
