# AWS CloudFormation Deployment Plan for Web Server Infrastructure

## 1. Key AWS Services Required

Based on the business need, the following AWS services are required:

- **Amazon VPC** - To create the virtual network infrastructure
- **Internet Gateway** - To provide internet connectivity to the VPC
- **Public Subnets** - To host the web servers and ALB in multiple Availability Zones
- **Route Table** - To route traffic from the VPC to the Internet Gateway
- **Application Load Balancer (ALB)** - To distribute traffic across EC2 instances
- **Auto Scaling Group** - To automatically scale EC2 instances based on demand
- **Launch Template** - To define the EC2 instance configuration and bootstrap script
- **EC2 Instances** - To run the Apache HTTP Server
- **IAM Role** - To grant EC2 instances permissions for SSM access
- **Security Groups** - To control inbound and outbound traffic
- **SSM Parameter Store** - To dynamically fetch the latest Amazon Linux AMI

## 2. Properties and Dependencies

The following properties and dependencies must be considered:

- **VPC** must be created before **Subnets**, **Internet Gateway**, and **Route Table**
- **Internet Gateway** must be attached to the VPC before creating routes
- **Subnets** must exist before creating the **ALB** and **Auto Scaling Group**
- **IAM Role** must be created before the **Launch Template** can reference it
- **Security Groups** must be created before they can be referenced by the **ALB** and **EC2 instances**
- **Launch Template** must exist before the **Auto Scaling Group** can use it

## 3. Safe AWS Defaults

Since specific properties are not provided, the following safe AWS defaults will be used:

- **VPC CIDR Block**: `10.0.0.0/16` (standard private range)
- **Public Subnet CIDR Blocks**: `10.0.1.0/24` and `10.0.2.0/24` (two subnets for high availability)
- **EC2 Instance Type**: `t3.micro` (free tier eligible, suitable for web servers)
- **SSH Key Pair Name**: `default-key` (safe default, user can override)
- **ALB Port**: `80` (standard HTTP port)
- **Target Group Port**: `80` (matches ALB port)
- **Auto Scaling Min/Max Instances**: `2` and `4` (ensures high availability with room for scaling)
- **Health Check Grace Period**: `300` seconds (allows time for instances to boot and Apache to start)

## 4. Dynamic Resource Fetching

To avoid hardcoding and ensure the template uses the latest resources:

- **AMI ID**: Use `AWS::SSM::Parameter::Value<AWS::EC2::Image::Id>` to fetch the latest Amazon Linux 2 AMI from SSM Parameter Store
- **Availability Zones**: Use `Fn::GetAZs` and `Fn::Select` to dynamically select AZs based on the region where the template is deployed

## 5. Architecture Overview

The infrastructure will be designed as follows:

1. **VPC Setup**: Create a VPC with two public subnets across different AZs for high availability
2. **Internet Connectivity**: Attach an Internet Gateway and configure route tables to allow internet access
3. **Security Configuration**: Create security groups to allow HTTP traffic from the ALB and SSH access for management
4. **IAM Role**: Create an IAM role with SSM permissions for EC2 instances to allow remote management
5. **Launch Template**: Define the EC2 instance configuration, including a bootstrap script to install Apache and generate a unique HTML page
6. **Application Load Balancer**: Create an ALB with a target group to distribute traffic to the EC2 instances
7. **Auto Scaling Group**: Configure an ASG using the launch template to maintain the desired number of instances across AZs

## 6. Security Considerations

- **IAM Role**: The EC2 instances will have an IAM role with SSM permissions, allowing secure remote access without SSH keys
- **Security Groups**: The web server security group will only allow HTTP traffic from the ALB and SSH traffic from trusted sources
- **ALB Security**: The ALB will be internet-facing but will only forward traffic to healthy instances in the target group

## 7. Bootstrap Script Details

The launch template will include a user data script that:

1. Updates the package manager
2. Installs Apache HTTP Server
3. Starts the Apache service and enables it to start on boot
4. Generates a unique HTML page with instance-specific information (instance ID, private IP, etc.)

This ensures each EC2 instance serves a dynamic page that can be verified through the ALB.

## 8. High Availability and Scalability

- **Multi-AZ Deployment**: Instances are distributed across two AZs for fault tolerance
- **Auto Scaling**: The ASG maintains the minimum instance count and can scale up to handle increased load
- **ALB Health Checks**: The ALB performs health checks on instances and only routes traffic to healthy ones

This architecture provides a robust, scalable, and secure web server infrastructure that meets the business requirements while adhering to AWS best practices.
