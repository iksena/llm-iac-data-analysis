# S3 Bucket for CloudWatch Export Data - Architecture Plan

## Key AWS Services Required
1. **AWS::S3::Bucket** - Main storage for CloudWatch export data
2. **AWS::S3::BucketPolicy** - Security policy for bucket access control
3. **AWS::S3::BucketVersioning** - Versioning configuration
4. **AWS::S3::BucketLifecycleConfiguration** - Automated lifecycle management
5. **AWS::IAM::ManagedPolicy** (optional) - For cross-account access if needed

## Architecture Design Decisions

### Security & Access Control
- **Bucket Policy**: Implement least-privilege access using IAM roles/policies
- **Encryption**: Enable default SSE-S3 encryption for data at rest
- **Access Logging**: Enable server access logging to monitor bucket activity
- **Block Public Access**: Enable all block public access settings to prevent accidental exposure

### Versioning Strategy
- **Status**: Configurable parameter (Enabled/Disabled/Suspended)
- **MFA Delete**: Disabled by default (can be enabled if required)
- **Default**: Disabled (cost optimization)

### Lifecycle Management
- **Current Version Expiration**: Configurable (default: 365 days)
- **Noncurrent Version Expiration**: Configurable (default: 90 days)
- **Abort Incomplete Multipart Uploads**: Enabled (default: 7 days)
- **Storage Class Transitions**: Configurable (default: transition to IA after 30 days)

### Cost Optimization
- **Storage Class Analysis**: Enabled to analyze access patterns
- **Intelligent Tiering**: Optional configuration for frequently accessed data
- **Lifecycle Rules**: Automated transition to cheaper storage classes

## Parameter Configuration (with Safe Defaults)

```yaml
Parameters:
  VersioningStatus:
    Type: String
    Default: Disabled
    AllowedValues:
      - Enabled
      - Disabled
      - Suspended

  CurrentVersionExpirationInDays:
    Type: Number
    Default: 365
    MinValue: 1
    MaxValue: 3653

  NoncurrentVersionExpirationInDays:
    Type: Number
    Default: 90
    MinValue: 1
    MaxValue: 3653

  TransitionToInfrequentAccessDays:
    Type: Number
    Default: 30
    MinValue: 1
    MaxValue: 3653

  BucketName:
    Type: String
    Default: cloudwatch-exports-${AWS::AccountId}
    AllowedPattern: ^[a-z0-9.-]{3,63}$
```

## Resource Dependencies & Properties

### S3 Bucket Configuration
- **BucketName**: Generated with account ID suffix to ensure uniqueness
- **PublicAccessBlockConfiguration**: All settings enabled
- **OwnershipControls**: ObjectOwnership set to BucketOwnerEnforced
- **AnalyticsConfiguration**: Storage class analysis enabled
- **ServerSideEncryptionConfiguration**: SSE-S3 enabled by default

### Versioning Configuration
- **Status**: Dynamically set based on parameter
- **MFADelete**: Explicitly set to Disabled (can be modified if needed)

### Lifecycle Rules
- **AbortIncompleteMultipartUpload**: Configured with default 7-day rule
- **Expiration**: Separate rules for current and noncurrent versions
- **Transitions**: Configurable storage class transitions

## Dynamic Resource Fetching

### Bucket Naming Strategy
- Use **Fn::Sub** to dynamically generate bucket names with account ID
- Example: `cloudwatch-exports-${AWS::AccountId}`

### Policy ARNs
- Use **AWS managed policies** where applicable
- Example: `arn:aws:iam::aws:policy/AmazonS3FullAccess` (for cross-account access)

## Security Best Practices Implementation

### IAM Policies
- **Least Privilege**: Grant only necessary permissions
- **Cross-Account Access**: Configurable policy for external accounts
- **Service Access**: Allow CloudWatch to write to bucket

### Encryption
- **Default**: SSE-S3 for all objects
- **Optional**: SSE-KMS for enhanced security (configurable)

### Access Logging
- **Target Bucket**: Separate logging bucket or same bucket with prefix
- **Prefix**: Configurable logging prefix

## Cost Optimization Features

### Storage Class Analysis
- **DataExport**: Enabled to analyze access patterns
- **OutputSchemaVersion**: V_1 for compatibility

### Lifecycle Transitions
- **Infrequent Access**: Configurable transition to IA storage class
- **Glacier**: Optional transition to Glacier for archival

## Error Handling & Validation

### Parameter Validation
- **AllowedValues**: Restrict versioning options
- **MinValue/MaxValue**: Prevent invalid expiration configurations

### Resource Validation
- **DeletionPolicy**: Set to Retain for data protection
- **UpdateReplacePolicy**: Aligned with DeletionPolicy

## Deployment Strategy

### Sequential Resource Creation
1. Create S3 bucket
2. Configure versioning
3. Apply bucket policy
4. Set up lifecycle configuration
5. Enable analytics and logging

### Error Prevention
- **Unique Names**: Ensure bucket names are globally unique
- **Region Validation**: Use `AWS::Region` pseudo parameter
- **Account Validation**: Use `AWS::AccountId` pseudo parameter

This architecture ensures a secure, cost-optimized, and production-ready S3 bucket for CloudWatch export data with all necessary configurations and best practices implemented.
