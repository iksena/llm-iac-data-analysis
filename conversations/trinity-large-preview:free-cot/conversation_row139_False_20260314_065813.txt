# Detailed Architectural Deployment Plan for Custom Lambda Arithmetic Resource

## Key AWS Services Required
1. **AWS Lambda** - To host the custom resource handler function
2. **AWS IAM** - For Lambda execution role and permissions
3. **AWS CloudFormation** - To deploy the template and manage the custom resource
4. **AWS Systems Manager (SSM) Parameter Store** - For dynamic AMI fetching (if needed for Lambda runtime)
5. **AWS CloudWatch Logs** - For Lambda logging

## Architecture Overview

The solution will consist of:
1. A Lambda function that performs arithmetic operations based on input parameters
2. A custom resource definition that invokes the Lambda function
3. Parameters for user input (numbers and operator)
4. A conditional failure mechanism for testing
5. Outputs to display the calculation result

## Detailed Implementation Plan

### 1. Lambda Function Design

**Runtime Selection:**
- Use Python 3.12 (latest supported runtime)
- This ensures compatibility and avoids deprecated runtime issues

**Function Logic:**
- Parse `FirstNumber`, `SecondNumber`, and `Operator` from event
- Perform the requested arithmetic operation
- Support operators: `+`, `-`, `*`, `/`
- Handle division by zero gracefully
- Implement conditional failure based on a `ForceFailure` parameter

**Error Handling:**
- Return structured error responses for CloudFormation
- Use proper HTTP status codes (200 for success, 400 for client errors)
- Include detailed error messages in the response body

### 2. IAM Role Configuration

**Permissions Required:**
- Basic Lambda execution permissions
- CloudWatch Logs write access for logging
- No additional permissions needed for this read-only operation

**Security Best Practices:**
- Use least privilege principle
- Create a dedicated IAM role for this Lambda function
- Avoid wildcard permissions

### 3. CloudFormation Template Structure

**Parameters Section:**
```yaml
Parameters:
  FirstNumber:
    Type: Number
    Default: 10
    Description: First number for arithmetic operation
  
  SecondNumber:
    Type: Number
    Default: 5
    Description: Second number for arithmetic operation
  
  Operator:
    Type: String
    Default: +
    AllowedValues:
      - +
      - -
      - *
      - /
    Description: Arithmetic operator to use
  
  ForceFailure:
    Type: String
    Default: false
    AllowedValues:
      - true
      - false
    Description: Force the custom resource to fail for testing
```

**Resources Section:**
1. IAM Role for Lambda execution
2. Lambda function with Python 3.12 runtime
3. Custom resource that invokes the Lambda
4. Optional: Lambda permissions if needed

**Outputs Section:**
- Display the calculation result
- Show operation details for verification

### 4. Custom Resource Implementation

**Request Schema:**
The Lambda function will expect an event with:
- `RequestType` (Create, Update, Delete)
- `ResourceProperties` containing:
  - `FirstNumber`
  - `SecondNumber`
  - `Operator`
  - `ForceFailure`

**Response Format:**
Follow CloudFormation custom resource response format:
```json
{
  "Status": "SUCCESS" | "FAILED",
  "Reason": "Error message if failed",
  "PhysicalResourceId": "unique-id",
  "StackId": "arn:aws:cloudformation:...",
  "RequestId": "request-id",
  "LogicalResourceId": "logical-resource-id",
  "Data": {
    "Result": "calculation-result",
    "Operation": "FirstNumber Operator SecondNumber = Result"
  }
}
```

### 5. Conditional Failure Mechanism

**Implementation Strategy:**
- Check the `ForceFailure` parameter value
- If `true`, return a failed response with a specific error message
- This allows testing of CloudFormation rollback behavior

**Failure Scenarios:**
- Division by zero (even if not forced)
- Invalid operator (though validation should prevent this)
- Forced failure for testing

### 6. Dynamic Parameter Processing

**Input Validation:**
- Validate that numbers are within reasonable ranges
- Ensure operator is one of the allowed values
- Handle edge cases like very large numbers or special float values

**Operation Logic:**
```python
operations = {
    '+': lambda a, b: a + b,
    '-': lambda a, b: a - b,
    '*': lambda a, b: a * b,
    '/': lambda a, b: a / b if b != 0 else None
}
```

### 7. CloudFormation Outputs

**Result Display:**
- Show the final calculation result
- Include the full operation for clarity
- Handle failure cases gracefully in outputs

**Example Outputs:**
```yaml
Outputs:
  CalculationResult:
    Description: Result of the arithmetic operation
    Value: !GetAtt CustomResource.Data.Result
    Export: true
  
  OperationDetails:
    Description: Details of the operation performed
    Value: !GetAtt CustomResource.Data.Operation
```

### 8. Best Practices Implementation

**Security:**
- No hardcoded credentials or secrets
- Minimal IAM permissions
- Secure Lambda execution environment

**Reliability:**
- Proper error handling and logging
- Graceful degradation for edge cases
- Idempotent operations where possible

**Maintainability:**
- Clear code structure and comments
- Consistent naming conventions
- Modular design for easy updates

### 9. Testing Strategy

**Unit Tests:**
- Test all arithmetic operations
- Verify error handling
- Test conditional failure mechanism

**Integration Tests:**
- Deploy template in test environment
- Verify custom resource behavior
- Test rollback scenarios

**Edge Cases:**
- Very large numbers
- Division by zero
- Invalid inputs (should be prevented by validation)

### 10. Deployment Considerations

**Region Selection:**
- Use the region where the template is deployed
- No hardcoded region dependencies

**Dependencies:**
- Lambda function must be created before custom resource
- IAM role must be created before Lambda

**Cleanup:**
- Ensure proper resource cleanup on stack deletion
- Handle physical resource ID appropriately

This architectural plan ensures a robust, secure, and maintainable CloudFormation template that demonstrates custom resource capabilities while following AWS best practices and avoiding common pitfalls that cause deployment failures.
