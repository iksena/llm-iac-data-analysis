# S3 Bucket for CloudWatch Export Data - Architecture Plan

## 1. Key AWS Services Required

- **AWS::S3::Bucket** - Primary storage for CloudWatch export data
- **AWS::S3::BucketPolicy** - Fine-grained access control
- **AWS::S3::BucketVersioning** - Versioning configuration
- **AWS::S3::BucketLifecycleConfiguration** - Automated lifecycle management
- **AWS::IAM::Role** - Service role for CloudWatch exports (if needed)
- **AWS::IAM::Policy** - Permissions for CloudWatch to write to S3

## 2. Properties and Dependencies

### Core Dependencies:
- S3 Bucket must be created before BucketPolicy, Versioning, and LifecycleConfiguration
- IAM Role (if used) must exist before CloudWatch can assume it

### Critical Properties:
- **Bucket Name**: Must be globally unique
- **Versioning**: Configurable enable/disable with MFA delete option
- **Lifecycle Rules**: Configurable rules for transition and expiration
- **Encryption**: Server-side encryption with SSE-S3 or SSE-KMS
- **Access Control**: Bucket policy with least privilege principle
- **Logging**: Optional server access logging configuration

## 3. Safe AWS Defaults (for missing specifications)

### Parameters with Defaults:
```yaml
Parameters:
  BucketName:
    Type: String
    Default: cloudwatch-exports-${AWS::AccountId}
    Description: S3 bucket name for CloudWatch exports

  EnableVersioning:
    Type: String
    Default: true
    AllowedValues: [true, false]
    Description: Enable S3 bucket versioning

  EnableMFADelete:
    Type: String
    Default: false
    AllowedValues: [true, false]
    Description: Enable MFA delete for versioning

  TransitionToIA:
    Type: Number
    Default: 30
    Description: Days to transition to Infrequent Access storage class

  TransitionToGlacier:
    Type: Number
    Default: 60
    Description: Days to transition to Glacier storage class

  ExpirationDays:
    Type: Number
    Default: 365
    Description: Days to expire old objects
```

### Safe Defaults for Properties:
- **Encryption**: SSE-S3 (default AWS-managed encryption)
- **Region**: `AWS::Region` pseudo parameter
- **Account**: `AWS::AccountId` pseudo parameter
- **ACL**: Private (no public access)
- **Storage Class**: STANDARD (for frequently accessed data)

## 4. Dynamic Resource Fetching Strategy

### AMI Fetching (Not Applicable Here):
Since this template doesn't provision EC2 instances, AMI fetching via SSM Parameter Store isn't needed. However, if we were to add Lambda functions for automation, we would use:

```yaml
!Sub 'arn:${AWS::Partition}:ssm:${AWS::Region}:${AWS::AccountId}:parameter/aws/service/lambda/runtime'
```

### Region-Aware Configuration:
- Use `AWS::Region` for region-specific configurations
- Use `AWS::Partition` for cross-partition compatibility (aws, aws-cn, aws-us-gov)

## 5. Security and Access Control Design

### Bucket Policy Structure:
```yaml
BucketPolicy:
  PolicyDocument:
    Statement:
      - Effect: Allow
        Principal:
          Service: logs.amazonaws.com
        Action:
          - s3:PutObject
          - s3:GetBucketAcl
        Resource:
          - !Sub 'arn:${AWS::Partition}:s3:::${BucketName}'
          - !Sub 'arn:${AWS::Partition}:s3:::${BucketName}/*'
```

### Encryption Configuration:
- **Default**: SSE-S3 (server-side encryption with Amazon S3-managed keys)
- **Optional**: SSE-KMS with customer-managed CMK for enhanced security

### Access Logging (Optional):
- Enable server access logging to separate audit bucket
- Configure log delivery prefix for organization

## 6. Lifecycle Management Strategy

### Default Lifecycle Rules:
1. **30 days**: Transition to S3 Infrequent Access (cost optimization)
2. **60 days**: Transition to S3 Glacier (archival)
3. **365 days**: Permanently delete expired objects (compliance)

### Versioning Integration:
- Non-current versions follow same lifecycle rules
- Option to retain only most recent non-current version for cost control

## 7. Cost Optimization Measures

### Storage Class Analysis:
- **STANDARD**: For frequently accessed CloudWatch export data (first 30 days)
- **STANDARD_IA**: For infrequently accessed data (30-60 days)
- **GLACIER**: For archival data (60+ days)

### Versioning Cost Control:
- Configure lifecycle rules to clean up non-current versions
- Set appropriate expiration for old versions

## 8. Monitoring and Compliance

### CloudWatch Metrics:
- S3 bucket metrics for storage usage and request counts
- Billing alerts for cost monitoring

### Compliance Features:
- Server-side encryption for data at rest
- Versioning for data protection and recovery
- Access logging for audit trails
- MFA delete option for enhanced security

