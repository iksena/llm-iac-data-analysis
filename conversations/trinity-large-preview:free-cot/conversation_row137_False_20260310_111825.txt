# Detailed Architectural Deployment Plan for AppSync GraphQL API with Lambda Backend

## 1. Key AWS Services Required

Based on the business need, the following AWS services are required:

1. **AWS AppSync** - For creating the GraphQL API
2. **AWS Lambda** - For the serverless backend function
3. **AWS IAM** - For creating execution roles with least privilege
4. **AWS CloudFormation** - For infrastructure as code deployment

## 2. Properties and Dependencies

The deployment follows this logical sequence:

1. **IAM Roles** must be created first (Lambda execution role and AppSync data source role)
2. **Lambda Function** depends on the execution role
3. **AppSync API** can be created independently
4. **AppSync Data Source** depends on both the API and Lambda function
5. **AppSync Schema and Resolvers** depend on the API and data source

## 3. Safe AWS Defaults

Since specific properties weren't provided, I'll use these safe defaults:

- **Lambda Runtime**: `python3.12` (latest supported)
- **Lambda Memory**: `128 MB` (minimum)
- **Lambda Timeout**: `10 seconds` (sufficient for mock data)
- **AppSync Authentication**: API_KEY (as specified)
- **API Key Expiration**: `365 days` from creation
- **Region**: Use `AWS::Region` pseudo-parameter for dynamic region detection

## 4. Dynamic Resource Fetching Strategy

For this deployment, dynamic fetching isn't strictly necessary since we're not using EC2 instances or specific AMIs. However, I'll use CloudFormation pseudo-parameters and functions where appropriate:

- **Region**: `AWS::Region` pseudo-parameter
- **Account ID**: `AWS::AccountId` pseudo-parameter
- **Lambda Function Name**: Use `AWS::StackName` for uniqueness

## 5. Architecture Design

### Lambda Function Design
The Lambda function will return a static dataset of mock pet data:
```python
[
    {"id": "1", "name": "Max", "type": "Dog"},
    {"id": "2", "name": "Bella", "type": "Cat"},
    {"id": "3", "name": "Charlie", "type": "Bird"}
]
```

### GraphQL Schema Design
The schema will support a single query:
```graphql
type Pet {
    id: ID!
    name: String!
    type: String!
}

type Query {
    getPets: [Pet]
}
```

### IAM Role Design (Least Privilege)
- **Lambda Execution Role**: Only allows Lambda execution and CloudWatch logging
- **AppSync Data Source Role**: Only allows AppSync to invoke the specific Lambda function

### Request/Response Mapping Templates
The resolver will use VTL templates to transform the Lambda output into the GraphQL schema format.

## 6. Security Considerations

- API Key authentication for anonymous access (as specified)
- Least privilege IAM roles
- No hardcoded secrets or sensitive data
- Lambda function uses latest runtime with appropriate timeout

## 7. Deployment Validation

The template will be designed to pass cfn-lint with these considerations:
- All required properties included
- No hardcoded values that could cause failures
- Proper dependency chaining
- Valid IAM policies
- Correct AppSync configuration

This architecture provides a minimal, secure, and functional serverless pet inventory query service that meets all specified requirements while following AWS best practices.
