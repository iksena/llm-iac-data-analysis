- `ecr:CompleteLayerUpload`
- `ecr:DescribeRepositories`
- `ecr:GetRepositoryPolicy`
- `ecr:ListImages`
- `ecr:DeleteRepository`
- `ecr:BatchDeleteImage`
- `ecr:SetRepositoryPolicy`
- `ecr:PutLifecyclePolicy`
- `ecr:StartLifecyclePolicyPreview`

The CloudFormation template will include:

   - ECR Repository with image scanning enabled

## Compliance with AWS Best Practices

- No hardcoded AMIs or AZs (not applicable here)

AWSTemplateFormatVersion: '2010-09-09'
Description: CloudFormation template to create an Amazon ECR repository with repository policy for Kubernetes container image management

Parameters:
  AccountId:
    Type: String
    Default: '123456789012'
    Description: AWS Account ID for root user permissions
    AllowedPattern: '\d{12}'

Resources:
  KubernetesECRRepository:
    Type: AWS::ECR::Repository
    Properties:
      RepositoryName: kubernetes-repo
      ImageScanningConfiguration:
        ScanOnPush: true
      Tags:
        - Key: Environment
          Value: Production
        - Key: ManagedBy
          Value: CloudFormation

  ECRRepositoryPolicy:
    Type: AWS::ECR::RepositoryPolicy
    Properties:
      RepositoryName: !Ref KubernetesECRRepository
      PolicyText:
        Version: '2012-10-17'
        Statement:
          - Sid: RootUserFullAccess
            Effect: Allow
            Principal:
              AWS: !Sub 'arn:aws:iam::${AccountId}:root'
            Action:
  ...
