# Architectural Deployment Plan for Multi-IP Elastic Network Interface

## Key AWS Services Required
1. **VPC** - Virtual Private Cloud for network isolation
2. **EC2** - Elastic Compute Cloud (for the instance that will use the ENI)
3. **Elastic Network Interface (ENI)** - Network interface with multiple IPs
4. **Elastic IP (EIP)** - Two public IP addresses
5. **SSM Parameter Store** - For dynamic AMI fetching (best practice)

## Properties and Dependencies Analysis

### VPC Dependencies
- VPC must exist before creating ENI
- ENI requires a subnet within the VPC
- EIPs require an Internet Gateway attached to the VPC

### ENI Configuration
- Primary private IP (auto-assigned)
- Two secondary private IPs (for the two EIPs)
- Private IPs must be within the subnet's CIDR range

### EIP Dependencies
- EIPs require an Internet Gateway attached to the VPC
- EIPs must be associated with the ENI's private IPs

## Safe AWS Defaults (since specific values weren't provided)

### VPC Configuration
- **CIDR Block**: `10.0.0.0/16` (standard private range)
- **Subnet CIDR**: `10.0.0.0/24` (first subnet in VPC)
- **Availability Zone**: Dynamically selected using `Fn::GetAZs` and `Fn::Select`

### ENI Configuration
- **Primary Private IP**: Auto-assigned by AWS
- **Secondary Private IPs**: Two IPs from the subnet range (e.g., 10.0.0.10 and 10.0.0.11)
- **Security Groups**: Default VPC security group (can be modified later)

### EIP Configuration
- **Domain**: `vpc` (required for VPC-based EIPs)

   - EC2 Instance Type (default: 't3.micro')

2. **Resources**

### ENI Configuration Details

- **Secondary Private IPs**: Two IPs from subnet range
- **Security Groups**: Default VPC security group
- **Source/Dest Check**: `false` (if instance will route traffic)

### EIP Association Strategy
- EIP 1 → Secondary Private IP 1
- EIP 2 → Secondary Private IP 2

- No hardcoded AMIs or AZs
- All Parameters have safe defaults
- Resources have appropriate dependencies
- No legacy properties used
- Template will pass cfn-lint validation

This architecture enables a single EC2 instance to handle multiple public-facing IP addresses through one network interface, providing flexibility for scenarios like:
- Multiple SSL certificates on a single instance
- Network isolation between services
- Simplified network management
- High availability configurations

The template will be designed for easy modification and extension, allowing customers to add their specific requirements while maintaining the core multi-IP functionality.
