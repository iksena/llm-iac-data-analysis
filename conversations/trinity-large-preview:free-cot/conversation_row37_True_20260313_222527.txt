# Secure S3 Bucket Architecture Plan

## Key AWS Services Required
1. **AWS::S3::Bucket** - Primary secure bucket
2. **AWS::S3::Bucket** - Replica bucket
3. **AWS::S3::Bucket** - Access log bucket
4. **AWS::S3::BucketPolicy** - Security policy for primary bucket
5. **AWS::S3::BucketEncryption** - Server-side encryption configuration
6. **AWS::S3::BucketPublicAccessBlock** - Public access blocking
7. **AWS::S3::BucketAnalyticsConfiguration** - Analytics for monitoring

## Properties and Dependencies

### S3 Bucket Properties (All Buckets)
- **BucketName**: Must be unique globally
- **VersioningConfiguration**: Enabled for data protection
- **AccessControl**: Set to `Private` (with OwnershipControls)
- **OwnershipControls**: Required when using AccessControl
- **PublicAccessBlockConfiguration**: Block all public access
- **BucketEncryption**: SSE-S3 encryption
- **LifecycleConfiguration**: For replica and log bucket management

### Security Dependencies
- **BucketPolicy**: Requires secure transport (HTTPS only)
- **Server-side encryption**: SSE-S3 with AWS-managed keys
- **Access logging**: Enabled on primary bucket pointing to log bucket

## Safe AWS Defaults (Where Values Not Specified)

### S3 Bucket Names (Using Safe Defaults)
```yaml
PrimaryBucketName: !Sub "secure-bucket-${AWS::AccountId}"
ReplicaBucketName: !Sub "replica-bucket-${AWS::AccountId}"
LogBucketName: !Sub "access-logs-${AWS::AccountId}"
```

### Encryption
- **SSE-S3** (Server-Side Encryption with S3-managed keys) - Default AWS choice
- No KMS key required unless specific compliance needs dictate otherwise

### Public Access Block Configuration
```yaml
BlockPublicAcls: true
IgnorePublicAcls: true
BlockPublicPolicy: true
RestrictPublicBuckets: true
```

### Bucket Policy for Secure Transport
```yaml
{
  "Version": "2012-10-17",
  "Id": "SecureTransportOnlyPolicy",
  "Statement": [
    {
      "Sid": "RequireSecureTransport",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": [
        "arn:aws:s3:::secure-bucket-ACCOUNT_ID/*",
        "arn:aws:s3:::secure-bucket-ACCOUNT_ID"
      ],
      "Condition": {
        "Bool": {
          "aws:SecureTransport": "false"
        }
      }
    }
  ]
}
```

## Architecture Implementation Plan

### 1. Primary Secure Bucket
- Enable versioning for data protection
- Configure SSE-S3 encryption
- Block all public access
- Enable server access logging to log bucket
- Apply bucket policy requiring HTTPS

### 2. Replica Bucket
- Enable versioning
- Configure SSE-S3 encryption
- Block all public access
- Implement lifecycle policy to transition to Glacier after 30 days
- Set up cross-region replication from primary bucket

### 3. Access Log Bucket
- Enable versioning
- Configure SSE-S3 encryption
- Block all public access
- Implement lifecycle policy to expire logs after 365 days
- Enable default encryption

### 4. Security Configuration
- **Server-side encryption**: SSE-S3 for all buckets
- **Public access**: Completely blocked on all buckets
- **Transport security**: Bucket policy enforces HTTPS only
- **Access logging**: Enabled on primary bucket to log bucket

### 5. Dependencies and Order of Creation
1. Create log bucket first (required for access logging)
2. Create replica bucket second
3. Create primary bucket last (depends on log bucket for access logging)

### 6. Compliance and Security Scanning
- All buckets will pass AWS security scanning by default
- Encryption at rest (SSE-S3) enabled
- No public access possible
- Secure transport enforced
- Versioning enabled for data protection
- Access logging enabled for audit trail

This architecture ensures a secure, compliant S3 environment that meets all specified requirements while following AWS best practices and avoiding common CloudFormation pitfalls.
