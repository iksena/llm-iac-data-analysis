- EC2 instances running Amazon Linux 2
- User Data script installing Node.js, CodeDeploy agent, and Etherpad
- CloudFormation WaitCondition for instance readiness signaling

### Application Layer:
- Application Load Balancer with health checks
- Target Group pointing to EC2 instances
- CodeDeploy integration for automated deployments

### Data Layer:
- RDS MySQL instance in Multi-AZ deployment
- DB Subnet Group across both subnets
- Security Group allowing only application traffic

### Management Layer:
- SSM Parameter Store for configuration
- IAM roles for EC2 and CodeDeploy
- CloudFormation WaitConditions for orchestration

### Network Security:
- Security Groups restricting traffic to necessary ports
- No direct SSH access from internet (require VPN/bastion)
- RDS in private subnets with VPC-only access

### Data Security:
- Database credentials stored in SSM Secure String
- Encrypted S3 bucket for artifacts
- IAM roles with least privilege principle

### Application Security:
- HTTPS termination at Load Balancer (future enhancement)
- Input validation in Etherpad configuration
- Regular security updates via EC2 instance maintenance

6. **CodeDeploy Setup**: Application and deployment group created
7. **Initial Deployment**: First CodeDeploy deployment triggered

### Database:
- Multi-AZ RDS for automatic failover
- Automated backups enabled
- Point-in-time recovery capability

### Application:
- Load Balancer health checks prevent routing to failed instances
- CodeDeploy rollback capabilities
- S3 versioning for artifact recovery
