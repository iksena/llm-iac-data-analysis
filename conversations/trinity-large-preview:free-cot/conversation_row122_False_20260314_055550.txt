# Detailed Architectural Deployment Plan for Shopify EventBridge Integration

4. **AWS::SSM::Parameter** - To store Shopify API key securely (optional, depending on implementation)
5. **AWS::SecretsManager::Secret** - Alternative secure storage for Shopify API key (optional)

## Architecture Overview

The solution will create a serverless integration where:
- Application events from `MyShopifyApp` are published to a custom EventBridge bus
- EventBridge triggers an API call to Shopify's product creation endpoint
- Authentication is handled using the Shopify admin API key
- The target shop is `testanu.myshopify.com`

## Detailed Implementation Plan

### 1. Event Bus Configuration
- Create a custom event bus named `MyShopifyAppBus`
- Configure event pattern to match events from source `MyShopifyApp`
- Use default event bus settings (no specific retention period specified)

### 2. API Destination Setup
- Target URL: `https://testanu.myshopify.com/admin/api/2024-01/products.json`
- HTTP method: POST
- Connection type: HTTPS
- Connection ARN: Will be created dynamically

### 3. IAM Role Configuration
- Create an IAM role for EventBridge to assume
- Attach necessary permissions for EventBridge to invoke API destinations
- No additional policies needed beyond EventBridge permissions

### 4. Security Considerations
- Shopify API key will be passed as a parameter with secure handling
- Use dynamic references for secrets if implementing Secrets Manager
- Ensure HTTPS connection to Shopify endpoint


## Error Handling & Validation

- Verify API destination connection can be established
- Test IAM role permissions
- Validate event bus creation

## Deployment Considerations

### Region Selection
- Use `AWS::Region` pseudo parameter for region-specific resources
- Ensure Shopify API endpoint is accessible from target region

### Naming Conventions
- Use consistent naming with stack name prefix
- Include environment identifier if deploying across multiple environments

1. **Least Privilege**: IAM role has only necessary permissions

4. **No Hardcoded Secrets**: API key passed as parameter
