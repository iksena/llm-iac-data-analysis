# EventBridge API Destination Deployment Plan

## Key AWS Services Required

1. **Amazon EventBridge** - Core service for event routing and management
2. **EventBridge Event Bus** - Custom event bus for routing events
3. **EventBridge API Destination** - Endpoint configuration for external API
4. **EventBridge Connection** - Secure connection with API key authentication
5. **AWS Secrets Manager** (optional) - For secure API key storage

## Properties and Dependencies

1. **EventBridge Connection** must be created before the **API Destination**
2. **API Destination** requires a valid **Connection** reference
3. **EventBridge Event Bus** can be created independently but must exist before creating rules that target the API Destination

## Architecture Design

### 1. EventBridge Connection
- **Type**: API_KEY authentication
- **Properties**:
  - Name: `WebhookConnection`
  - Description: Connection to external webhook endpoint
  - AuthorizationType: `API_KEY`
  - ApiKeyAuthParameters:
    - ApiKeyName: `x-api-key` (or similar header)
    - ApiKeyValue: Dynamic reference to Secrets Manager or Parameter Store

### 2. EventBridge API Destination
- **Properties**:
  - Name: `WebhookAPI`
  - Description: External webhook endpoint
  - ConnectionArn: Reference to created Connection
  - InvocationEndpoint: `https://webhook.site/testtest`
  - HttpMethod: `POST` (default for API destinations)

### 3. EventBridge Event Bus
- **Properties**:
  - Name: `CustomEventBus` (or use default `default` bus)
  - Description: Custom event bus for routing events

### 4. Event Pattern Rule (Optional)
- Create a rule to route specific events to the API Destination
- Pattern: Custom event structure matching your needs

## Dynamic Resource Fetching

### API Key Management
- Use **AWS Secrets Manager** to store the API key securely
- Reference using dynamic reference: `{{resolve:secretsmanager:MySecret:SecretString:}}`
- Alternative: Use **AWS Systems Manager Parameter Store** with secure string parameter

### AMI Considerations
- Not applicable for this EventBridge configuration
- Would use SSM Parameter Store for Lambda functions if needed: `resolve:ssm:/aws/service/lambda/amazon-linux-2-arm64`

## Safe Defaults

### Event Bus
- **Name**: `CustomEventBus` (default: `default`)
- **Description**: `Custom event bus for routing events`

### Connection
- **Name**: `WebhookConnection`
- **Description**: `Connection to external webhook endpoint`
- **AuthorizationType**: `API_KEY`
- **ApiKeyAuthParameters**:
  - **ApiKeyName**: `x-api-key`
  - **ApiKeyValue**: Dynamic reference (no default - must be provided)

### API Destination
- **Name**: `WebhookAPI`
- **Description**: `External webhook endpoint`
- **HttpMethod**: `POST`
- **InvocationEndpoint**: `https://webhook.site/testtest`

## Security Best Practices

1. **API Key Storage**: Use Secrets Manager or Parameter Store with secure string
2. **Connection Security**: API_KEY authentication with proper header naming
3. **Resource Naming**: Use descriptive names with prefixes for easy identification
4. **IAM Permissions**: Ensure EventBridge has permissions to create connections and destinations

## Template Structure

```yaml
Resources:
  EventBus:
    Type: AWS::EventBridge::EventBus
    Properties:
      Name: CustomEventBus
      Description: Custom event bus for routing events

  Connection:
    Type: AWS::EventBridge::Connection
    Properties:
      Name: WebhookConnection
      Description: Connection to external webhook endpoint
      AuthorizationType: API_KEY
      ApiKeyAuthParameters:
        ApiKeyName: x-api-key
        ApiKeyValue: !Sub '{{resolve:secretsmanager:${MySecret}:SecretString:}}'

  APIDestination:
    Type: AWS::EventBridge::ApiDestination
    Properties:
      Name: WebhookAPI
      Description: External webhook endpoint
      ConnectionArn: !GetAtt Connection.Arn
      InvocationEndpoint: https://webhook.site/testtest
      HttpMethod: POST

  # Optional: Event Pattern Rule
  EventRule:
    Type: AWS::EventBridge::Rule
    Properties:
      EventBusName: !Ref EventBus
      EventPattern: !Sub |
        {
          "source": ["myapp.events"],
          "detail-type": ["Custom Event"]
        }
      Targets:
        - Arn: !GetAtt APIDestination.Arn
          Id: WebhookTarget
```

## Key Considerations

1. **API Key Management**: The template requires a Secrets Manager secret or Parameter Store value to be created separately
2. **Endpoint Validation**: The webhook.site endpoint should be tested for accessibility
3. **Event Patterns**: Customize the event pattern to match your specific event structure
4. **Error Handling**: Consider adding dead-letter queues or error targets for failed invocations
5. **Monitoring**: Enable EventBridge logging and metrics for visibility

This architecture provides a secure, scalable way to route custom events from EventBridge to external API endpoints using API key authentication.
