# Serverless Calculator Workflow Architecture Plan

## Key AWS Services Required

1. **AWS Step Functions** - State machine orchestration
2. **AWS Lambda** - Serverless compute for arithmetic operations
3. **Amazon CloudWatch Logs** - Log retention and monitoring
4. **AWS Identity and Access Management (IAM)** - Service permissions and execution roles

## Architecture Overview

The solution will implement a serverless calculator workflow using AWS Step Functions to orchestrate arithmetic operations executed by a Lambda function. The state machine will route requests based on the operator type and handle errors for unsupported operations.

## Detailed Component Analysis

### 1. Lambda Function (`Calculator`)

**Properties:**
- Runtime: `python3.12` (latest supported runtime)
- Handler: `calculator.lambda_handler`
- Memory: `128 MB` (safe default for lightweight calculations)
- Timeout: `10 seconds` (sufficient for arithmetic operations)
- Environment variables: None required
- Code location: Inline code in CloudFormation template

**IAM Role Requirements:**
- `AWSLambdaBasicExecutionRole` - For CloudWatch Logs access
- `AWSStepFunctionsFullAccess` - For Step Functions integration

### 2. Step Functions State Machine (`Calculator-StateMachine`)

**Properties:**
- Definition: JSON-based state machine definition
- Role: IAM role with Step Functions execution permissions
- Type: `STANDARD` (default, suitable for this use case)

**State Machine Definition:**
- Choice state to route based on `operator` input
- Task states for each arithmetic operation
- Fail state for unsupported operators
- Pass state to return results

### 3. CloudWatch Logs Group

**Properties:**
- Log group name: `/aws/lambda/Calculator`
- Retention policy: `3 days` (as specified in business requirements)

## Dependencies and Execution Order

1. IAM Role creation (no dependencies)
2. Lambda function creation (depends on IAM role)
3. CloudWatch Logs group creation (no dependencies)
4. Step Functions state machine creation (depends on Lambda function)

## Dynamic Resource Handling

### AMI Handling
Not applicable for this serverless architecture.

### Availability Zones
Not applicable for this serverless architecture.

## Security Considerations

1. **Least Privilege Principle:**
   - Lambda execution role will have only necessary permissions
   - Step Functions role will have only required permissions

2. **No Hardcoded Secrets:**
   - No secrets required for this implementation
   - All configurations will use safe defaults

3. **Resource Protection:**
   - No stateful resources requiring deletion policies
   - Lambda function will use safe defaults for retention

## Error Handling Strategy

1. **Unsupported Operators:**
   - State machine will fail with descriptive error message
   - Error will be logged in CloudWatch Logs

2. **Division by Zero:**
   - Lambda function will handle and return error
   - State machine will capture and log the error

3. **Invalid Input:**
   - Lambda function will validate input
   - State machine will handle validation errors

## Monitoring and Logging

1. **CloudWatch Logs:**
   - Lambda function logs will be retained for 3 days
   - Step Functions execution logs will be available in console

2. **Metrics:**
   - Lambda function metrics (invocations, errors, duration)
   - Step Functions execution metrics

## Cost Optimization

1. **Serverless Architecture:**
   - Pay-per-execution model
   - No idle resource costs

2. **Resource Sizing:**
   - Lambda function sized appropriately for arithmetic operations
   - Step Functions state machine using STANDARD type

## Deployment Strategy

1. **Template Structure:**
   - Parameters section with safe defaults
   - Resources section with all AWS resources
   - Outputs section for key resource identifiers

2. **Validation:**
   - Template will pass cfn-lint validation
   - All required properties will be included
   - No hardcoded values or deprecated configurations

## Implementation Notes

1. **Lambda Code:**
   - Will be embedded as inline code in CloudFormation template
   - Will handle all four arithmetic operations
   - Will include input validation and error handling

2. **State Machine Definition:**
   - Will use JSON format for state machine definition
   - Will include choice state for operator routing
   - Will handle success and failure paths

3. **IAM Roles:**
   - Will use AWS managed policies where possible
   - Will include custom policies for specific permissions
   - Will follow least privilege principle

This architecture plan ensures a robust, secure, and cost-effective serverless calculator workflow that meets all specified requirements while adhering to AWS best practices and CloudFormation linting rules.
