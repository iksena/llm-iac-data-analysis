4. **AWS::IAM::Role** - IAM role for Lambda execution with necessary permissions
5. **AWS::IAM::Policy** - IAM policy granting S3 access to the Lambda role

## Properties and Dependencies

The deployment follows this logical sequence:
1. IAM Role creation (no dependencies)
2. IAM Policy attachment to the role
3. Lambda Function creation (depends on IAM Role)
4. S3 Bucket creation (no dependencies)
5. Lambda Permission creation (depends on Lambda Function and S3 Bucket)
6. S3 Bucket notification configuration (depends on Lambda Function and S3 Bucket)

    Type: String

## Architecture Plan Details

- Trust policy allowing Lambda service to assume the role
- Permissions for CloudWatch Logs (write execution logs)

### Lambda Function Configuration

- Timeout: 30 seconds (safe default)
- Memory: 128 MB (safe default)
- Runtime: python3.12 (latest supported)
- Environment variables for configuration

### S3 Bucket Configuration
- Versioning: Disabled (safe default)
- Public access: Blocked (secure default)
- Encryption: SSE-S3 (secure default)
- Notification configuration pointing to Lambda function

### Lambda Permission
- Explicit permission for S3 service to invoke the Lambda function
- Source account set to the bucket owner's account
- Source ARN set to the S3 bucket ARN

### S3 Bucket Policy
- Policy granting S3 service permission to invoke the Lambda function
- Principal set to the bucket owner's account
- Action set to lambda:InvokeFunction

- **SecurityGroup**: No specific security group (Lambda uses AWS-managed security)
- **Timeout**: 30 seconds (standard default)
- **Memory**: 128 MB (standard default)
- **Environment Variables**: None (unless business requirements specify)
- **Tags**: None (unless business requirements specify)

## Error Prevention Strategy

To ensure successful deployment:
1. All parameters have safe defaults
2. No hardcoded AMIs or AZs
3. Latest Lambda runtime is used
4. Proper IAM policies with least privilege
5. S3 bucket policy instead of AccessControl property
6. Both DeletionPolicy and UpdateReplacePolicy included
7. No unnecessary Fn::Sub usage
8. Exact AWS managed policy ARNs and resource type names

This architecture ensures a secure, maintainable, and production-ready deployment that follows AWS best practices and will pass cfn-lint validation.
