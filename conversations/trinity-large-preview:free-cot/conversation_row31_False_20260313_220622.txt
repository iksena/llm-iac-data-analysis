- **Application Load Balancer (ALB)** - Traffic distribution and internet-facing endpoint
- **Amazon RDS (MySQL)** - Database storage for Etherpad
- **AWS CodeDeploy** - Automated application deployment
- **Amazon S3** - Storage for deployment artifacts
- **AWS Systems Manager (SSM)** - Secure parameter storage and dynamic AMI fetching
- **AWS Identity and Access Management (IAM)** - Service permissions and roles
- **AWS Secrets Manager** - Secure storage of database credentials

## 2. Properties and Dependencies

### Critical Dependencies:
1. **VPC Creation** → **Internet Gateway Attachment** → **Route Table Configuration** → **Subnet Provisioning**
2. **IAM Roles** → **CodeDeploy Application/Deployment Group** → **Launch Template** → **Auto Scaling Group**
3. **RDS Instance** → **SSM Parameter Storage** → **Application Configuration**
4. **S3 Bucket** → **CodeDeploy Application** → **Deployment Configuration**

### Property Dependencies:
- ALB requires subnets with internet connectivity
- Auto Scaling Group requires Launch Template with proper IAM instance profile
- RDS requires VPC security group allowing application access
- CodeDeploy requires IAM roles with proper permissions
- Launch Template user data requires SSM parameters to be available

## 3. Safe AWS Defaults for Missing Properties

### Network Configuration:
- **VPC CIDR**: `10.0.0.0/16` (standard private range)
- **Public Subnet CIDRs**: `10.0.1.0/24` and `10.0.2.0/24`
- **Availability Zones**: Dynamically selected using `Fn::GetAZs` and `Fn::Select`
- **Instance Type**: `t3.medium` (balanced performance for Etherpad)

### Security Configuration:
- **SSH Key Pair**: Default to `default-key` (must exist in region or be created)
- **Database Master Password**: Use Secrets Manager with auto-generated password
- **Database Port**: `3306` (MySQL default)
- **Application Port**: `9001` (Etherpad default)

### Other Defaults:
- **EC2 Instance AMI**: Latest Amazon Linux 2 using SSM parameter `/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2`
- **RDS Instance Class**: `db.t3.medium`
- **S3 Bucket Name**: Auto-generated unique name with `etherpad-deployments-${AWS::AccountId}-${AWS::Region}`

## 4. Dynamic Resource Fetching Strategy

### AMI Fetching:
```yaml

```

### Availability Zones:
```yaml
!Select [0, !GetAZs ""]
!Select [1, !GetAZs ""]
```

### Database Credentials:
- Use Secrets Manager to generate and store database credentials
- Reference credentials using dynamic references in application configuration

## 5. Architecture Overview

The deployment will follow this sequence:

1. **Network Layer**: VPC with public subnets, internet gateway, and route tables
2. **Security Layer**: Security groups for ALB, EC2, and RDS with proper port configurations
3. **Database Layer**: RDS MySQL instance with automated backups
4. **Application Layer**: Auto Scaling Group with Launch Template containing user data
5. **Deployment Layer**: CodeDeploy application with S3 artifact storage
6. **Load Balancing Layer**: Application Load Balancer with health checks

## 6. Security Considerations

- All traffic between ALB and EC2 instances will use HTTPS/TLS
- RDS will be placed in private subnets with VPC security group restrictions
- IAM roles will follow principle of least privilege
- SSM parameters will store non-sensitive configuration
- Secrets Manager will handle database credentials

## 7. Failure Recovery Strategy

- Auto Scaling Group will replace failed instances automatically
- RDS automated backups with point-in-time recovery
- ALB health checks will remove unhealthy instances from rotation
- CloudFormation rollback on deployment failure

## 8. Cost Optimization

- Use burstable instance types (t3) for cost efficiency
- Enable RDS automated backups with appropriate retention period
- Configure Auto Scaling Group with minimum instances during off-peak
- Use S3 Standard storage class for deployment artifacts

This architecture provides a production-ready, automated deployment pipeline for the Etherpad application while adhering to AWS best practices and ensuring maintainability through CloudFormation.
