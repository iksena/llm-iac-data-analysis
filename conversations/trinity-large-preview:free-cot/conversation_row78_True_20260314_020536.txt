# AWS EC2 IPAM Deployment Architecture Plan

## Key AWS Services Required
1. **AWS::EC2::IPAM** - The core IP Address Manager resource
2. **AWS::EC2::IPAMScope** - Public and private scopes for IP allocation
3. **AWS::EC2::IPAMPool** - IP pools within each scope
4. **AWS::EC2::IPAMResourceDiscovery** - Optional resource discovery for existing VPCs
5. **AWS::EC2::IPAMResourceDiscoveryAssociation** - Associates discovered resources with IPAM

## Properties and Dependencies

### Core IPAM Configuration
- **IPAM Name**: Required identifier for the IPAM
- **Description**: Optional description for documentation
- **Operating Regions**: List of regions where IPAM will be active

### Scope Configuration
- **Public Scope**: For public IP address management
- **Private Scope**: For private IP address management
- **CIDR Blocks**: Define allocation ranges for each scope

### Pool Configuration
- **Pool Name**: Identifier for each IP pool
- **CIDR Blocks**: Specific CIDR ranges within each scope
- **Allocation Method**: Automatic or manual allocation

## Safe AWS Defaults (Due to Missing Specific Properties)

### IPAM Configuration Defaults
- **Default Name**: "DefaultIPAM"
- **Default Description**: "Managed IP Address Manager"
- **Operating Regions**: Single region based on stack deployment region

### Scope Defaults
- **Public Scope CIDR**: "0.0.0.0/0" (global public range)
- **Private Scope CIDR**: "10.0.0.0/8" (RFC 1918 private range)

### Pool Defaults
- **Pool Name**: "DefaultPool"
- **Allocation Method**: "automatic"

## Dynamic Resource Fetching Strategy

### AMI Fetching (Not Applicable for IPAM)
IPAM doesn't require AMIs, but if we were to include EC2 instances for management purposes:
- Use SSM Parameter Store: `/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2`
- Fetch latest AMI ID dynamically using `Fn::ImportValue` or `Fn::Sub` with SSM resolve syntax

### Availability Zone Handling
- Use `Fn::GetAZs` to fetch available AZs in the deployment region
- Use `Fn::Select` to choose specific AZs if needed
- Avoid hardcoding AZ identifiers like 'us-east-1a'

## Cross-Stack Reference Strategy

### Critical Exports
The template will export the following identifiers as stack outputs:
1. **IPAM ARN**: `Fn::GetAtt: [IPAM, Arn]`
2. **IPAM ID**: `Ref: IPAM`
3. **Public Scope ID**: `Ref: PublicScope`
4. **Private Scope ID**: `Ref: PrivateScope`

### Export Naming Convention
- Use descriptive export names with stack name prefix
- Format: `${AWS::StackName}-IPAM-ARN`, `${AWS::StackName}-IPAM-ID`, etc.

## Security and Access Control

### IAM Permissions
- IPAM requires specific IAM permissions for cross-account resource discovery
- Define least-privilege IAM roles if cross-account access is needed

### Resource Protection
- Set appropriate `DeletionPolicy` and `UpdateReplacePolicy` for stateful resources
- Consider `Retain` policy for IPAM to prevent accidental deletion

## Template Structure Plan

### Parameters Section
- **IPAMName**: String with default "DefaultIPAM"
- **Description**: String with default "Managed IP Address Manager"
- **OperatingRegions**: Comma-separated list with safe default (single region)

### Resources Section
1. **IPAM Resource**: Core IPAM creation
2. **PublicScope Resource**: Public IP scope creation
3. **PrivateScope Resource**: Private IP scope creation
4. **PublicPool Resource**: Public IP pool within public scope
5. **PrivatePool Resource**: Private IP pool within private scope

### Outputs Section
- Export all critical identifiers for cross-stack references
- Include descriptive output values for easy identification

## Error Prevention and Best Practices

### Linting Compliance
- All Parameters have safe default values
- No hardcoded AMIs or AZs
- Proper use of intrinsic functions
- Correct property names matching CloudFormation schema

### Validation Strategy
- Validate CIDR block formats using AllowedPattern
- Ensure scope IDs are unique within the IPAM
- Verify pool CIDR blocks don't overlap with existing allocations

This architecture provides a robust, production-ready IPAM deployment that follows AWS best practices and ensures successful CloudFormation deployment with proper cross-stack referencing capabilities.
