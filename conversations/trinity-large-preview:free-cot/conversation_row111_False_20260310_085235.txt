- **Dependencies**: IAM role for EventBridge permissions

- **Trust Policy**: EventBridge service
- **Dependencies**: None (created independently)

### 4. Dead-Letter SQS Queue

- **Redrive Policy**: Retain failed messages for analysis
- **Dependencies**: None (standard SQS queue)

### 5. SSM Parameter for API Key
- **Name**: `/DataDog/LogForwarding/ApiKey`
- **Type**: SecureString
- **Dependencies**: None (SSM parameter)

- **HttpMethod**: `POST`

- **Body**: JSON payload with log events

### IAM Role Properties

- Use JSON schema to capture relevant log fields

## Security Considerations

1. **API Key Protection**: Store API key in SSM SecureString, never in plaintext
2. **Least Privilege**: IAM role only has permissions to invoke the specific API destination
3. **Network Security**: EventBridge API Destinations use HTTPS endpoints
4. **Audit Trail**: All API invocations logged in CloudTrail

## Error Handling & Fault Tolerance

1. **Dead-Letter Queue**: Failed log deliveries automatically routed to SQS DLQ
2. **Retry Logic**: EventBridge automatically retries failed API invocations
3. **Monitoring**: CloudWatch metrics and alarms for API destination health
4. **Alerting**: SNS notifications for DLQ message accumulation

## Deployment Sequence

1. Create IAM role for EventBridge API invocation
2. Create SSM parameter for DataDog API key
3. Create EventBridge event bus
4. Create API destination with secure connection
5. Create dead-letter SQS queue
6. Create event rule to route events to API destination

## Validation & Testing

1. **Template Validation**: Use cfn-lint to validate CloudFormation syntax
2. **Security Scan**: Verify no hardcoded secrets in template
3. **Connectivity Test**: Test API destination connectivity to DataDog
4. **End-to-End Test**: Publish test events and verify log delivery to DataDog
