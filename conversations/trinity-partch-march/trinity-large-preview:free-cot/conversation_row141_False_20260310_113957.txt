# Image Recognition Pipeline Architecture Plan

## Key AWS Services Required

1. **Amazon S3** - Storage bucket for image uploads
2. **Amazon EventBridge** - Event-driven notifications for S3 uploads
3. **AWS Step Functions** - Orchestration workflow for image processing
4. **Amazon Rekognition** - Image analysis for face detection
5. **Amazon SNS** - SMS notification service
6. **AWS Lambda** - Custom resource for S3 bucket cleanup during deletion
7. **AWS IAM** - Identity and access management for all services

## Architecture Overview

The solution follows an event-driven architecture where:
1. Users upload images to an S3 bucket
2. EventBridge detects the upload and triggers Step Functions
3. Step Functions orchestrates the workflow:
   - Validates image file extension
   - Invokes Rekognition for label detection
   - Sends SMS alert via SNS if human faces are detected
4. Lambda function ensures clean stack deletion by emptying the bucket

## Detailed Component Breakdown

### 1. S3 Bucket Configuration
- **Properties**: Versioning enabled, server access logging (optional)
- **EventBridge Integration**: EventBridge Rule with S3 as source, filtering for `ObjectCreated:*` events
- **Security**: Bucket Policy allowing EventBridge to invoke Step Functions

### 2. Step Functions State Machine
- **Definition**: JSON-based workflow with error handling and retry logic
- **States**:
  - `ValidateImage`: Checks file extension (.jpg, .jpeg, .png)
  - `RekognitionDetectLabels`: Invokes Rekognition with "Human" and "Person" labels
  - `CheckResults`: Evaluates Rekognition response
  - `SendAlert`: Publishes to SNS topic if faces detected
  - `Cleanup`: Final state for workflow completion

### 3. IAM Role Configuration

**EventBridge Role**:
- Trust policy allowing EventBridge to assume role
- Permissions: `states:StartExecution` on the Step Functions state machine

**Step Functions Role**:
- Trust policy allowing Step Functions to assume role
- Permissions:
  - `s3:GetObject` on the bucket
  - `rekognition:DetectLabels` on Rekognition
  - `sns:Publish` on the SNS topic
  - `states:GetExecutionHistory` for logging

### 4. Lambda Function for Cleanup
- **Runtime**: Python 3.12 (latest supported)
- **IAM Role**: Permissions to list and delete objects in the S3 bucket
- **Logic**: Lists all objects and deletes them recursively
- **Integration**: Custom resource with `DELETE` request type

### 5. SNS Topic Configuration
- **Properties**: SMS subscription with the provided phone number
- **Message**: Customizable alert message indicating human detection

## Properties and Dependencies

### S3 Bucket Dependencies
- IAM Role for EventBridge must exist before creating EventBridge Rule
- EventBridge Rule depends on both S3 bucket and IAM Role

### Step Functions Dependencies
- IAM Role must exist before creating state machine
- State machine depends on IAM Role

### Lambda Function Dependencies
- IAM Role must exist before creating Lambda function
- Lambda function depends on IAM Role

### SNS Topic Dependencies
- IAM Role for Step Functions must exist before creating SNS topic
- SNS topic depends on IAM Role

## Safe AWS Defaults

### Missing Properties
- **Region**: Use `AWS::Region` pseudo parameter
- **Account ID**: Use `AWS::AccountId` pseudo parameter
- **Image Extensions**: Default to `["jpg", "jpeg", "png"]`
- **Rekognition Labels**: Default to `["Human", "Person"]`

### Dynamic Resource Fetching
- **AMIs**: Not applicable (no EC2 instances)
- **AZs**: Use `Fn::GetAZs` with empty string for current region
- **Region-specific resources**: Use pseudo parameters

## Security Considerations

1. **Least Privilege**: Each IAM role has only the permissions it needs
2. **Encryption**: S3 bucket can be configured with SSE-S3 (default)
3. **Logging**: Enable CloudTrail for all API calls (optional)
4. **Monitoring**: CloudWatch alarms for failed executions (optional)

## Error Handling Strategy

1. **Image Validation**: Return error if invalid file type
2. **Rekognition Failures**: Retry with exponential backoff (3 attempts)
3. **SNS Failures**: Log error and continue (non-critical)
4. **Step Functions**: Implement catch-all error handling with notifications

## Deployment Sequence

1. IAM Roles (EventBridge, Step Functions, Lambda)
2. S3 Bucket
3. SNS Topic
4. Step Functions State Machine
5. EventBridge Rule
6. Lambda Function (Custom Resource)
7. EventBridge Rule Target (Step Functions execution)

This architecture ensures a robust, secure, and maintainable image recognition pipeline that meets all specified requirements while adhering to AWS best practices.
