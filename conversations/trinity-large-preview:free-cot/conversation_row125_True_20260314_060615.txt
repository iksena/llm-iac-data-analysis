- **AWS Systems Manager (SSM) Parameter Store** - To dynamically fetch latest AMI IDs (if needed for future extensions)
- **AWS Key Management Service (KMS)** - For API key encryption (implicit in API Gateway)
- **Amazon CloudWatch Logs** - For Lambda logging and monitoring

- **Lambda Execution Role**: Requires SES send permissions and CloudWatch logging permissions

For this template, dynamic fetching is primarily relevant for:

- **Lambda Runtime**: Using the latest Python 3.12 runtime
- **SES Actions**: Using the standard `ses:SendEmail` action (no dynamic fetching needed)

## Architecture Overview

The solution will follow this flow:

1. **API Gateway** receives POST request with JSON payload
2. **Request Validation** ensures payload matches JSON schema
3. **API Key Authentication** verifies usage plan compliance
4. **API Gateway** invokes Lambda function with validated payload
5. **Lambda Function** transforms JSON to SES-compatible format
6. **Lambda** calls SES `SendEmail` API with transformed data
7. **Response** returns success/failure to API Gateway client

## Security Implementation

- **API Key Authentication**: Enforced at method level
- **Usage Plan**: Configured with 500 req/sec throttle and 10,000 monthly quota
- **IAM Role**: Least-privilege permissions for SES send only
- **Request Validation**: JSON schema validation to prevent malformed requests
- **Encryption**: API keys encrypted at rest using KMS

## Error Handling Strategy

The Lambda function will implement comprehensive error handling:

- **400**: Invalid JSON schema
- **401**: Missing/invalid API key
- **429**: Rate limit exceeded
- **500**: SES service errors or Lambda failures
- **200**: Successful email delivery

## Template Structure Plan

The CloudFormation template will be organized as follows:

1. **Parameters** (with safe defaults)
2. **Mappings** (if needed for region-specific configurations)
3. **Resources**:
   - IAM Role for Lambda
   - Lambda Function
   - API Gateway REST API
   - API Gateway Resource
   - API Gateway Method (POST)
   - API Gateway Integration
   - API Gateway Usage Plan
   - API Gateway API Key
   - API Gateway Stage
4. **Outputs** (API endpoint URL, API key value)

## JSON Schema Design

The request validation will enforce this structure:

```json
{
  "type": "object",
  "properties": {
    "to": {"type": "string", "format": "email"},
    "from": {"type": "string", "format": "email"},
    "subject": {"type": "string"},
    "text": {"type": "string"},
    "html": {"type": "string"}
  },
  "required": ["to", "from", "subject", "text"]
}
```

This ensures all required fields are present and properly formatted before reaching the Lambda function.

## Lambda Function Logic

The Lambda function will:
1. Parse the incoming event
2. Extract email parameters from JSON
3. Construct SES `SendEmail` request
4. Call SES API
5. Return appropriate HTTP response

The function will be written in Python 3.12 and will use the boto3 SES client.

This architectural plan ensures a secure, scalable, and production-ready email sending service via REST API, adhering to all specified requirements and AWS best practices.
