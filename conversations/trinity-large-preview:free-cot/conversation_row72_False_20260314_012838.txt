# Detailed Architectural Deployment Plan for Secure VS Code Development Environment

## Key AWS Services Required

1. **VPC** - Virtual Private Cloud for network isolation
2. **InternetGateway** - Provides internet connectivity for the VPC
3. **VPCGatewayAttachment** - Attaches the Internet Gateway to the VPC
4. **Subnet** - Public subnet within the VPC
5. **RouteTable** - Manages routing for the subnet
6. **Route** - Defines the default route to the Internet Gateway
7. **SubnetRouteTableAssociation** - Associates the subnet with the route table
8. **EC2 Instance** - Ubuntu 22.04 server running code-server
9. **IAM Role** - Instance profile for EC2 with SSM permissions
10. **SecurityGroup** - Controls inbound traffic from CloudFront IP ranges
11. **CloudFront Distribution** - Global HTTPS access and caching
12. **CloudFront OriginAccessIdentity** - Secure access to the EC2 instance
13. **SSM Parameter Store** - Dynamic AMI fetching for Ubuntu 22.04

## Properties and Dependencies

### Critical Dependencies
1. VPC must be created before Internet Gateway attachment
2. Internet Gateway must be attached before creating routes
3. Security Group must be created before EC2 instance
4. IAM Role must be created before EC2 instance
5. CloudFront Distribution must be created after EC2 instance is available

### Property Dependencies
- EC2 instance requires IAM Role ARN
- Security Group requires VPC ID
- Subnet requires VPC ID
- Route requires Route Table ID and Internet Gateway ID
- CloudFront Distribution requires EC2 instance's public IP

## Safe AWS Defaults (Where Properties Were Missing)

### Network Configuration
- **VPC CIDR**: `10.0.0.0/16` (standard private range)
- **Subnet CIDR**: `10.0.1.0/24` (single public subnet)
- **EC2 Instance Type**: `t3.medium` (balanced performance for development)
- **SSH Key Pair**: `default-key` (safe default parameter)

### CloudFront Configuration
- **Viewer Protocol Policy**: `redirect-to-https` (enforces HTTPS)
- **Allowed HTTP Methods**: `GET, HEAD, OPTIONS, PUT, POST, PATCH, DELETE` (full CRUD support)
- **Cache TTL**: `31536000` seconds (1 year as specified)
- **Compress**: `true` (enables compression for better performance)

### Code-Server Configuration
- **Port**: `8080` (as specified)
- **Authentication**: Disabled (as specified for demo use)
- **Version**: Parameter with default `4.10.0` (latest stable)

## Dynamic Resource Fetching Strategy

### Ubuntu 22.04 AMI
Use SSM Parameter Store to fetch the latest Ubuntu 22.04 AMI:
```
resolve:ssm:/aws/service/canonical/ubuntu/server/22.04/stable/current/amd64/hvm/ebs-gp2/ami-id
```

### CloudFront IP Ranges
Use AWS CLI or SSM to fetch current CloudFront IP ranges dynamically:
```
https://ip-ranges.amazonaws.com/ip-ranges.json
```

## Security Considerations and Trade-offs

### Security Group Design
- Restricts inbound traffic to CloudFront's regional IP ranges only
- Allows HTTP traffic on port 8080 (code-server default)
- No SSH access from internet (managed via SSM)

### IAM Role Design
- Includes `AmazonSSMManagedInstanceCore` policy for SSM access
- No additional permissions beyond what's necessary

### CloudFront Distribution
- Custom caching policy with 1-year TTL
- Request forwarding enabled for development flexibility
- HTTPS enforcement for secure access

## Architecture Flow

1. **Network Layer**: VPC → Internet Gateway → Subnet → Route Table
2. **Security Layer**: IAM Role → Security Group
3. **Compute Layer**: EC2 Instance with Ubuntu 22.04 and code-server
4. **Distribution Layer**: CloudFront Distribution with Origin Access Identity
5. **Management Layer**: SSM integration for instance management

## Output Values

The template will output:
- CloudFront Distribution URL (for HTTPS access)
- EC2 Instance Public IP (for direct access if needed)
- Security Group ID (for reference)
- IAM Instance Profile ARN (for reference)

## Production Hardening Recommendations

For production deployment, the following changes would be necessary:
1. Enable authentication in code-server
2. Add TLS termination at the origin
3. Implement VPC endpoints for private connectivity
4. Add monitoring and logging
5. Implement backup and disaster recovery
6. Add cost controls and budget alerts
7. Implement proper access controls and audit logging

This architecture prioritizes simplicity and rapid deployment for demo purposes while maintaining a secure foundation that can be hardened for production use.
