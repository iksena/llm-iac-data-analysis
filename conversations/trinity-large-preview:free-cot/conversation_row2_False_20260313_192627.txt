Based on AWS best practices and the requirement for automation:

- **Bucket Name**: Use a parameterized name with a default value that includes the stack name and timestamp to ensure uniqueness
- **Region**: Default to `AWS::Region` (the stack's region)
- **Versioning**: Enabled by default for data protection
- **Encryption**: SSE-S3 (server-side encryption with S3-managed keys)
- **Access Control**: Public access blocked by default
- **DeletionPolicy**: `Retain` to prevent accidental data loss
- **UpdateReplacePolicy**: `Retain` to match deletion policy

- **SSM Parameter Store** for latest AMI IDs: `/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2`
- **Fn::GetAZs** for availability zones instead of hardcoding
- **Fn::Select** for selecting specific AZs from the list

- **Public Access**: Block all public access by default
- **Encryption**: Enable SSE-S3 for data at rest
- **Access Logging**: Optional but recommended for audit trails
- **Tags**: Include standard tags for cost allocation and management

   - Optional S3 Bucket Policy for specific access requirements
   - Optional S3 Bucket Analytics Configuration

3. **Outputs Section**:
   - Bucket ARN
   - Bucket Name
   - Website URL (if configured as static website)

To ensure cfn-lint compliance:

1. All parameters will have safe default values
