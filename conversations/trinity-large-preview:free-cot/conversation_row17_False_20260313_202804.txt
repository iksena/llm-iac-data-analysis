   - Name: "Monthly-Billing-Alarm-20USD"
   - Threshold: 20 USD
   - Same configuration as Alarm 1, different threshold

4. **Alarm 4**:
   - Name: "Monthly-Billing-Alarm-40USD"
   - Threshold: 40 USD
   - Same configuration as Alarm 1, different threshold

The CloudFormation template will include:

1. **Parameters**:
   - Email address for notifications (with default placeholder)
   - Currency (with default USD)

2. **Resources**:
   - SNS Topic
   - SNS Subscription
   - Four CloudWatch Alarms

### Critical Implementation Details

**Billing Metric Requirements**:
- The `EstimatedCharges` metric requires specific dimensions:
  - `ServiceName`: Must be set to "AWS" to monitor all services
  - `Currency`: Must match the currency used (default USD)

**Alarm Configuration**:
- Using 6-hour periods balances between timely alerts and avoiding alert fatigue
- Each alarm triggers independently when its threshold is breached
- All alarms will send notifications to the same SNS topic

**SNS Subscription**:
- The email subscription will be created but requires manual confirmation
- Users must click the confirmation link in the subscription email to activate notifications

**Template Best Practices**:
- Will use `AWS::SNS::Topic` and `AWS::CloudWatch::Alarm` resources
- Will include appropriate `DependsOn` relationships
- Will use `Fn::GetAtt` to reference the SNS topic ARN in alarm configurations

This architecture provides a scalable, maintainable solution for monitoring AWS costs with graduated alert thresholds, allowing users to respond appropriately as spending increases.
