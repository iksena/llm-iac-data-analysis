- **EC2 Image Builder Resources**: Image Pipeline, Image Recipe, and Infrastructure Configuration
- **SSM Parameter Store**: For dynamic AMI fetching

1. **IAM Role Creation**: Must create the IAM role before the instance profile
2. **S3 Bucket Creation**: Must create the bucket before attaching the policy
3. **Image Builder Resources**: Must create the infrastructure configuration before the image recipe, and the recipe before the pipeline

### Required Properties:
- **IAM Role**: `AssumeRolePolicyDocument` with EC2 and Image Builder service principals
- **S3 Bucket**: `PublicAccessBlockConfiguration` with all public access blocked
- **IAM Policy**: `PolicyDocument` with S3 permissions for the logs bucket
- **Image Builder**: `Name`, `Description`, `Platform`, and `SemanticVersion`

## 3. Safe AWS Defaults (Where Information is Missing)

Since specific values aren't provided, I'll use these safe defaults:

- **VPC & Subnets**: Use the default VPC and its subnets (avoiding hardcoded VPC creation)
- **Security Group**: Allow SSH (port 22) from 0.0.0.0/0 for testing (should be restricted in production)
- **Instance Type**: `t3.micro` (free tier eligible, suitable for testing)
- **AMI**: Use SSM Parameter Store to fetch the latest Amazon Linux 2 AMI dynamically
- **Owner Email**: Default to `default-owner@example.com` (must match AllowedPattern if specified)
- **Region**: Use `AWS::Region` pseudo parameter for region-specific resources

## 4. Dynamic Resource Fetching Strategy

### AMI Fetching:
- Use `Fn::ImportValue` or `Fn::Sub` with SSM Parameter Store:
  ```
  AMI: !Sub 'resolve:ssm:/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2'
  ```

### Availability Zones:
- Use `Fn::GetAZs` and `Fn::Select` to avoid hardcoding:
  ```
  AZ: !Select [0, !GetAZs '']
  ```

## 5. Detailed Resource Architecture

### 1. IAM Role and Instance Profile
- **Role Name**: `ImageBuilderRole`
- **Policies**: 
  - AWS managed policy: `AmazonSSMManagedInstanceCore`
  - Inline policy for S3 logs bucket access
- **Instance Profile**: `ImageBuilderInstanceProfile`

### 2. S3 Bucket for Logs
- **Bucket Name**: Dynamically generated with stack name and random suffix
- **Public Access**: All blocked using `PublicAccessBlockConfiguration`
- **Tags**: `OwnerEmail: u12345678@anu.edu.au`
- **Encryption**: SSE-S3 enabled by default

### 3. IAM Managed Policy
- **Policy Name**: `ImageBuilderS3LogsPolicy`
- **Permissions**: 
  - `s3:PutObject` and `s3:GetObject` on the logs bucket
  - `s3:ListBucket` on the logs bucket

### 4. EC2 Image Builder Resources
- **Infrastructure Configuration**:
  - Instance type: `t3.micro`
  - Security group: Allow SSH
  - SNS topic: For build notifications
- **Image Recipe**:
  - Components: Use AWS managed components for Amazon Linux 2
  - Parent image: Fetched dynamically via SSM
- **Image Pipeline**:
  - Schedule: Daily builds
  - Infrastructure configuration: Reference created above

## 6. Security and Best Practices Implementation

### Security Measures:
- **Least Privilege**: IAM policies grant only necessary permissions
- **Encryption**: S3 bucket encrypted at rest
- **No Hardcoded Secrets**: Use dynamic references for any sensitive data
- **Resource Protection**: Apply `DeletionPolicy: Retain` for S3 bucket

### CloudFormation Best Practices:
- **Parameters with Defaults**: All parameters have safe default values
- **Linting Compliance**: No hardcoded AMIs, AZs, or invalid properties
- **Modular Design**: Resources logically grouped and dependent

## 7. Potential Challenges and Mitigations

### Challenge 1: S3 Bucket Naming
- **Issue**: Bucket names must be globally unique
- **Mitigation**: Use stack name and timestamp in bucket name

### Challenge 2: IAM Policy Limits
- **Issue**: Inline policies have size limits
- **Mitigation**: Use managed policies where possible, keep inline policies minimal

### Challenge 3: Image Builder Component Compatibility
- **Issue**: Components must match the OS version
- **Mitigation**: Use AWS managed components designed for Amazon Linux 2

This architecture ensures a secure, automated, and maintainable EC2 Image Builder setup that adheres to AWS best practices and will pass cfn-lint validation.
