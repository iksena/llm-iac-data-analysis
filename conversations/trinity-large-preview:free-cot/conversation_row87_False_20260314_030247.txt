### Key Properties Required:
- S3 Bucket: BucketName, AccessControl (or BucketPolicy), PublicAccessBlockConfiguration
- KMS Key: Description, KeyUsage, CustomerMasterKeySpec, KeyPolicy, EnableKeyRotation
- KMS Alias: AliasName, TargetKeyId
- IAM Policies: PolicyDocument with specific KMS permissions

- **KMS Key Rotation**: Enabled (true)
- **KMS Alias Name**: "alias/lambda-artifacts-key"
- **IAM Policy Names**: "LambdaArtifactsKMSAdminPolicy", "LambdaArtifactsKMSUsagePolicy"

For this template, dynamic fetching is not required for AMIs or AZs since we're only creating S3 and KMS resources. However, I'll use dynamic naming for the S3 bucket to ensure uniqueness across AWS accounts.

## 5. Architecture Plan

- Create a bucket with a unique name using Fn::Join and Fn::Select to generate a random suffix
- Configure public access using a BucketPolicy rather than the deprecated AccessControl property
- Implement PublicAccessBlockConfiguration to explicitly allow public access
- Use AWS::S3::BucketPolicy to grant public read access to the bucket

### KMS Key Configuration:
- Create a symmetric customer-managed KMS key
- Enable automatic key rotation
- Define a key policy that allows:
  - Root account full administrative access
  - Imported service role administrative and usage access
- Create a user-friendly alias for simplified key referencing

### IAM Policies:
- Create separate policies for administrative and usage permissions
- Grant the root account full KMS permissions
- Grant the imported service role both administrative and usage permissions
- Use dynamic references to the KMS key ARN in policy documents

### Security Considerations:
- Use least-privilege principles in IAM policies
- Enable key rotation for enhanced security
- Use explicit public access configuration rather than relying on defaults
- Implement proper resource naming conventions for easy identification

### Template Structure:
1. Parameters (if any required inputs)
2. Resources section with:
   - S3 Bucket
   - S3 Bucket Policy
   - KMS Key
   - KMS Alias
   - IAM Policies
3. Outputs (if needed for cross-stack references)

This architecture ensures a secure, publicly accessible S3 bucket for Lambda artifacts with proper encryption via KMS, while maintaining granular access control through IAM policies.

AWSTemplateFormatVersion: '2010-09-09'
Description: CloudFormation template to create an S3 bucket for Lambda artifacts and a KMS key for encryption

Resources:
  LambdaArtifactsBucket:
    Type: AWS::S3::Bucket
    Properties:
      BucketName: !Join
        - '-'
        - - 'lambda-artifacts'
          - !Select
            - 0
            - Fn::Split:
              - '-'
              - Fn::Select:
                - 0
                - Fn::GetAZs: ''
      PublicAccessBlockConfiguration:
        BlockPublicAcls: false
        BlockPublicPolicy: false
        IgnorePublicAcls: false
        RestrictPublicBuckets: false

  LambdaArtifactsBucketPolicy:
    Type: AWS::S3::BucketPolicy
    Properties:
      Bucket: !Ref LambdaArtifactsBucket
      PolicyDocument:
        Version: '2012-10-17'
        Statement:
          - Effect: Allow
            Principal: '*'
            Action: s3:GetObject
            Resource: !Sub 'arn:${AWS::Partition}:s3:::${LambdaArtifactsBucket}/*'
