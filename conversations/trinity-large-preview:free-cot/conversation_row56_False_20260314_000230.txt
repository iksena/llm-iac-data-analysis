# Secure Amazon Cognito User Pool Deployment Architecture Plan

## Key AWS Services Required

1. **AWS::Cognito::UserPool** - Core user directory for authentication and authorization
2. **AWS::Cognito::UserPoolClient** - Application client for accessing the user pool
3. **AWS::Cognito::UserPoolDomain** - Custom domain for user pool (optional but recommended)
4. **AWS::Cognito::UserPoolResourceServer** - OAuth2 resource server for API protection
5. **AWS::Cognito::UserPoolGroup** - Administrator group for role-based access control
6. **AWS::Cognito::UserPoolUserToGroupAttachment** - Initial administrator user assignment
7. **AWS::Cognito::UserPoolUICustomization** - Custom UI branding (optional)
8. **AWS::Cognito::UserPoolIdentityProvider** - External identity provider integration (if needed)
9. **AWS::Cognito::UserPoolRiskConfigurationAttachment** - Adaptive authentication configuration
10. **AWS::Cognito::UserPoolResourceServerScope** - API scope definitions

## Properties and Dependencies

### Authentication Configuration Dependencies
- Email verification requires an email source (SES or Cognito-managed)
- MFA configuration depends on SMS configuration availability
- Admin-only user creation requires disabling self-registration

### Resource Dependencies
- UserPoolDomain depends on UserPool creation completion
- UserPoolClient depends on UserPool existence
- UserPoolGroup and UserToGroupAttachment depend on UserPool creation
- ResourceServer depends on UserPool creation

- VPC CIDR: `10.0.0.0/16`

- Availability Zones: `Fn::GetAZs` with `Fn::Select` for first AZ

### Cognito Configuration Defaults
- UserPool name: `SecureUserPool-${AWS::AccountId}`
- AutoVerifiedAttributes: `email`
- EmailVerificationMessage: `Your verification code is {####}`
- EmailVerificationSubject: `Your verification code`
- UsernameAttributes: `email`
- Policies: Minimum password length: `8`
- Schema attributes: email (required), name (required), given_name, family_name, phone_number

### Security Defaults
- AccountRecoverySetting: `verified_email`
- MfaConfiguration: `OFF` (can be enabled later)
- AdminCreateUserConfig: `AllowAdminCreateUserOnly: true`
- DeviceConfiguration: `ChallengeRequiredOnNewDevice: true`

### Region-Aware Configuration
- Use `AWS::Region` pseudo parameter for region-specific configurations

- Use `AWS::URLSuffix` for domain configurations

## Security Best Practices Implementation

### Password Policy
- Minimum length: 8 characters
- Require uppercase, lowercase, numbers, and special characters
- Prevent password reuse (count: 5)
- Temporary password validity: 7 days

### Account Recovery
- Only allow verified email for account recovery
- No SMS recovery to prevent social engineering
- Email verification required for all new accounts

### Admin Controls
- Disable self-registration
- Require admin approval for new users
- Enable email verification for all users
- Log all authentication events

## Template Structure Plan

### Parameters Section
- Domain prefix (with safe default)
- Admin email (with validation pattern)

- Custom UI logo URL (optional)

### Resources Section

4. UserPoolGroup for administrators
5. Initial admin user assignment
6. ResourceServer for API protection
7. UI Customization for branding
8. Risk Configuration for adaptive authentication

### Outputs Section
- UserPool ID and ARN
- UserPoolDomain URL
- Client ID and secret
- Administrator group ARN

## Compliance and Monitoring

### Logging Configuration
- Enable CloudWatch Logs for authentication events
- Configure log retention policy (30 days default)
- Set up log metric filters for suspicious activities

### Compliance Controls
- Enforce password complexity requirements
- Enable multi-factor authentication option
- Configure account takeover protection
- Implement adaptive authentication based on risk
