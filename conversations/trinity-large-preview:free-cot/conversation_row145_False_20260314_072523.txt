# Detailed Architectural Deployment Plan for Secure Static Website on AWS

## Key AWS Services Required

1. **AWS::S3::Bucket** - For static website hosting and content storage
2. **AWS::CloudFront::Distribution** - For global CDN delivery and HTTPS enforcement
3. **AWS::CloudFront::OriginAccessControl** - For secure S3 access control (modern replacement for OAI)
4. **AWS::IAM::Role** - For Lambda execution permissions
5. **AWS::Lambda::Function** - For automated bucket initialization and cleanup
6. **AWS::Lambda::Permission** - For CloudFormation custom resource invocation
7. **AWS::S3::BucketPolicy** - For securing bucket access
8. **AWS::CloudFormation::CustomResource** - For triggering Lambda during stack operations

## Architecture Overview

The solution implements a secure, automated static website deployment with the following flow:
- S3 bucket configured for static website hosting with Origin Access Control
- CloudFront distribution with custom cache policy and HTTPS enforcement
- Lambda custom resource for automated content management during stack lifecycle
- Secure access control preventing direct S3 access

## Detailed Implementation Plan

### 1. S3 Bucket Configuration

**Properties:**
- `AccessControl: BucketOwnerFullControl` (required for Origin Access Control)
- `WebsiteConfiguration` with `IndexDocument: index.html`
- `OwnershipControls` with `Rules` containing `ObjectOwnership: BucketOwnerPreferred`
- `PublicAccessBlockConfiguration` to block public access

**Dependencies:**
- Bucket must be created before Origin Access Control
- Bucket policy must reference Origin Access Control ID

### 2. Origin Access Control (OAC)

**Properties:**
- `Description: Secure S3 access for CloudFront`
- `OriginAccessControlOriginType: S3`
- `SigningProtocol: sigv4`
- `SigningBehavior: always`

**Dependencies:**
- Must be created before CloudFront distribution
- Referenced by CloudFront distribution's S3 origin

### 3. CloudFront Distribution

**Properties:**
- `Enabled: true`
- `DefaultRootObject: index.html`
- `HttpVersion: http2`
- `IPV6Enabled: true`
- `PriceClass: PriceClass_100` (US, EU, Japan)

**Origins:**
- S3 origin with `OriginAccessControlId` referencing OAC
- `DomainName` from S3 bucket's website endpoint

**Cache Policy:**
- Custom cache policy with `MinTTL: 60`, `MaxTTL: 31536000`, `DefaultTTL: 60`
- Query string and cookie forwarding disabled for optimal caching

**Viewer Certificate:**
- `AcmCertificateArn` (parameter with default to allow deployment)
- `SslSupportMethod: sni-only`
- `MinimumProtocolVersion: TLSv1.2_2021`

**Dependencies:**
- Requires OAC to be created first
- Requires S3 bucket to exist

### 4. Lambda Custom Resource

**Runtime:** `python3.12` (latest supported)

**IAM Role:**
- `AWSLambdaBasicExecutionRole` managed policy
- `AmazonS3FullAccess` for bucket operations
- `AWSL3ObjectLambdaFullAccess` if using Lambda@Edge

**Function Logic:**
- **Create:** Upload default `index.html` with basic content
- **Delete:** Empty bucket contents before deletion
- **Update:** No operation (idempotent)

**Dependencies:**
- IAM role must exist before Lambda creation
- Lambda permission required for CloudFormation invocation

### 5. S3 Bucket Policy

**Properties:**
- Policy document allowing CloudFront OAC to access bucket objects
- Restricts access to only CloudFront distribution

**Dependencies:**
- Requires OAC ID and CloudFront distribution ARN

### 6. Parameters with Safe Defaults

**Required Parameters:**
- `CertificateArn`: Default to empty string (allows deployment without HTTPS initially)
- `WebsiteIndexDocument`: Default `index.html`
- `CacheTTL`: Default `60` (matches cache policy)
- `LambdaRuntime`: Default `python3.12`

**Optional Parameters:**
- `PriceClass`: Default `PriceClass_100`
- `HttpVersion`: Default `http2`

### 7. SSM Parameter Usage

**Dynamic References:**
- No AMIs required for this static website solution
- If Lambda layer needed: Use `resolve:ssm:/aws/service/lambda/python/lib-layer` for latest layer ARN

### 8. Error Handling and Best Practices

**Deletion Policies:**
- S3 bucket: No deletion policy (manual cleanup recommended)
- Lambda: `Delete` (safe for stateless functions)
- IAM Role: `Delete` (safe with proper policy design)

**UpdateReplacePolicies:**
- Match deletion policies for stateful resources
- Lambda: `Delete` (safe for stateless functions)

**Security:**
- Use dynamic references for any secrets (none required here)
- Principle of least privilege for IAM roles
- HTTPS enforcement at CloudFront level

### 9. Stack Lifecycle Management

**Creation Sequence:**
1. IAM Role
2. Lambda Function
3. S3 Bucket
4. Origin Access Control
5. CloudFront Distribution
6. Bucket Policy
7. Custom Resource Permission

**Deletion Sequence:**
1. Custom Resource Permission
2. Bucket Policy
3. CloudFront Distribution
4. Origin Access Control
5. S3 Bucket (emptied by Lambda)
6. Lambda Function
7. IAM Role

### 10. Validation and Testing

**Pre-deployment:**
- Validate CloudFormation template with cfn-lint
- Test Lambda function locally with test events
- Verify parameter defaults and patterns

**Post-deployment:**
- Test website accessibility via CloudFront URL
- Verify HTTPS enforcement
- Confirm cache behavior with TTL settings
- Validate OAC security by attempting direct S3 access (should fail)

This architecture ensures a secure, automated static website deployment that follows AWS best practices and passes all linting requirements while providing optimal performance through CloudFront caching and secure access control.
