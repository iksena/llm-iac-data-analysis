# Architectural Deployment Plan for Secure Static Website on AWS

## Key AWS Services Required

1. **S3** - For static website hosting and content storage
2. **CloudFront** - For global CDN distribution and HTTPS enforcement
3. **Lambda** - For automated bucket initialization and cleanup via custom resource
4. **IAM** - For service permissions and security policies
5. **ACM** (optional) - For SSL certificate management if custom domain is used

## Architecture Overview

The solution implements a secure, automated static website deployment with the following key components:

1. **S3 Bucket** with Origin Access Control (OAC) for secure CloudFront access
2. **CloudFront Distribution** with custom cache policy and HTTPS enforcement
3. **Lambda Custom Resource** for automated bucket initialization and cleanup
4. **IAM Roles** for Lambda execution and CloudFront access

## Detailed Component Analysis

### 1. S3 Bucket Configuration

**Properties:**
- `AccessControl: BucketOwnerFullControl` (required for OAC to work properly)
- `OwnershipControls` with `ObjectOwnership: BucketOwnerPreferred`
- `WebsiteConfiguration` for static hosting with `IndexDocument: index.html`
- `PublicAccessBlockConfiguration` to block public access
- `CorsConfiguration` for cross-origin requests if needed

**Dependencies:**
- OAC must be created before CloudFront distribution
- Bucket policy must reference OAC canonical user ID

### 2. Origin Access Control (OAC)

**Properties:**
- `OriginAccessControlOriginType: s3`
- `SigningProtocol: sigv4`
- `SigningBehavior: when-in-use`
- `Description` for documentation

**Dependencies:**
- Must be created before CloudFront distribution
- References the S3 bucket's Amazon Resource Name (ARN)

### 3. CloudFront Distribution

**Properties:**
- `Enabled: true`
- `DefaultRootObject: index.html`
- `HttpVersion: http2`
- `IPV6Enabled: true`
- `PriceClass: PriceClass_100` (US, Europe, Israel)
- `ViewerCertificate` with `MinimumProtocolVersion: TLSv1.2_2021`
- `Restrictions` with `GeoRestriction: none`

**Cache Policy:**
- Custom cache policy with `MinTTL: 60`, `MaxTTL: 31536000`, `DefaultTTL: 60`
- `HeaderBehavior: none`
- `QueryStringBehavior: none`
- `CookieBehavior: none`

**Origin Configuration:**
- References OAC for secure S3 access
- `OriginReadTimeout: 60`
- `OriginKeepaliveTimeout: 5`

### 4. Lambda Custom Resource

**Runtime:** `python3.12` (latest supported)

**IAM Role:**
- `AWSLambdaBasicExecutionRole` for logging
- `AWSLambdaExecute` for S3 access
- Custom inline policy for bucket operations

**Handler Logic:**
- `onCreate`: Uploads default `index.html` with basic content
- `onUpdate`: No operation (idempotent)
- `onDelete`: Empties bucket before deletion to avoid retention conflicts

### 5. Security & Best Practices

**IAM Policies:**
- Principle of least privilege for all roles
- No hardcoded credentials
- Dynamic references for any secrets

**HTTPS Enforcement:**
- CloudFront viewer policy enforces HTTPS
- S3 bucket blocks public access
- OAC ensures only CloudFront can access content

## Dependencies and Execution Order

1. **IAM Role** → Lambda function creation
2. **S3 Bucket** → Bucket creation with ownership controls
3. **OAC** → Origin access control creation
4. **CloudFront Distribution** → CDN setup with OAC reference
5. **Lambda Custom Resource** → Automated content management

## Safe Defaults (as required)

**Missing Properties:**
- **Region:** `AWS::Region` (inferred from stack)
- **ACM Certificate:** Not used (CloudFront default certificate)
- **Custom Domain:** Not configured (uses CloudFront default domain)
- **Logging:** Disabled (can be enabled with S3 logging bucket)

**Safe Parameter Defaults:**
- **Bucket Name:** `${AWS::StackName}-static-website`
- **Lambda Timeout:** `30` seconds
- **Cache TTL:** `60` seconds (as specified)
- **Price Class:** `PriceClass_100`

## Dynamic Resource Fetching

**AMIs:** Not applicable (no EC2 instances)
**AZs:** Use `Fn::GetAZs` and `Fn::Select` for any AZ-specific configurations
**Latest Runtime:** Use `python3.12` for Lambda

## Linting Compliance Strategy

1. **No hardcoded AMIs** - Not applicable
2. **No hardcoded AZs** - Use intrinsic functions
3. **S3 Modernization** - Use `OwnershipControls` with `AccessControl`
4. **Deletion/Update Policies** - Apply to Lambda function
5. **No unnecessary Fn::Sub** - Use direct string references
6. **Latest Runtimes** - Use `python3.12`
7. **Exact IAM Policy ARNs** - Use `arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole`

This architecture ensures a secure, automated, and production-ready static website deployment that adheres to AWS best practices and CloudFormation linting rules.
