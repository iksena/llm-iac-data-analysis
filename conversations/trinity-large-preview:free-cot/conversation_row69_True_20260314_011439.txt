## 2. Properties and Dependencies

- VPC requires CIDR block (default: 10.0.0.0/16)
- Internet Gateway must be attached to VPC before any route tables can reference it
- Subnets require VPC ID and Availability Zone
- Route Tables require VPC ID

### Security Group Dependencies:
- Security Group requires VPC ID
- Security Group rules require referencing the Security Group itself

### S3 Bucket Dependencies:
- S3 Bucket requires unique name (using Fn::Join with stack name)
- Bucket Policy requires bucket ARN

### DynamoDB Table Dependencies:
- DynamoDB Table requires provisioned throughput settings
- Table name should be unique (using stack name)

## 3. Safe AWS Defaults

### VPC Defaults:
- CIDR Block: `10.0.0.0/16` (standard private range)
- Number of Availability Zones: 2 (for high availability)

- Private Subnet CIDR: `10.0.10.0/24`, `10.0.11.0/24`

### Security Group Defaults:
- VPC ID: Referenced from VPC resource
- Inbound rules: Allow SSH (port 22) from 0.0.0.0/0 (can be restricted later)
- Outbound rules: Allow all traffic to 0.0.0.0/0

### S3 Bucket Defaults:
- Bucket Name: `stack-name-deletion-protected-bucket`
- Access Control: Private (using Bucket Policy)
- Deletion Protection: Enabled via DeletionPolicy

### DynamoDB Table Defaults:
- Table Name: `stack-name-deletion-protected-table`
- Primary Key: `id` (String type)

- Deletion Protection: Enabled via DeletionPolicy

- All resources will use stack-specific naming to avoid conflicts
