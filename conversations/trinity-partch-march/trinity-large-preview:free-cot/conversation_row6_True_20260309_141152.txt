# EC2 Instance with Security Group Deployment Plan

## Key AWS Services Required
1. **VPC** - Virtual Private Cloud for network isolation
2. **Internet Gateway** - For internet connectivity
3. **Route Table** - To route traffic to the internet
4. **Subnet** - Network segment for the EC2 instance
5. **Security Group** - Network firewall for the EC2 instance
6. **EC2 Instance** - The compute resource
7. **IAM Role** (optional) - For instance profile permissions

## Architecture Design

### Network Configuration
- **VPC**: Single VPC with a /16 CIDR block (default: 10.0.0.0/16)
- **Subnet**: Single public subnet with /24 CIDR block (default: 10.0.0.0/24)
- **Internet Gateway**: Attached to VPC for internet access
- **Route Table**: Default route (0.0.0.0/0) pointing to Internet Gateway

### Security Group Configuration
- **Inbound Rules**:
  - SSH (port 22) from anywhere (0.0.0.0/0) - for management access
  - HTTP (port 80) from anywhere (0.0.0.0/0) - for web access
- **Outbound Rules**: All traffic (0.0.0.0/0) on all ports

### EC2 Instance Configuration
- **AMI**: Amazon Linux 2 (fetched dynamically via SSM Parameter Store)
- **Instance Type**: t3.micro (free tier eligible)
- **Key Pair**: Parameter with default value 'default-key'
- **IAM Role**: Optional instance profile for enhanced permissions

## Properties and Dependencies

### Critical Dependencies
1. VPC must be created before Internet Gateway
2. Internet Gateway must be attached to VPC before Route Table creation
3. Route Table must be associated with Subnet
4. Security Group must be created before EC2 instance

### Property Values (Safe Defaults)
- **VPC CIDR**: 10.0.0.0/16
- **Subnet CIDR**: 10.0.0.0/24
- **Availability Zone**: First AZ from region (using Fn::GetAZs and Fn::Select)
- **Instance Type**: t3.micro
- **SSH Key Pair**: Parameter with default 'default-key'
- **AMI**: Latest Amazon Linux 2 via SSM Parameter Store

## Dynamic Resource Fetching Strategy

### AMI Retrieval
Use SSM Parameter Store to fetch the latest Amazon Linux 2 AMI:
```yaml
!Sub 'arn:${AWS::Partition}:ssm:${AWS::Region}/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2'
```

### Availability Zone Selection
Use region-specific AZs dynamically:
```yaml
!Select [0, !GetAZs '']
```

## Parameter Design

### Required Parameters
1. **KeyName**: EC2 key pair name (default: 'default-key')
2. **SSHLocation**: CIDR block for SSH access (default: '0.0.0.0/0')
3. **HTTPLocation**: CIDR block for HTTP access (default: '0.0.0.0/0')

### Parameter Validation
- KeyName: AllowedPattern for valid key pair names
- SSHLocation/HTTPLocation: AllowedPattern for valid CIDR notation

## Security Best Practices

### Security Group Design
- Minimal necessary ports exposed (22 and 80)
- Default deny all inbound traffic
- Default allow all outbound traffic

### IAM Considerations
- Optional instance profile with least privilege
- Use AWS managed policies where possible
- Avoid hardcoded credentials

## Template Structure

### Logical Resource Order
1. VPC
2. Internet Gateway
3. VPC Gateway Attachment
4. Subnet
5. Route Table
6. Route
7. Subnet Route Table Association
8. Security Group
9. IAM Role (if needed)
10. EC2 Instance

### Metadata and Documentation
- Resource descriptions for clarity
- Parameter descriptions with examples
- Logical naming conventions

## Error Prevention Measures

### Linting Compliance
- All required properties included
- No hardcoded values for AMIs or AZs
- Proper dependency chains established
- Valid parameter patterns enforced

### Validation Steps
- Template syntax validation
- Resource dependency verification
- Parameter default value testing
- Security group rule validation

This architecture ensures a robust, deployable CloudFormation template that follows AWS best practices and will pass cfn-lint validation while providing a functional EC2 instance with appropriate network security.
