# AWS CloudFormation Deployment Plan for MySQL RDS with Networking Infrastructure

## Key AWS Services Required

1. **AWS::EC2::VPC** - Virtual Private Cloud for network isolation
2. **AWS::EC2::InternetGateway** - External connectivity for public subnets
3. **AWS::EC2::VPCGatewayAttachment** - Attach IGW to VPC
4. **AWS::EC2::Subnet** (x2) - Public subnets across two AZs
5. **AWS::EC2::RouteTable** - Routing table for public subnets
6. **AWS::EC2::Route** - Default route to IGW
7. **AWS::EC2::SubnetRouteTableAssociation** (x2) - Associate subnets with route table
8. **AWS::EC2::SecurityGroup** - Security group for RDS instance
9. **AWS::RDS::DBSubnetGroup** - Subnet group for RDS deployment
10. **AWS::RDS::DBInstance** - MySQL RDS instance
11. **AWS::RDS::DBSecurityGroup** - Database security group (if needed for backward compatibility)

## Properties and Dependencies

### Critical Dependencies
1. VPC must be created before Internet Gateway attachment
2. Internet Gateway must be attached to VPC before creating routes
3. Subnets must exist before creating DBSubnetGroup
4. SecurityGroup must exist before referencing it in DBInstance
5. DBSubnetGroup must exist before creating DBInstance

### Property Dependencies
- RouteTable must reference existing VPC
- SubnetRouteTableAssociation must reference existing Subnet and RouteTable
- DBInstance must reference existing DBSubnetGroup and SecurityGroup

## Safe AWS Defaults

### Network Configuration
- **VPC CIDR**: `10.0.0.0/16` (standard private range)
- **Public Subnet CIDRs**: 
  - AZ1: `10.0.1.0/24`
  - AZ2: `10.0.2.0/24`
- **Availability Zones**: Dynamically fetched using `Fn::GetAZs` and `Fn::Select`

### RDS Configuration
- **DB Instance Class**: `db.t3.micro` (as specified)
- **Engine**: `mysql`
- **Engine Version**: Latest stable MySQL (will use default)
- **Master Username**: `admin` (safe default)
- **Master Password**: Dynamic reference to AWS Secrets Manager (secure approach)
- **Storage**: 20GB (safe default for testing)
- **Backup Retention**: 7 days (standard automated snapshots)
- **Multi-AZ**: False (cost-effective for single instance)

## Dynamic Resource Fetching Strategy

### AMI Handling
- Not required for this template (no EC2 instances)

### Availability Zones
```yaml
Fn::GetAZs: !Ref 'AWS::Region'
```
Then use `Fn::Select` to pick first two AZs:
```yaml
!Select [0, !GetAZs !Ref 'AWS::Region']
!Select [1, !GetAZs !Ref 'AWS::Region']
```

## Security Considerations

1. **RDS Security Group**: Allow traffic only on port 3306 from specific sources
2. **Password Management**: Use AWS Secrets Manager dynamic reference instead of plain text
3. **VPC Security**: Public subnets should be minimized; consider private subnets for production
4. **IAM Permissions**: Template should have least-privilege permissions

## Template Structure Plan

1. **Parameters Section**: All parameters with safe defaults
2. **Resources Section**: Ordered by dependencies
3. **Outputs Section**: Key resource identifiers for downstream use

## Key Implementation Details

### Parameter Strategy
- All parameters will have `Default` values
- Allowed patterns will match defaults
- Example: CIDR parameters with default `10.0.0.0/16`

### Security Group Configuration
- Ingress rule for MySQL (port 3306)
- Source will be configurable parameter with safe default
- Egress rule allowing all outbound traffic

### RDS Configuration
- Automated backups enabled (7-day retention)
- Storage encryption enabled (default AWS-managed CMK)
- Performance insights disabled (cost optimization)
- Deletion protection enabled for production safety

### Networking Best Practices
- Public subnets will have direct internet access
- Route table will direct 0.0.0.0/0 to IGW
- Subnets will be tagged for identification

This plan ensures a production-ready, secure, and maintainable CloudFormation template that follows AWS best practices and will pass cfn-lint validation.
