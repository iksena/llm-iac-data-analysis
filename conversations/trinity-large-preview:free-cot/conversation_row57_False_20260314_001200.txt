# ECS Cluster with Configurable Monitoring - Architecture Plan

## Key AWS Services Required

1. **VPC** - Network isolation for the ECS cluster
2. **Internet Gateway** - Internet connectivity for public subnets
3. **Subnets** - Public and private subnets across multiple AZs
4. **Route Tables** - Routing configuration for public and private subnets
5. **Security Groups** - Network access control for ECS tasks and ALB
6. **IAM Roles & Policies** - Service-to-service permissions for ECS, EC2, and CloudWatch
7. **Auto Scaling Group** - Managed EC2 instances for the ECS cluster
8. **Launch Configuration** - EC2 instance configuration for the ASG
9. **Elastic Load Balancer (ALB)** - Application load balancer for service exposure
10. **ECS Cluster** - The core container orchestration service
11. **CloudWatch Alarms & Dashboards** - Monitoring and alerting
12. **SSM Parameter Store** - Dynamic AMI fetching

## Properties and Dependencies

### Network Dependencies
- VPC must be created before Subnets
- Subnets must exist before Route Tables can reference them
- Internet Gateway must be attached to VPC before Route Tables can use it
- Security Groups must reference VPC ID

### IAM Dependencies
- IAM roles must be created before they can be referenced by ECS instances or services
- IAM policies must be attached to roles before use

### ECS Dependencies
- ECS cluster must exist before services can be registered
- Auto Scaling Group must be created before ECS instances can join the cluster

## Safe AWS Defaults (for missing properties)

### VPC Configuration
- **CIDR Block**: `10.0.0.0/16` (standard private network)
- **Availability Zones**: Dynamically fetched using `Fn::GetAZs` and `Fn::Select`
- **Public Subnets**: `10.0.1.0/24`, `10.0.2.0/24`, `10.0.3.0/24`
- **Private Subnets**: `10.0.101.0/24`, `10.0.102.0/24`, `10.0.103.0/24`

### EC2 Configuration
- **Instance Type**: `t3.medium` (balanced performance for ECS)
- **AMI**: Fetched dynamically using SSM Parameter Store
- **KeyPair**: Default parameter with 'default-key' as default value
- **Volume Size**: 50GB (standard for ECS workloads)

### Security Groups
- **SSH Access**: Restricted to specific CIDR blocks (default: `0.0.0.0/0` for demo, but should be customized)
- **HTTP/HTTPS Access**: Load balancer open to internet, ECS instances restricted to ALB only

### IAM Configuration
- **ECS Instance Role**: `AmazonEC2ContainerServiceforEC2Role` (legacy but still valid)
- **ECS Autoscale Role**: `AWSApplicationAutoscalingECSServicePolicy`
- **Task Execution Role**: `AmazonEC2ContainerServiceforEC2Role`

## Dynamic Resource Fetching Strategy

### AMI Retrieval
```yaml
AMIId:
  Fn::ImportValue:
    Fn::Sub: "aws-service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2"
```

### Availability Zones
```yaml
Fn::Select:
  - 0
  - Fn::GetAZs:
      Ref: AWS::Region
```

## Monitoring Configuration

### CloudWatch Alarms
- **CPU Utilization**: Alarm if > 80% for 5 minutes
- **Memory Utilization**: Alarm if > 90% for 5 minutes
- **Service Unhealthy**: Alarm if ECS service reports unhealthy
- **Container Instance Count**: Alarm if instances drop below threshold

### CloudWatch Dashboard
- ECS cluster metrics (CPU, memory, services)
- EC2 instance metrics (CPU, network, disk)
- ALB metrics (request count, latency, error rates)

## Security Best Practices

1. **Principle of Least Privilege**: IAM roles limited to necessary permissions
2. **Network Segmentation**: Public vs private subnets
3. **Encryption**: EBS volumes encrypted by default
4. **Logging**: All ECS logs forwarded to CloudWatch Logs
5. **Patch Management**: ECS-optimized AMIs with automatic updates

## Template Structure Plan

1. **Parameters Section**: Configurable options with safe defaults
2. **Mappings Section**: AMI mappings and other static configurations
3. **Resources Section**: All AWS resources in dependency order
4. **Outputs Section**: Key resource identifiers for downstream use

## Key Implementation Details

### ECS Cluster Configuration
- **Capacity Provider**: EC2 with managed scaling
- **Logging**: AWS CloudWatch with retention policies
- **Task IAM Role**: Separate role for task permissions

### Auto Scaling Configuration
- **Min Instances**: 2 (for high availability)
- **Max Instances**: 10 (configurable)
- **Desired Capacity**: 2 (configurable)
- **Health Check**: EC2 and ECS cluster health checks

### Load Balancer Configuration
- **Scheme**: Internet-facing
- **Listener**: HTTP:80 and HTTPS:443 (HTTPS requires certificate)
- **Target Group**: ECS service integration

This architecture provides a production-ready ECS foundation with comprehensive monitoring, security, and scalability features while adhering to AWS best practices and CloudFormation linting rules.
