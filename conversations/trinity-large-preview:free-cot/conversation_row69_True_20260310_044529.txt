### DynamoDB Table Dependencies:
- DynamoDB Table requires a unique name (using Fn::Join or Fn::Sub with stack name)
- DynamoDB Table requires a primary key schema
- DynamoDB Table requires billing mode (default: PAY_PER_REQUEST)

- **UpdateReplacePolicy: Retain** - Prevents replacement during updates

- Security Group will have no inbound rules by default (principle of least privilege)
- S3 Bucket will have versioning enabled for object recovery
- All resources will have appropriate Tags for cost allocation and identification
