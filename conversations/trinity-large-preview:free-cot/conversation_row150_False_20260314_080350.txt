# Detailed Architectural Deployment Plan for Redis Caching Solution

## Key AWS Services Required

1. **VPC** - Virtual Private Cloud for network isolation
2. **EC2** - Compute instance for application hosting
3. **ElastiCache** - Managed Redis cluster for caching
4. **IAM** - Identity and Access Management for security
5. **SSM** - Systems Manager for EC2 instance management
6. **Security Groups** - Network security controls
7. **Subnets** - Network segmentation (public and private)
8. **Internet Gateway** - Internet connectivity for public subnet
9. **Route Tables** - Network traffic routing

## Architecture Overview

The solution will deploy a VPC with two subnets: one public for the EC2 instance and one private for the ElastiCache cluster. The EC2 instance will have an IAM role with SSM permissions for secure remote access without SSH keys.

## Properties and Dependencies

### VPC Dependencies
- VPC must be created before subnets
- Internet Gateway must be attached to VPC before route table associations
- Route table must have Internet Gateway route before EC2 instance can access internet

### EC2 Dependencies
- IAM role must exist before EC2 instance creation
- Security group must exist before EC2 instance creation
- Subnet must exist before EC2 instance creation

### ElastiCache Dependencies
- VPC must exist before subnet group creation
- Subnet group must exist before cluster creation
- Security group must exist before cluster creation

## Safe AWS Defaults

### Network Configuration
- **VPC CIDR**: 10.0.0.0/16 (standard private range)
- **Public Subnet CIDR**: 10.0.1.0/24
- **Private Subnet CIDR**: 10.0.2.0/24
- **Availability Zones**: Dynamically selected using `Fn::GetAZs` and `Fn::Select`

### EC2 Configuration
- **Instance Type**: t3.micro (free tier eligible)
- **AMI**: Latest Amazon Linux 2 using SSM Parameter Store
- **KeyPair**: 'default-key' (safe default parameter)

### ElastiCache Configuration
- **Node Type**: cache.t3.micro (cost-effective)
- **Engine**: redis
- **Port**: 6379 (default Redis port)
- **NumCacheNodes**: 1 (minimal deployment)

## Dynamic Resource Fetching Strategy

### AMI Retrieval
Use SSM Parameter Store to fetch the latest Amazon Linux 2 AMI:
```
'/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2'
```

### Availability Zones
Use `Fn::GetAZs` to get all AZs in the region, then `Fn::Select` to pick the first one:
```
Fn::Select: [0, Fn::GetAZs: !Ref 'AWS::Region']
```

## Security Considerations

### IAM Role for EC2
Create an IAM role with the following policies:
- AmazonSSMManagedInstanceCore (for SSM access)
- Custom policy for ElastiCache access if needed

### Security Groups
- **EC2 Security Group**: Allow inbound HTTP/HTTPS from anywhere, SSH from SSM
- **ElastiCache Security Group**: Allow inbound from EC2 security group only

### Network ACLs
Use VPC's default NACL or create custom rules allowing necessary traffic

## SSM Integration Strategy

The EC2 instance will be configured with:
1. IAM role allowing SSM access
2. SSM agent pre-installed on Amazon Linux 2 AMI
3. No SSH key pair required (SSM provides access)

## Template Structure Plan

1. **Parameters**: Define all configurable options with safe defaults
2. **Resources**: Create VPC, subnets, security groups, IAM role, EC2, ElastiCache
3. **Outputs**: Export important resource IDs for reference

## Key Implementation Details

### VPC Configuration
- Enable DNS hostnames and DNS support
- Create public and private subnets in different AZs
- Configure route table with Internet Gateway for public subnet

### EC2 Instance
- Use latest Amazon Linux 2 AMI
- Associate public IP address
- Install SSM agent (pre-installed on Amazon Linux 2)
- Configure user data for any additional setup

### ElastiCache Cluster
- Create subnet group using private subnet
- Configure cluster with minimal node count
- Set up security group allowing only EC2 access

### Parameter Validation
- Use AllowedPattern for CIDR blocks
- Provide meaningful default values
- Include description for each parameter

This architecture ensures a secure, minimal Redis caching solution with SSM-enabled EC2 access, following AWS best practices and avoiding common CloudFormation deployment failures.
