## 2. Properties and Dependencies Analysis

For an SQS Standard Queue, the key properties to consider are:

- **QueueName** (optional) - If not provided, AWS generates a unique name
- **DelaySeconds** (optional, default: 0) - Time in seconds that the delivery of all messages in the queue will be delayed
- **MaximumMessageSize** (optional, default: 262144 bytes) - The limit of how many bytes a message can contain
- **MessageRetentionPeriod** (optional, default: 345600 seconds/4 days) - The number of seconds Amazon SQS retains a message
- **ReceiveMessageWaitTimeSeconds** (optional, default: 0) - Enables long polling
- **VisibilityTimeout** (optional, default: 30 seconds) - The length of time during which a message will be unavailable once a message is delivered from the queue
- **RedrivePolicy** (optional) - Configuration for dead-letter queue
- **KmsMasterKeyId** (optional) - ID of the AWS-managed customer master key (CMK) for server-side encryption
- **KmsDataKeyReusePeriodSeconds** (optional, default: 300) - Length of time in seconds for which Amazon SQS can reuse a data key
