# EventBridge API Destination Deployment Plan

## Key AWS Services Required

1. **Amazon EventBridge** - Core service for event routing
2. **AWS Secrets Manager** - Secure storage for API key
3. **AWS Systems Manager (SSM) Parameter Store** - Dynamic resource fetching (if needed for future extensibility)
4. **AWS Identity and Access Management (IAM)** - Service-linked roles for EventBridge

## Architecture Overview

The deployment will create:
- An EventBridge custom event bus
- An API destination pointing to the external HTTPS endpoint
- A secure connection using API key authentication
- A service-linked role for EventBridge to manage API calls

## Properties and Dependencies

### EventBridge Event Bus
- **Name**: `CustomEventBus` (default)
- **EventPattern**: Optional, can be configured to filter specific events
- **Description**: "Custom event bus for routing to external API"

### API Destination
- **Name**: `WebhookAPIDestination`
- **ConnectionName**: References the API connection resource
- **InvocationEndpoint**: `https://webhook.site/testtest`
- **HttpMethod**: `POST` (default for API destinations)
- **InvocationRateLimitPerSecond**: `1` (default, can be adjusted)

### API Connection
- **Name**: `WebhookConnection`
- **Description**: "Connection to webhook.site with API key authentication"
- **AuthorizationType**: `API_KEY`
- **ApiKeyName**: `x-api-key` (default header name)
- **ApiKey**: References Secrets Manager secret
- **InvocationEndpoint**: `https://webhook.site/testtest`

### Secrets Manager
- **Name**: `WebhookAPIKey`
- **SecretString**: Contains the actual API key value
- **Description**: "API key for webhook.site authentication"

## Safe Defaults and Dynamic Resource Handling

### Missing Properties - Safe Defaults
- **Event Bus Name**: `CustomEventBus`
- **Invocation Rate Limit**: `1` request per second
- **HTTP Method**: `POST`
- **API Key Header Name**: `x-api-key`
- **Connection Description**: "API connection for webhook.site"

### Dynamic Resource Fetching
While not strictly necessary for this deployment, I'll use SSM Parameter Store patterns for future extensibility:
- No AMIs needed (no EC2 instances)
- No AZ selection needed (no VPC resources)
- All resources are global/EventBridge-specific

## Security and Best Practices Implementation

### API Key Management
- Store the API key in Secrets Manager, not as plain text in CloudFormation
- Use dynamic references to fetch the secret at runtime
- Implement least privilege by only granting EventBridge access to the specific secret

### IAM Configuration
- Create a service-linked role for EventBridge API destinations
- Grant permissions to invoke the API destination
- Ensure the role has proper trust policy for EventBridge service

### Resource Protection
- Set `DeletionPolicy: Retain` on the API connection and destination
- Include corresponding `UpdateReplacePolicy: Retain` for stateful resources
- This prevents accidental deletion of configuration

## Dependencies and Resource Ordering

1. **Secrets Manager Secret** → Created first to store API key
2. **API Connection** → Depends on the secret being available
3. **API Destination** → Depends on the connection being created
4. **Event Bus** → Created last, can be created independently

## Error Prevention and Linting Compliance

### Parameter Handling
- No Parameters required for this template (all values are defaults or references)
- If Parameters were needed, they would include safe defaults matching AllowedPattern constraints

### Property Validation
- All properties align with current CloudFormation schemas
- No deprecated properties or runtimes used
- No hardcoded values that could cause deployment failures
