# EC2 Instance with Ephemeral Drives Deployment Plan

## Key AWS Services Required
1. **EC2** - For the virtual server instance
2. **SSM Parameter Store** - To dynamically fetch the latest Amazon Linux 2 AMI
3. **EC2 Key Pairs** - For SSH access
4. **VPC** - Pre-existing default VPC (no creation needed)
5. **Subnet** - Pre-existing default subnet (no creation needed)
6. **IAM** - For instance profile if needed (not required for basic EC2)

## Architecture Analysis

### Resource Dependencies
1. **VPC/Subnet**: Since we're using pre-existing default VPC/subnet, no creation dependencies exist
2. **AMI**: Must be fetched before EC2 creation using SSM Parameter Store
3. **Key Pair**: Must be created before EC2 instance creation
4. **EC2 Instance**: Depends on AMI, Key Pair, VPC, and Subnet

### Missing Properties & Safe Defaults

**Instance Type**: Not specified - will use `t3.micro` (free tier eligible, safe default)

**CIDR Blocks**: Not needed since using default VPC

**Availability Zone**: Not specified - will use `Fn::Select` with `Fn::GetAZs` to dynamically select first AZ

**Ephemeral Drives**: Not specified - will use standard ephemeral drive configuration:
- `/dev/sdb` (8GB)
- `/dev/sdc` (8GB)

## Detailed Implementation Plan

### 1. Parameters Section
- **KeyName**: Parameter with `Default: default-key` (safe default for automated deployment)
- **InstanceType**: Parameter with `Default: t3.micro` (free tier eligible)

### 2. Resources Section

#### EC2 Key Pair Creation
- Use `AWS::EC2::KeyPair` resource
- Generate key material using `Fn::Base64` with random string
- Export public key for SSH access

#### AMI Fetching Strategy
- Use `Fn::ImportValue` with SSM Parameter Store path: `/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2`
- This ensures always using the latest Amazon Linux 2 AMI

#### EC2 Instance Configuration
- **ImageId**: Dynamic AMI from SSM Parameter Store
- **InstanceType**: From parameter with safe default
- **KeyName**: From parameter with safe default
- **BlockDeviceMappings**: Configure ephemeral drives
- **NetworkProperties**: Use default VPC/subnet references
- **Subnet**: Use `Fn::Select` with `Fn::GetAZs` for dynamic AZ selection

#### Ephemeral Drive Configuration
- Use `BlockDeviceMappings` with `VirtualName: ephemeral0` and `ephemeral1`
- These map to `/dev/sdb` and `/dev/sdc` respectively
- No `Ebs` configuration needed for ephemeral drives

### 3. Security Considerations
- No Security Groups specified - will use default VPC's default security group
- SSH access will be available through default security group rules
- Key pair provides secure authentication

### 4. Output Section
- Export instance ID for reference
- Export public IP for SSH access
- Export key pair name for documentation

### 5. Error Prevention Strategies
- All parameters have safe defaults
- No hardcoded AMIs or AZs
- Uses only supported properties per CloudFormation schema
- No deprecated runtimes or features

This architecture ensures the template will deploy successfully without manual intervention, uses current best practices, and provides a functional EC2 instance with ephemeral storage ready for SSH access.
