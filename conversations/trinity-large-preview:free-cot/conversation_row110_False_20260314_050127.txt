## Architecture Plan

- Configure VPC link with specified VPC endpoint ID
- Create route with Lambda integration
- Deploy stage with custom domain mapping

### 3. IAM Roles and Policies
- Create Lambda execution role with CloudWatch logging permissions
- Create API Gateway invocation policy for Lambda access

### 4. Custom Domain Setup
- Create custom domain name with ACM certificate
- Map custom domain to API Gateway stage

### 5. Cross-Account Considerations
- Ensure proper IAM trust relationships for cross-account access
- Configure VPC endpoint access control for private API Gateway

## Security Considerations

1. **VPC Endpoint Access Control**: Configure API Gateway to allow access only from specified VPC endpoint ID
2. **IAM Permissions**: Use least privilege principle for IAM roles and policies
3. **Custom Domain Security**: Use ACM certificate for custom domain with HTTPS
4. **Lambda Security**: Use latest runtime and minimal permissions

## Error Handling and Validation

1. **Parameter Validation**: Ensure all parameters have default values and proper patterns
2. **Resource Dependencies**: Use proper DependsOn declarations for resource creation order
3. **Error Handling**: Implement proper error handling in Lambda function

## Deployment Strategy

1. **Sequential Resource Creation**: Create resources in proper order to satisfy dependencies
2. **Cross-Account Permissions**: Ensure proper IAM trust relationships for cross-account access
3. **VPC Endpoint Configuration**: Verify VPC endpoint ID is valid and accessible
4. **Custom Domain Validation**: Ensure custom domain and certificate are properly configured

This architecture plan ensures a secure, production-ready deployment of a cross-account private API Gateway with Lambda integration, following AWS best practices and CloudFormation linting rules.
