# AWS EventBridge Pipes Workflow Architecture Plan

## Key AWS Services Required
1. **Amazon SQS** - Event source queue
2. **Amazon EventBridge Pipes** - Workflow orchestration
3. **Amazon CloudWatch Logs** - Target for enriched logs
4. **AWS IAM** - Permissions management
5. **AWS Systems Manager (SSM) Parameter Store** - Dynamic resource fetching (not needed for this specific template but noted for best practices)

## Architecture Overview
The EventBridge Pipe will create an event-driven workflow that:
1. Consumes messages from SQS
2. Extracts zip code from message body
3. Calls the external API (https://api.zippopotam.us/us/*) with the zip code
4. Enriches the original message with API response data
5. Logs the enriched message to CloudWatch Logs

## Resource Properties & Dependencies

### SQS Queue
- **QueueName**: `EventBridgePipeSourceQueue` (default)
- **VisibilityTimeout**: 300 seconds (5 minutes)
- **MessageRetentionPeriod**: 345600 seconds (4 days)
- **MaximumMessageSize**: 262144 bytes (256 KB)

### EventBridge Pipe
- **Name**: `ZipDataEnrichmentPipe` (default)
- **Source**: SQS queue ARN
- **Target**: CloudWatch Log Group ARN
- **Enrichment**: API Destination with dynamic path parameter

### API Destination
- **Name**: `ZipCodeAPI` (default)
- **Endpoint**: `https://api.zippopotam.us/us/`
- **HttpMethod**: `GET`
- **InvocationRateLimitPerSecond**: 1 (default)

### CloudWatch Log Group
- **LogGroupName**: `/aws/eventbridge/pipes/zip-enrichment` (default)
- **RetentionInDays**: 30 days (default)

### IAM Role
- **RoleName**: `EventBridgePipeExecutionRole` (default)
- **Trust Policy**: EventBridge Pipes service
- **Permissions**:
  - SQS: `sqs:ReceiveMessage`, `sqs:DeleteMessage`
  - CloudWatch Logs: `logs:CreateLogStream`, `logs:PutLogEvents`
  - X-Ray: `xray:PutTraceSegments` (for monitoring)

## Security & Permissions Strategy

### IAM Role Permissions
The IAM role will have the following managed policies:
- `AWSXRayDaemonWriteAccess` (for tracing)
- Custom inline policy for SQS and CloudWatch Logs access

### API Destination Security
The API Destination will use HTTPS endpoint with no authentication (public API).

## Message Enrichment Logic

The enrichment process will:
1. Parse the SQS message body (expected JSON format)
2. Extract the `zip_code` field
3. Construct API URL: `https://api.zippopotam.us/us/{zip_code}`
4. Make HTTP GET request to the API
5. Merge API response with original message
6. Forward enriched message to CloudWatch Logs

## Error Handling & Retry Strategy

### EventBridge Pipe Configuration
- **MaximumRetryAttempts**: 3
- **MaximumEventAgeInSeconds**: 86400 (24 hours)
- **OnPartialBatchItemFailure**: Enabled

### Dead Letter Queue
A DLQ will be created for messages that fail processing after retries.

## Template Structure Plan

```yaml
AWSTemplateFormatVersion: '2010-09-09'
Description: EventBridge Pipes workflow for SQS message enrichment with external API data

Parameters:
  # Safe defaults for all parameters
  SourceQueueName:
    Type: String
    Default: EventBridgePipeSourceQueue
    Description: Name of the source SQS queue

Resources:
  # SQS Queue
  SourceQueue:
    Type: AWS::SQS::Queue
    Properties:
      QueueName: !Ref SourceQueueName
      # Other properties as defined above

  # CloudWatch Log Group
  LogGroup:
    Type: AWS::Logs::LogGroup
    Properties:
      LogGroupName: !Sub '/aws/eventbridge/pipes/zip-enrichment'
      RetentionInDays: 30

  # API Destination
  ZipCodeAPI:
    Type: AWS::Events::ApiDestination
    Properties:
      Name: ZipCodeAPI
      ConnectionArn: !GetAtt ApiConnection.Arn
      InvocationEndpoint: 'https://api.zippopotam.us/us/'
      HttpMethod: GET

  # API Connection
  ApiConnection:
    Type: AWS::Events::Connection
    Properties:
      AuthorizationType: HTTP
      Name: ZipCodeAPIAuth

  # IAM Role
  ExecutionRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: EventBridgePipeExecutionRole
      AssumeRolePolicyDocument:
        Version: '2012-10-17'
        Statement:
          - Effect: Allow
            Principal:
              Service: events.amazonaws.com
            Action: sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/AWSXRayDaemonWriteAccess
      Policies:
        - PolicyName: SQSAndLogsAccess
          PolicyDocument:
            Version: '2012-10-17'
            Statement:
              - Effect: Allow
                Action:
                  - sqs:ReceiveMessage
                  - sqs:DeleteMessage
                Resource: !GetAtt SourceQueue.Arn
              - Effect: Allow
                Action:
                  - logs:CreateLogStream
                  - logs:PutLogEvents
                Resource: !GetAtt LogGroup.Arn

  # EventBridge Pipe
  ZipDataEnrichmentPipe:
    Type: AWS::Events::Pipe
    Properties:
      Name: ZipDataEnrichmentPipe
      Source:
        SourceType: SQS
        SourceParameters:
          QueueARN: !GetAtt SourceQueue.Arn
      Target:
        TargetType: AWS::Logs::LogGroup
        TargetParameters:
          InputTemplate: !Sub |
            {
              "message": <enriched_message_logic>
            }
        Endpoint: !GetAtt LogGroup.Arn
      Enrichment:
        Type: ApiDestination
        Parameters:
          ApiDestinationArn: !GetAtt ZipCodeAPI.Arn
          InvocationParameters:
            PathParameterValues:
              - !Select [0, !Split ["/", !GetAtt SourceQueue.Arn]]
      RoleArn: !GetAtt ExecutionRole.Arn
      MaximumRetryAttempts: 3
      MaximumEventAgeInSeconds: 86400
```
