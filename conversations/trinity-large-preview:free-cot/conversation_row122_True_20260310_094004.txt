# Detailed Architectural Deployment Plan for EventBridge Shopify Integration

## Key AWS Services Required

1. **AWS::Events::EventBus** - Custom event bus to capture events from `MyShopifyApp` source
2. **AWS::Events::ApiDestination** - API destination to target Shopify's product creation endpoint
3. **AWS::IAM::Role** - IAM role for EventBridge to assume when calling Shopify API
4. **AWS::SSM::Parameter** - To store Shopify API key securely (optional, depending on implementation)
5. **AWS::SecretsManager::Secret** - Alternative secure storage for Shopify API key (optional)

## Properties and Dependencies

1. **EventBus** - No dependencies, can be created standalone
2. **ApiDestination** - Depends on EventBus and IAM Role being created first
3. **IAM Role** - No dependencies, but must have proper trust policy for EventBridge
4. **SSM Parameter/Secrets Manager** - Optional, depends on security requirements

## Safe AWS Defaults (where values are missing)

1. **Shopify API Key Parameter**: Default to a dummy value like "dummy-api-key" with `AllowedPattern: "[a-zA-Z0-9-]+"`
2. **Shopify Shop Name**: Default to "testanu" (as specified in business need)
3. **Event Source Name**: Default to "MyShopifyApp" (as specified)
4. **Event Pattern**: Default to a basic pattern matching any event from the source
5. **Shopify Endpoint**: Default to "https://testanu.myshopify.com/admin/api/2024-01/products.json" (using latest stable API version)

## Dynamic Resource Fetching Strategy

1. **AMIs**: Not required for this template as no EC2 instances are needed
2. **API Versions**: Use latest stable Shopify API version (2024-01) as default

## Security Considerations

1. **API Key Storage**: Use AWS Secrets Manager to store Shopify API key securely
2. **IAM Role**: Create least-privilege IAM role for EventBridge to call Shopify API
3. **Network Configuration**: No VPC required as this is API-based integration

## Architecture Plan

### 1. Parameters Section
- `ShopifyApiKey`: String parameter with dummy default, `AllowedPattern: "[a-zA-Z0-9-]+"`
- `ShopifyShopName`: String parameter with default "testanu"
- `EventSourceName`: String parameter with default "MyShopifyApp"
- `ShopifyApiVersion`: String parameter with default "2024-01"

### 2. Resources Section

#### a. IAM Role for EventBridge
- Trust policy allowing events.amazonaws.com to assume role
- Permission to make HTTPS requests to Shopify endpoint
- Optional: Policy to read Shopify API key from Secrets Manager

#### b. Custom Event Bus
- Name: "ShopifyIntegrationBus"
- Event source: Parameterized (default "MyShopifyApp")
- Event pattern: Match events from specified source

#### c. API Destination
- Name: "ShopifyProductCreation"
- Connection: HTTPS connection to Shopify endpoint
- Invocation endpoint: Shopify product creation URL
- HTTP method: POST
- Invocation rate limit: Configurable (default 1 request per second)
- Authentication: API key in header

#### d. Optional: Secrets Manager Setup
- Secret to store Shopify API key
- Policy to allow EventBridge role to read the secret

### 3. Event Pattern Design

The event pattern will match events from the specified source:
```json
{
  "source": ["MyShopifyApp"]
}
```

### 4. API Destination Configuration

- **Endpoint**: `https://testanu.myshopify.com/admin/api/2024-01/products.json`
- **Method**: POST
- **Headers**: Include Shopify API key as `X-Shopify-Access-Token`
- **Rate Limiting**: Configurable to prevent exceeding Shopify API limits

### 5. Error Handling Strategy

1. **Event Replay**: Enable event replay on the custom event bus
2. **Dead Letter Queue**: Optional, for failed API calls
3. **CloudWatch Alarms**: Monitor API destination invocation failures

### 6. Deployment Sequence

1. Create IAM Role
2. Create Custom Event Bus
3. Create API Destination
4. (Optional) Create Secrets Manager secret and update API destination

### 7. Testing Strategy

1. **Unit Testing**: Use AWS CLI to create test events and verify API calls
2. **Integration Testing**: Deploy to test account and verify end-to-end flow
3. **Performance Testing**: Test with high event volume to validate rate limiting

### 8. Cost Optimization

1. **Event Bus**: Custom event buses have no additional cost
2. **API Destination**: Pay-per-use pricing, no provisioned capacity needed
3. **Secrets Manager**: Minimal cost for storing API key

### 9. Monitoring and Logging

1. **CloudWatch Logs**: Enable logging for API destination invocations
2. **CloudWatch Metrics**: Monitor invocation success/failure rates
3. **CloudWatch Alarms**: Alert on high failure rates or latency

This architecture provides a secure, scalable, and maintainable EventBridge integration with Shopify, following AWS best practices and ensuring the template will pass cfn-lint validation.
