- Use AWS managed policy AWSLambdaBasicExecutionRole
- Minimal permissions principle - only grant what's necessary
- No hardcoded credentials or secrets required

**Lambda Function:**
- Use latest Node.js runtime (nodejs20.x)
- No VPC configuration needed for simple function
- No environment variables required for hello world

1. **Template Structure:**
   - Parameters section: None required (no user inputs needed)
   - Resources section: Lambda function and IAM role
   - Outputs section: Lambda function ARN for reference

2. **Error Prevention:**
   - No hardcoded values that could become stale
   - No external dependencies that could fail
   - Minimal configuration reduces failure points

3. **Linting Compliance:**
   - All required properties included
   - No deprecated runtimes or properties
   - Proper IAM policy structure

## Final Architecture Decision

This is a minimal viable deployment with:
- Single Lambda function with inline code
- Associated IAM execution role
- No external dependencies
- Safe default configurations
- Full compliance with AWS best practices
