1. User makes request to custom domain

6. Response flows back through API Gateway to user

### Security Model:
- Private API Gateway accessible only via specified VPC endpoint
- Lambda executes with minimal IAM permissions
- Custom domain provides SSL/TLS encryption
- VPC isolation for backend resources

- Code: Simple response function
- VPC Configuration: Optional (can be public Lambda)

### 3. API Gateway
- Type: REST API
- Endpoint Configuration: PRIVATE
- VPC Endpoint IDs: Parameter input
- Deployment: With stage
- Integration: Lambda proxy integration

### 4. Custom Domain
- Domain Name: Parameter input
- Certificate: ACM certificate ARN parameter
- Base Path: Optional (default empty)

### 5. API Mapping
- Stage: prod
- Domain Name: Custom domain
- API: Private API Gateway

## 7. Parameter Design

### Required Parameters:
- `VpcEndpointId`: No default (must be provided)
- `DomainName`: No default (must be provided)
- `CertificateArn`: No default (must be provided)
- `StageName`: Default `prod`

### Optional Parameters:
- `LambdaMemory`: Default `128`
- `LambdaTimeout`: Default `3`
- `LambdaRuntime`: Default `python3.12`

## 8. Security Considerations

- Principle of least privilege for Lambda role
- Resource-level permissions for API Gateway
- No wildcard permissions in IAM policies

### Network Security:
- Private API Gateway with VPC endpoint restriction
- Optional VPC configuration for Lambda
- No public accessibility

### Data Protection:
- SSL/TLS via custom domain and ACM certificate
- No sensitive data in plain text parameters

- Resource dependencies are properly ordered

### Runtime Validation:
- Lambda function includes error handling
- API Gateway has proper integration responses

- Use resource-based policies for cross-account access

- Lambda function versioning for safe deployments
