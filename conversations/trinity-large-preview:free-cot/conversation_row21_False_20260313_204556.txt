# Amazon Timestream Database Architecture Deployment Plan

## Key AWS Services Required

1. **AWS::Timestream::Database** - Primary database resource
2. **AWS::Timestream::Table** - Four tables with different retention policies
3. **AWS::IAM::Role** - Service role for Timestream database (if needed)

## Architecture Analysis

### Database Configuration
- **Database Name**: `poc-database` (default)
- **Table Names**: 
  - `poc-table-01-batch-write`
  - `poc-table-02-older-records-memonly`
  - `poc-table-03-older-records-magnetic`
  - `poc-table-04-common-attributes`

### Retention Policies
Based on the table names, I'll configure different retention strategies:
- **poc-table-01-batch-write**: Standard (memory + magnetic storage)
- **poc-table-02-older-records-memonly**: Memory-only (no magnetic storage)
- **poc-table-03-older-records-magnetic**: Magnetic-only (no memory storage)
- **poc-table-04-common-attributes**: Standard with common attributes

## Detailed Deployment Plan

### 1. Parameters Definition (with safe defaults)

```yaml
Parameters:
  DatabaseName:
    Type: String
    Default: poc-database
    Description: Name of the Timestream database

  RetentionHours:
    Type: Number
    Default: 24
    Description: Default retention period in hours for tables

  MagneticStoreRetentionDays:
    Type: Number
    Default: 365
    Description: Retention period for magnetic store in days
```

### 2. Database Resource

```yaml
Resources:
  TimestreamDatabase:
    Type: AWS::Timestream::Database
    Properties:
      DatabaseName: !Ref DatabaseName
      KmsKeyId: !Sub "arn:${AWS::Partition}:kms:${AWS::Region}:${AWS::AccountId}:alias/aws/timestream"
```

### 3. Table Resources with Different Retention Policies

```yaml
  TableBatchWrite:
    Type: AWS::Timestream::Table
    Properties:
      DatabaseName: !Ref DatabaseName
      TableName: poc-table-01-batch-write
      RetentionHours: !Ref RetentionHours
      MagneticStoreRetentionDays: !Ref MagneticStoreRetentionDays

  TableOlderRecordsMemOnly:
    Type: AWS::Timestream::Table
    Properties:
      DatabaseName: !Ref DatabaseName
      TableName: poc-table-02-older-records-memonly
      RetentionHours: !Ref RetentionHours
      MagneticStoreRetentionDays: 0  # Memory-only

  TableOlderRecordsMagnetic:
    Type: AWS::Timestream::Table
    Properties:
      DatabaseName: !Ref DatabaseName
      TableName: poc-table-03-older-records-magnetic
      RetentionHours: 0  # Magnetic-only
      MagneticStoreRetentionDays: !Ref MagneticStoreRetentionDays

  TableCommonAttributes:
    Type: AWS::Timestream::Table
    Properties:
      DatabaseName: !Ref DatabaseName
      TableName: poc-table-04-common-attributes
      RetentionHours: !Ref RetentionHours
      MagneticStoreRetentionDays: !Ref MagneticStoreRetentionDays
```

### 4. Error Handling & Best Practices

#### Deletion Policies
```yaml
  TimestreamDatabase:
    Type: AWS::Timestream::Database
    DeletionPolicy: Retain
    UpdateReplacePolicy: Retain
    Properties:
      DatabaseName: !Ref DatabaseName
```

#### IAM Role (if needed for database operations)
```yaml
  TimestreamServiceRole:
    Type: AWS::IAM::Role
    Properties:
      AssumeRolePolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Effect: Allow
            Principal:
              Service: timestream.amazonaws.com
            Action: sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/service-role/AmazonTimestreamFullAccess
```

### 5. Outputs for Reference

```yaml
Outputs:
  DatabaseName:
    Value: !Ref DatabaseName
    Description: Timestream Database Name

  TableNames:
    Value: !Join [",", [!Ref TableBatchWrite, !Ref TableOlderRecordsMemOnly, !Ref TableOlderRecordsMagnetic, !Ref TableCommonAttributes]]
    Description: List of created table names
```

3. **Deletion Policies**: Set to `Retain` to prevent accidental data loss
4. **No Hardcoding**: No hardcoded values for IDs or regions
5. **Error Handling**: Basic structure with safe defaults and retention policies

## Missing Information & Assumptions

1. **KMS Key**: Using default Timestream KMS key
