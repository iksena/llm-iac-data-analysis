# IAM Role for EC2 with Read-Only IAM Access - Architecture Plan

## Key AWS Services Required
1. **AWS::IAM::Role** - The primary resource to create the IAM role for EC2 instances
2. **AWS::IAM::Policy** - To attach the read-only IAM permissions to the role
3. **AWS::IAM::InstanceProfile** (optional) - To create an instance profile for EC2 to assume the role

## Properties and Dependencies
- The IAM Role must have a trust relationship allowing EC2 to assume it
- The IAM Policy must be attached to the role after the role is created
- No external dependencies like VPCs, Internet Gateways, or EC2 instances are required for this IAM-only template

## Safe AWS Defaults
- **Region**: No region specified - IAM is global
- **Role Name**: No explicit name - let CloudFormation generate a unique name
- **Policy Name**: No explicit name - let CloudFormation generate a unique name
- **Path**: Default to "/" (root path) for both role and policy

## Architecture Design

### IAM Role Configuration
The IAM role will be configured with:
- **Service**: ec2.amazonaws.com in the trust policy
- **Permissions**: Read-only access to IAM using AWS managed policy
- **Path**: "/" (root path)
- **Description**: "EC2 role with read-only IAM access"

### IAM Policy Configuration
The IAM policy will use the AWS managed policy:
- **Policy ARN**: arn:aws:iam::aws:policy/IAMReadOnlyAccess
- This provides read-only access to IAM resources including:
  - List roles, policies, users, groups
  - Get role, policy, user, group details
  - List attached policies
  - No modification or deletion permissions

### Instance Profile (Optional)
If an instance profile is needed for EC2 to assume the role:
- **Resource**: AWS::IAM::InstanceProfile
- **Path**: Same as role ("/")
- **Roles**: Reference to the created IAM role

## Security Considerations
- Principle of least privilege: Only read-only IAM access granted
- No hardcoded credentials or secrets
- No external dependencies that could introduce vulnerabilities
- IAM is global service - no regional considerations needed

## Template Structure
The CloudFormation template will include:
1. **Resources section** with:
   - IAM::Role for the EC2 role
   - IAM::Policy to attach read-only permissions
   - IAM::InstanceProfile (if required)
2. **No Parameters section** - no user input required
3. **No Mappings or Conditions** - straightforward deployment

## Dynamic Resource Handling
Not applicable for this IAM-only template as:
- No AMIs to fetch
- No Availability Zones to select
- No external resource IDs to resolve

## Linting Compliance
The template will adhere to all specified rules:
- No Parameters without defaults (template has no Parameters)
- No hardcoded AMIs or AZs
- No legacy S3 properties (not using S3)
- Proper DeletionPolicy/UpdateReplacePolicy pairing if stateful resources added
- No unnecessary Fn::Sub usage
- No hallucinated properties
- Uses current AWS managed policy ARNs

This architecture provides a minimal, secure IAM role that EC2 instances can assume to retrieve and list IAM resources without any modification capabilities.
