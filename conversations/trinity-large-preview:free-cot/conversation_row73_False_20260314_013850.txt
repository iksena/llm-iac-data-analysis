# ECS Cluster with Fargate and Fargate Spot Capacity Providers - Architecture Plan

## Key AWS Services Required

1. **VPC** - Virtual Private Cloud for network isolation
2. **Internet Gateway** - For public internet access
3. **Subnets** - Public and private subnets across multiple AZs
4. **Route Tables** - For routing traffic within the VPC
5. **ECS Cluster** - The main container orchestration service
6. **Capacity Providers** - Fargate and Fargate Spot for compute capacity
7. **IAM Roles** - For cluster and task execution permissions
8. **Security Groups** - For network security controls
9. **AWS SSM Parameter Store** - For dynamic AMI fetching (though not needed for Fargate)

## Properties and Dependencies

1. **VPC Creation** must precede:
   - Subnet creation (Subnets depend on VPC ID)
   - Internet Gateway attachment (IGW must attach to VPC)
   - Route Table creation (Route Tables belong to VPC)

2. **Subnets** must be created before:
   - Route Table associations
   - Security Group creation (Security Groups belong to VPC)

3. **Internet Gateway** must be created and attached before:
   - Route Table routes can reference it
   - Public subnets can route to the internet

4. **IAM Roles** must be created before:
   - ECS Cluster creation (Cluster needs execution role)
   - Capacity Provider creation (Capacity Providers need IAM roles)

## Safe AWS Defaults for Missing Properties

**VPC Configuration:**
- CIDR Block: `10.0.0.0/16` (standard private range)

- Private Subnets: `10.0.3.0/24`, `10.0.4.0/24`, `10.0.5.0/24`

**Availability Zones:**

**AZs:** Use `Fn::GetAZs` to dynamically fetch available AZs in the region

### 3. ECS Cluster Creation

### 4. Security Configuration
- Create security groups for cluster communication
- Configure appropriate ingress/egress rules

### 5. Capacity Provider Configuration

- Configure Fargate Spot capacity provider with appropriate weight and base
