# AWS CloudFormation Architecture Plan for MySQL RDS with Provisioned IOPS

## Key AWS Services Required

1. **AWS::EC2::VPC** - Dedicated VPC for network isolation
2. **AWS::EC2::InternetGateway** - For VPC internet connectivity
3. **AWS::EC2::VPCGatewayAttachment** - Attach IGW to VPC
4. **AWS::EC2::Subnet** (x2) - Two private subnets in different AZs
5. **AWS::EC2::RouteTable** - Route table for private subnets
6. **AWS::EC2::Route** - Default route (if needed)
7. **AWS::EC2::SubnetRouteTableAssociation** (x2) - Associate subnets with route table
8. **AWS::RDS::DBSubnetGroup** - Subnet group for RDS instance
9. **AWS::EC2::SecurityGroup** - Security group for RDS instance
10. **AWS::RDS::DBInstance** - MySQL RDS instance with Provisioned IOPS

## Properties and Dependencies

### VPC Dependencies
- Internet Gateway must be attached to VPC before creating route table entries
- Subnets must be created within the VPC
- Route table must exist before associating subnets

### RDS Dependencies
- DB Subnet Group must reference existing subnets
- Security Group must exist before referencing in DBInstance
- DBInstance depends on DBSubnetGroup and SecurityGroup

## Safe AWS Defaults (where values were not specified)

### VPC Configuration
- **CIDR Block**: `10.0.0.0/16` (standard private range)
- **Subnet CIDRs**: 
  - Subnet1: `10.0.1.0/24` (AZ1)
  - Subnet2: `10.0.2.0/24` (AZ2)

### RDS Configuration
- **DB Instance Class**: `db.r5.large` (as specified)
- **Engine**: `mysql`
- **Engine Version**: `8.0.34` (latest stable)
- **Storage Type**: `io1` (Provisioned IOPS)
- **Allocated Storage**: `100` GB (reasonable default for production)
- **IOPS**: `1000` (1x storage for baseline, can be adjusted)
- **Master Username**: `admin` (safe default)
- **Master Password**: Dynamic reference to Secrets Manager (secure approach)

### Security Group
- **VPC**: References the created VPC
- **Port**: `3306` (MySQL default)
- **Source**: `10.0.0.0/16` (internal VPC traffic only)

## Dynamic Resource Fetching Strategy

### AMI Handling
- Not required for this RDS-only deployment (no EC2 instances)

### Availability Zones
- Use `Fn::GetAZs` to dynamically fetch available AZs for the region
- Use `Fn::Select` to pick first two AZs for subnet distribution

## Architecture Flow

1. **Create VPC** with 10.0.0.0/16 CIDR
2. **Create Internet Gateway** and attach to VPC
3. **Create two private subnets** in different AZs using dynamic AZ selection
4. **Create route table** and associate with both subnets
5. **Create DB Subnet Group** referencing both private subnets
6. **Create Security Group** allowing port 3306 from VPC CIDR
7. **Create RDS Instance** with:
   - db.r5.large instance class
   - io1 storage with 100GB and 1000 IOPS
   - MySQL 8.0.34 engine
   - Security group and subnet group references
   - Secure password from Secrets Manager

## Security Considerations

- All resources deployed within private VPC
- Security group restricts access to VPC internal traffic only
- Database credentials stored in Secrets Manager (not hardcoded)
- No public accessibility configured
- Provisioned IOPS storage provides consistent performance

## Parameter Strategy

Since no parameters were specified in the business need, I will:
- Use hardcoded safe defaults for all properties
- Avoid creating unnecessary Parameters that would require manual input
- Use dynamic references for secrets/passwords instead of Parameters

This architecture ensures network isolation, high availability across AZs, and production-ready performance with Provisioned IOPS storage.
