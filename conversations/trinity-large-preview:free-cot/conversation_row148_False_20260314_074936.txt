# Jenkins Multi-AZ Auto Scaling Deployment Architecture Plan

## Key AWS Services Required

1. **VPC** - Core networking foundation
2. **EC2** - Jenkins server instances
3. **Auto Scaling Group** - Ensures minimum instance availability
4. **IAM Role & Instance Profile** - SSM access for management
5. **Launch Template** - Defines EC2 instance configuration and bootstrap script
6. **Internet Gateway** - Public internet access
7. **Route Tables** - Network traffic routing
8. **NACLs** - Network traffic control
9. **Security Groups** - Instance-level firewall rules
10. **SSM Parameter Store** - Dynamic AMI fetching

## Architecture Overview

The deployment will create a highly available Jenkins environment across two Availability Zones using Auto Scaling to maintain at least one running instance. The solution leverages SSM for instance management and dynamic AMI retrieval to ensure the latest Amazon Linux 2 image is used.

## Detailed Component Breakdown

### 1. VPC Configuration
- **CIDR Block**: Default to `10.0.0.0/16` (safe default for small deployments)
- **Subnets**: Two public subnets across two AZs
- **Internet Gateway**: Attached to VPC for public internet access
- **Route Tables**: Main route table with default route to IGW
- **NACLs**: Allow inbound/outbound HTTP/HTTPS and SSH traffic

### 2. EC2 Instances via Auto Scaling
- **AMI**: Dynamically fetched using SSM Parameter Store (`/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2`)
- **Instance Type**: Default to `t3.medium` (balanced performance for Jenkins)
- **IAM Role**: Attached with SSM permissions for Systems Manager access
- **Launch Template**: Contains instance configuration and user data script

### 3. Auto Scaling Group
- **Min Size**: 1 (ensures at least one instance running)
- **Max Size**: 2 (allows scaling if needed)
- **Desired Capacity**: 1 (maintains single instance by default)
- **VPC Zone Identifier**: References both public subnets for AZ distribution

### 4. Security Configuration
- **Security Groups**: Allow inbound HTTP (80), HTTPS (443), and SSH (22) from anywhere
- **IAM Policies**: Include `AmazonSSMManagedInstanceCore` for SSM access
- **User Data**: Bash script to install Jenkins, Java, and configure services

## Key Dependencies and Properties

1. **VPC Creation**: Must complete before subnet creation
2. **Internet Gateway**: Must be created and attached to VPC before route table configuration
3. **Subnets**: Created after VPC, used in Auto Scaling Group configuration
4. **Route Tables**: Created after VPC and IGW, associated with subnets
5. **IAM Role**: Created before Launch Template, referenced in instance configuration
6. **Launch Template**: Created after IAM Role, contains instance configuration
7. **Auto Scaling Group**: Created last, references all dependent resources

## Dynamic Resource Fetching Strategy

### AMI Retrieval
```yaml
Fn::ImportValue: !Sub "aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2"
```

### Availability Zones
```yaml
Fn::GetAZs: !Ref "AWS::Region"
```

### Key Pair Parameter
```yaml
Type: String
Default: "default-key"
```

## Security Best Practices

1. **IAM Role**: Least privilege principle with only SSM permissions
2. **Security Groups**: Restrict traffic to necessary ports only
3. **User Data**: Script installs Jenkins with default security configuration
4. **Network Isolation**: Public subnets with controlled access via NACLs

## Failure Recovery and High Availability

- **Auto Scaling**: Automatically replaces failed instances
- **Multi-AZ Deployment**: Spreads instances across Availability Zones
- **SSM Integration**: Enables remote management and troubleshooting
- **Launch Template**: Ensures consistent instance configuration

## Missing Properties and Safe Defaults

| Property | Default Value | Rationale |
|----------|---------------|-----------|
| VPC CIDR | 10.0.0.0/16 | Standard private network range |
| Instance Type | t3.medium | Balanced performance for Jenkins |
| Key Pair | default-key | Safe placeholder for SSH access |
| Jenkins Port | 8080 | Default Jenkins web interface port |
| SSH Port | 22 | Standard SSH access port |

This architecture ensures a robust, scalable Jenkins deployment that adheres to AWS best practices while maintaining simplicity and reliability through automated configuration and dynamic resource management.
