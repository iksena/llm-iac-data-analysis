# AWS Security Hub Controls Management - CloudFormation Architecture Plan

## Key AWS Services Required

1. **AWS Lambda** - Core compute service for the control management function
2. **AWS IAM** - Identity and Access Management for Lambda execution role and permissions
3. **AWS Security Hub** - Target service for control enable/disable operations
4. **AWS CloudWatch Logs** - Logging for Lambda function execution
5. **AWS Systems Manager Parameter Store** - Dynamic AMI fetching (if needed for future extensions)
6. **AWS CloudFormation** - Infrastructure as Code deployment

## Architecture Overview

The solution consists of a Lambda function triggered on-demand (via API Gateway or manual invocation) that updates Security Hub controls based on user-defined parameters. The CloudFormation template provisions all necessary resources with appropriate permissions and configurations.

## Detailed Resource Breakdown

### 1. Lambda Function
- **Runtime**: Python 3.12 (latest supported)
- **Handler**: `index.lambda_handler`
- **Memory**: 128 MB (minimum sufficient for API calls)
- **Timeout**: 30 seconds (adequate for batch control updates)

### 2. IAM Role & Permissions
- **Managed Policies**:
  - `AWSLambdaBasicExecutionRole` - CloudWatch Logs access
  - Custom inline policy for Security Hub control management
- **Optional Permissions Boundary**: Configurable boundary ARN parameter
- **Trust Policy**: Lambda service only

### 3. CloudWatch Logs
- **Log Group**: `/aws/lambda/SecurityHubControlsManager`
- **Retention**: Configurable parameter (default: 30 days)

### 4. Security Hub Control Management Logic
The Lambda function will:
- Accept parameters for CIS and Foundational Best Practices controls
- Use `batchUpdateStandardsControlAssociations` API
- Handle control enable/disable based on boolean parameters

## Parameter Design with Safe Defaults

```yaml
Parameters:
  DisableCIS140Controls:
    Type: String
    Default: "false"
    AllowedPattern: "^(true|false)$"
    Description: Disable CIS AWS Foundations Benchmark v1.4.0 controls (140.x series)
  
  DisableFoundationalControls:
    Type: String
    Default: "false"
    AllowedPattern: "^(true|false)$"
    Description: Disable AWS Foundational Security Best Practices controls
  
  PermissionsBoundaryArn:
    Type: String
    Default: ""
    Description: Optional IAM permissions boundary ARN
  
  LogRetentionInDays:
    Type: Number
    Default: 30
    MinValue: 1
    MaxValue: 3650
    Description: CloudWatch Logs retention period in days
```

## Lambda Function Code Structure

The function will use boto3 to interact with Security Hub:

```python
import boto3
import json

def lambda_handler(event, context):
    securityhub = boto3.client('securityhub')
    
    # Control update logic based on parameters
    controls_to_update = []
    
    if event.get('DisableCIS140Controls', 'false').lower() == 'true':
        controls_to_update.extend([
            {'StandardsControlArn': 'CIS-aws-foundations-benchmark/v/1.4.0/140.1', 'ControlStatus': 'DISABLED'},
            # ... additional CIS controls
        ])
    
    if event.get('DisableFoundationalControls', 'false').lower() == 'true':
        controls_to_update.extend([
            {'StandardsControlArn': 'AWS-foundational-security-best-practices/v/1.0.0/control1', 'ControlStatus': 'DISABLED'},
            # ... additional foundational controls
        ])
    
    if controls_to_update:
        response = securityhub.batch_update_standards_control_associations(
            StandardsControlAssociationUpdates=controls_to_update
        )
        return {
            'statusCode': 200,
            'body': json.dumps(response)
        }
    return {'statusCode': 200, 'body': 'No controls to update'}
```

## IAM Policy Design

```yaml
Policies:
  - PolicyName: SecurityHubControlsManagerPolicy
    PolicyDocument:
      Version: '2012-10-17'
      Statement:
        - Effect: Allow
          Action:
            - securityhub:BatchUpdateStandardsControlAssociations
            - securityhub:DescribeStandardsControls
          Resource: '*'
```

## CloudFormation Template Structure

```yaml
Resources:
  LambdaExecutionRole:
    Type: AWS::IAM::Role
    Properties:
      AssumeRolePolicyDocument:
        Version: '2012-10-17'
        Statement:
          - Effect: Allow
            Principal:
              Service: lambda.amazonaws.com
            Action: sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole
      PermissionsBoundary: !Ref PermissionsBoundaryArn
      Policies:
        - PolicyName: SecurityHubControlsManagerPolicy
          PolicyDocument:
            Version: '2012-10-17'
            Statement:
              - Effect: Allow
                Action:
                  - securityhub:BatchUpdateStandardsControlAssociations
                  - securityhub:DescribeStandardsControls
                Resource: '*'
  
  LambdaFunction:
    Type: AWS::Lambda::Function
    Properties:
      Handler: index.lambda_handler
      Runtime: python3.12
      Role: !GetAtt LambdaExecutionRole.Arn
      Code:
        ZipFile: !Sub |
          import boto3
          import json
          ${LambdaCode}
      Timeout: 30
      MemorySize: 128
  
  LogGroup:
    Type: AWS::Logs::LogGroup
    Properties:
      LogGroupName: !Sub /aws/lambda/${LambdaFunction}
      RetentionInDays: !Ref LogRetentionInDays
```

## Dependencies & Properties

1. **IAM Role before Lambda**: Lambda function must reference an existing IAM role
2. **Log Group after Lambda**: Log group name depends on Lambda function name
3. **Permissions Boundary**: Optional parameter, only applied if provided

## Dynamic Resource Fetching

While not immediately required for this template, the architecture supports:
- Future extensions using SSM Parameter Store for dynamic configuration
- Environment-specific parameter overrides via CloudFormation StackSets

## Security Best Practices Applied

1. **Least Privilege**: IAM policy only allows necessary Security Hub actions
2. **No Hardcoded Secrets**: All configuration via parameters
3. **Secure Defaults**: Boolean parameters default to 'false' (controls enabled)
4. **Logging**: CloudWatch Logs with configurable retention
5. **Runtime**: Latest Python runtime (3.12)

## Error Handling & Validation

The Lambda function includes:
- Input validation for boolean parameters
- Error handling for Security Hub API failures
- Graceful handling when no controls need updating
- Structured JSON responses for easy integration

This architecture provides a production-ready, secure, and maintainable solution for Security Hub control management that adheres to AWS best practices and CloudFormation linting requirements.
