## Architecture Plan

### 1. Query Definitions (AWS::Logs::QueryDefinition)
I'll create four saved query definitions for the dashboard widgets:

**Query 1: Top Source IPs by Outbound Traffic**
```sql
fields @timestamp, srcAddr, bytes
| filter interfaceId like 'eni-' and destinationPrefixListIds = []
| stats sum(bytes) as totalBytes by srcAddr
| sort totalBytes desc
| limit 20
```

**Query 2: Top Source IPs by Rejected Connections**
```sql
fields @timestamp, srcAddr, action
| filter interfaceId like 'eni-' and action = 'REJECT'
| stats count(*) as rejectedCount by srcAddr
| sort rejectedCount desc
| limit 20
```

**Query 3: Top Destination IPs by Outbound Traffic**
```sql
fields @timestamp, destAddr, bytes
| filter interfaceId like 'eni-' and destinationPrefixListIds = []
| stats sum(bytes) as totalBytes by destAddr
| sort totalBytes desc
| limit 20
```

**Query 4: Top Destination IPs by Rejected Connections**
```sql
fields @timestamp, destAddr, action
| filter interfaceId like 'eni-' and action = 'REJECT'
| stats count(*) as rejectedCount by destAddr
| sort rejectedCount desc
| limit 20
```

### 7. Security Considerations
