## Detailed Architecture Plan

### S3 Bucket Configuration

The ALBAccessLogsBucket will be configured with the following properties:

1. **Encryption**: SSE-S3 will be enabled to ensure all logs are encrypted at rest using AWS-managed keys. This is the default and most straightforward encryption method for S3.

2. **Ownership Controls**: The `BucketOwnerEnforced` ownership control will be set to ensure that the bucket owner has full control over all objects in the bucket, preventing object-level permissions from being set by other accounts.

3. **Public Access Block**: All public access will be blocked using the `PublicAccessBlockConfiguration` property, which includes:
   - BlockPublicAcls: true
   - IgnorePublicAcls: true
   - BlockPublicPolicy: true
   - RestrictPublicBuckets: true

4. **Lifecycle Configuration**: A lifecycle rule will be created to transition objects to Glacier after 30 days and expire them after 90 days, ensuring automatic deletion of old logs.

### Bucket Policy Configuration

The bucket policy will be carefully crafted to:

1. **Allow ELB Accounts**: Grant s3:PutObject permissions to the AWS ELB service accounts for the specific bucket and object prefix.

2. **Allow delivery.logs.amazonaws.com**: Grant s3:PutObject permissions to delivery.logs.amazonaws.com for the specific bucket and object prefix.

3. **Restrict Other Access**: By default, all other access will be denied, ensuring the bucket remains private.

### Compliance Considerations

The architecture ensures compliance with AWS requirements for ALB logging by:

1. **Proper Permissions**: Only the necessary AWS services and accounts can write to the bucket.

2. **Encryption**: All logs are encrypted at rest.

3. **Retention Policy**: Automatic deletion of logs after 90 days prevents indefinite storage of potentially sensitive data.

4. **Access Control**: The bucket is private with no public access, reducing the attack surface.

### Security Best Practices

The deployment follows these security best practices:

1. **Principle of Least Privilege**: Only the necessary permissions are granted.

2. **Encryption at Rest**: All data is encrypted using SSE-S3.

3. **Access Logging**: While not explicitly requested, S3 server access logging could be enabled on this bucket for audit purposes.

4. **Versioning**: Not enabled, as the retention policy handles log rotation.

5. **MFA Delete**: Not enabled, as the retention policy provides sufficient protection.

This architecture provides a secure, compliant, and automated solution for ALB access log storage that meets all specified requirements while following AWS best practices.
