# Jenkins CI/CD Server CloudFormation Architecture Plan

## Key AWS Services Required

1. **VPC** - Network isolation for the Jenkins instance
2. **EC2** - Jenkins server deployment
3. **Elastic IP** - Static public IP for consistent access
4. **Security Group** - Network access control
5. **IAM Role** - SSM access for secure management
6. **CloudWatch Alarm** - Automated failure detection
7. **SSM Parameter Store** - Dynamic AMI fetching
8. **Internet Gateway** - VPC internet connectivity
9. **Route Table** - Network routing configuration

## Architecture Dependencies & Properties

### Network Layer Dependencies
- VPC must be created before Subnets
- Internet Gateway must be attached to VPC before Route Table creation
- Route Table must be associated with Subnets before EC2 instance creation

### Security Dependencies
- Security Group must exist before EC2 instance creation
- IAM Role must be created before EC2 instance creation

### Recovery Dependencies
- CloudWatch Alarm requires Metric to exist (system-level metrics are always available)
- Alarm must be created before EC2 instance creation for proper association

## Safe AWS Defaults (Explicitly Stated)

Since specific properties are missing, I'll use these safe defaults:

- **VPC CIDR**: `10.0.0.0/16` (standard private range)
- **Public Subnet CIDR**: `10.0.1.0/24` (first /24 subnet)
- **Instance Type**: `t3.medium` (balanced performance for Jenkins)
- **SSH Key Pair**: `default-key` (safe default parameter)
- **Region**: `us-east-1` (safe default parameter)
- **Jenkins Port**: `8080` (standard Jenkins port)
- **Admin Email**: `admin@example.com` (safe default parameter)

## Dynamic Resource Fetching Strategy

### AMI Resolution
Use SSM Parameter Store to fetch the latest Amazon Linux 2 AMI:
```yaml
Fn::ImportValue: !Sub "resolve:ssm:/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2"
```

### Availability Zones
Use dynamic AZ selection:
```yaml
Fn::GetAZs: !Ref 'AWS::Region'
```

## Security & Access Design

### Security Group Configuration
- **HTTP (Port 8080)**: Allow from 0.0.0.0/0 for Jenkins web access
- **ICMP**: Allow from 0.0.0.0/0 for basic connectivity testing
- **SSH**: Not required since SSM provides secure access

### IAM Role for SSM
- Attach `AmazonSSMManagedInstanceCore` policy
- No inline policies needed for basic SSM functionality

### CloudWatch Alarm Configuration
- **Metric**: `StatusCheckFailed_System` (EC2 system status checks)
- **Period**: 60 seconds
- **Evaluation Periods**: 2 (2-minute failure threshold)
- **Alarm Actions**: EC2 instance recovery action
- **Alarm Description**: "Jenkins instance system failure detected"

## Recovery Mechanism

The CloudWatch alarm will trigger EC2 instance recovery when system-level failures occur (hardware issues, network problems). This ensures:
- Automatic instance restart without manual intervention
- Preservation of Elastic IP association
- Minimal downtime for CI/CD operations

## SSM Integration Benefits

Using SSM instead of SSH provides:
- No need for SSH key management
- Secure access through AWS credentials
- Integration with AWS Systems Manager Session Manager
- Audit trail through AWS CloudTrail
- No open ports for SSH attacks

## Template Structure Plan

The CloudFormation template will follow this logical order:
1. VPC and networking components
2. Security configurations (IAM Role, Security Group)
3. EC2 instance with recovery alarm
4. Elastic IP association

This ensures all dependencies are satisfied and the template passes cfn-lint validation.
