# ECS Fargate Cluster Deployment Architecture Plan

## Key AWS Services Required

1. **VPC** - Virtual Private Cloud for network isolation
2. **Subnets** - Public and private subnets across multiple AZs
3. **Internet Gateway** - For public internet access
4. **NAT Gateway** - For private subnet outbound internet access
5. **Route Tables** - Public and private route tables
6. **ECS Cluster** - The core Fargate cluster
7. **IAM Roles** - For ECS service and task execution
8. **Security Groups** - For network security
9. **Log Groups** - For container logging via CloudWatch

## Architecture Design

### VPC Configuration
- **CIDR Block**: 10.0.0.0/16 (default safe value)
- **Subnets**: 2 public + 2 private subnets across 2 AZs
- **AZ Selection**: Dynamic using `Fn::GetAZs` and `Fn::Select` to avoid hardcoding

### Network Dependencies
1. VPC must be created before Subnets
2. Internet Gateway must be attached to VPC before Route Table associations
3. NAT Gateway requires Elastic IP allocation before creation
4. Public Route Table must be associated with public subnets
5. Private Route Table must be associated with private subnets and point to NAT Gateway

### ECS Cluster Configuration
- **Cluster Name**: ecs-fargate-cluster (default)
- **Capacity Provider**: Fargate only (serverless)
- **Logging**: CloudWatch Logs integration

### IAM Configuration
- **ECS Task Execution Role**: With AmazonECSTaskExecutionRolePolicy
- **ECS Service Role**: With AmazonEC2ContainerServiceRolePolicy

### Security Groups
- **ECS Security Group**: Allow inbound traffic from ALB on ports 80/443
- **ALB Security Group**: Allow inbound HTTP/HTTPS from anywhere

## Dynamic Resource Fetching Strategy

### AMI Selection
- Use SSM Parameter Store to fetch latest Amazon Linux 2 AMI:
  - Parameter: `/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2`
  - This ensures we always get the latest patched AMI without hardcoding

### AZ Selection
- Use `Fn::GetAZs` to get all AZs in region
- Use `Fn::Select` to pick first 2 AZs for redundancy
- Example: `Fn::Select: [0, Fn::GetAZs: !Ref 'AWS::Region']`

## Safe Default Values

### Parameters with Defaults
```yaml
Parameters:
  VpcCidr:
    Type: String
    Default: 10.0.0.0/16
    AllowedPattern: (\d{1,3}\.){3}\d{1,3}\/\d{1,2}
  
  ClusterName:
    Type: String
    Default: ecs-fargate-cluster
  
  KeyPairName:
    Type: String
    Default: default-key
```

### Resource Properties
- **Instance Type**: Not applicable for Fargate (serverless)
- **CIDR Blocks**: Use safe defaults (10.0.0.0/16 for VPC, 10.0.X.0/24 for subnets)
- **Security Group Rules**: Use 0.0.0.0/0 for internet access (with caution)

## Resource Dependencies Chain

1. **VPC** → **Internet Gateway** → **Public Route Table**
2. **VPC** → **Subnets** (public and private)
3. **Elastic IP** → **NAT Gateway** → **Private Route Table**
4. **IAM Roles** → **ECS Cluster**
5. **Security Groups** → **ECS Services**

## Security Best Practices

- **IAM Policies**: Use least privilege principle
- **Security Groups**: Only allow necessary ports
- **Logging**: Enable CloudWatch Logs for all containers
- **Network Isolation**: Public vs private subnets separation

## Linting Compliance Strategy

- All Parameters have safe Default values
- No hardcoded AMI IDs or AZs
- No legacy S3 properties
- All stateful resources have DeletionPolicy/UpdateReplacePolicy pairs
- No unnecessary Fn::Sub usage
- Latest Lambda runtimes if used
- Exact IAM policy ARNs

This architecture ensures a production-ready, secure, and maintainable ECS Fargate deployment that follows AWS best practices and passes cfn-lint validation.
