# Jenkins CI/CD Server Deployment Architecture Plan

## Key AWS Services Required

1. **VPC** - Network isolation for the Jenkins instance
2. **EC2 Instance** - Jenkins server deployment
3. **Elastic IP** - Static public IP for consistent access
4. **Security Group** - Network access control
5. **IAM Role** - SSM access for secure management
6. **CloudWatch Alarm** - Automated failure detection
7. **SSM Parameter Store** - Dynamic AMI fetching
8. **Internet Gateway** - VPC internet connectivity
9. **Route Table** - Network routing configuration

## Properties and Dependencies

### Critical Dependencies:
1. VPC must be created before Security Group
2. VPC must be created before EC2 instance
3. Internet Gateway must be attached to VPC before Route Table creation
4. Route Table must be associated with VPC before EC2 instance provisioning
5. Security Group must exist before EC2 instance creation
6. IAM Role must be created before EC2 instance creation

### Property Dependencies:
- Elastic IP requires VPC parameter (true for EC2-VPC)
- CloudWatch Alarm requires EC2 instance ID
- Route Table requires VPC ID
- Security Group requires VPC ID

## Safe AWS Defaults (Explicitly Stated)

Since specific properties are missing, I will use these safe defaults:

**VPC Configuration:**
- CIDR Block: `10.0.0.0/16` (standard private range)
- Subnet CIDR: `10.0.1.0/24` (single public subnet)

**EC2 Configuration:**
- Instance Type: `t3.medium` (balanced performance for Jenkins)
- Key Pair: `default-key` (safe default parameter)
- AMI: Latest Amazon Linux 2 via SSM Parameter Store
- Volume Size: `20` GB (standard for Jenkins)

**Security Group:**
- SSH Port: `22` (required for initial setup)
- HTTP Port: `8080` (Jenkins web interface)
- ICMP: `0-65535` (for troubleshooting)

**CloudWatch Alarm:**
- Metric: `StatusCheckFailed_System`
- Period: `60` seconds
- Evaluation Periods: `2`
- Threshold: `1` (any failure triggers)

## Dynamic Resource Fetching Strategy

### AMI Retrieval:
Use SSM Parameter Store to fetch the latest Amazon Linux 2 AMI:
```
resolve:ssm:/aws/service/ami-amazon-linux-latest/amzn2-ami-hvm-x86_64-gp2
```

### Availability Zones:
Use `Fn::GetAZs` and `Fn::Select` to dynamically select AZs:
```
Fn::Select: [0, Fn::GetAZs: !Ref 'AWS::Region']
```

## Architecture Flow

1. **VPC Creation** - Isolated network environment
2. **Internet Gateway** - Provides internet access
3. **Route Table** - Routes traffic to internet
4. **Security Group** - Controls inbound/outbound traffic
5. **IAM Role** - Enables SSM access
6. **EC2 Instance** - Jenkins server deployment
7. **Elastic IP** - Static public IP assignment
8. **CloudWatch Alarm** - Automated recovery mechanism

## Security Considerations

- IAM Role with SSM permissions eliminates SSH key management
- Security Group restricts access to necessary ports
- CloudWatch Alarm enables automated recovery without manual intervention
- Elastic IP ensures consistent access despite instance replacement

## Recovery Mechanism

The CloudWatch Alarm monitors `StatusCheckFailed_System` metric. When triggered:
1. Alarm activates after 2 consecutive failures (2 minutes)
2. AWS automatically initiates recovery action
3. EC2 service preserves instance ID and private IP
4. Elastic IP reassociates automatically
5. Jenkins service restarts on recovered instance

This architecture ensures high availability through AWS-managed recovery while maintaining consistent access points and secure management capabilities.
