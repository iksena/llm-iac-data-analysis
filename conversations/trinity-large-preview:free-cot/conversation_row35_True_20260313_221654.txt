The template will follow this logical deployment sequence:

   - Create Internet Gateway and attach to VPC

   - Associate Subnets with Route Table
   - Create RDS MySQL instance with automated snapshots enabled

- Use Secrets Manager for database credentials (not plain text parameters)
- Ensure RDS is deployed in public subnets only if explicitly required (otherwise consider private subnets)
- Set appropriate backup retention period for automated snapshots
- Use least privilege principles for any IAM roles (though none are explicitly required here)

- All Parameters will have Default values matching AllowedPattern if specified

- Only valid AWS resource types and properties will be used
